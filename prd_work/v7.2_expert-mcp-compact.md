# PRD：v1.1.1_hotfix_expert-mcp

## 1. 文档信息

```text
产品：copilot-desktop
模块：screen/Hermes
子模块：Experts / Expert Teams / Expert Runs / Artifacts / Workbench
版本：v1.1.1_hotfix_expert-mcp
后端合约：NoDeskClaw Expert MCP Gateway v6.1-client-contract
适用端：copilot-desktop
服务端：nodeskclaw Expert MCP Gateway
目标：修复 Expert MCP Gateway 对接链路，统一专家与专家团队调用方式。
```

---

## 2. 背景

copilot-desktop 已接入 nodeskclaw，但当前测试出现以下问题：

```text
1. Experts 页面展示了所有 skill/tool 明细，而不是专家清单。
2. 点击专家详情时报：
   Expert hermes-xieyi-customer-profiling not found
3. 召唤专家仍走旧本地 Profile 安装 / 启动流程。
4. 专家与专家团队调用逻辑分裂，未对齐 nodeskclaw v6.1 Expert MCP Gateway 合约。
```

nodeskclaw 新合约已经明确：

```text
1. Expert Gateway 基础路径：
   /api/v1/expert

2. 获取专家 / 专家团队目录：
   POST /api/v1/expert/mcp
   method = tools/list

3. 获取某个专家 / 专家团队能力：
   POST /api/v1/expert/mcp/{slug}
   method = tools/list

4. 调用某个专家 / 专家团队能力：
   POST /api/v1/expert/mcp/{slug}
   method = tools/call

5. Desktop 不决定 Runtime 路由。
6. expert 与 expert_team 在调用层完全一致。
7. Desktop 只根据 annotations.kind 做 UI 分类。
```

---

## 3. 本次 Hotfix 核心结论

### 3.1 专家与专家团队统一建模

v1.1.1 不再把专家和专家团队拆成两套远端调用链路。

统一对象：

```text
RemoteCatalogItem
```

通过 `annotations.kind` 区分：

```text
expert       → Experts 页面
expert_team  → Expert Teams 页面
```

---

### 3.2 能力统一建模

专家能力与专家团队能力也统一建模。

统一对象：

```text
RemoteExpertSkill
```

通过 `annotations.kind` 区分：

```text
expert_skill       → 专家能力
expert_team_skill  → 专家团队能力
```

---

### 3.3 调用统一接口

Desktop Main Process 统一调用：

```text
callCatalogSkill({
  slug,
  skillName,
  prompt,
  context
})
```

其中：

```text
slug 可以是 expert slug，也可以是 expert_team slug。
skillName 来自 /api/v1/expert/mcp/{slug} tools/list 的 tool.name。
```

---

### 3.4 不再由 Desktop 决定 Runtime 路由

Desktop 禁止传：

```text
_routing
_execution
route_config
profile
agent_alias
agentAlias
runtime
runtime_name
runtimeName
api_server_url
apiServerUrl
workspace_override
workspaceOverride
```

这些字段必须在 Main Process 层校验，不允许只依赖 Renderer。

## 4. 版本目标

### 4.1 产品目标

```text
1. Experts 页面只展示 annotations.kind=expert 的目录项。
2. Expert Teams 页面只展示 annotations.kind=expert_team 的目录项。
```

## 7. 核心数据模型

### 7.1 JsonRpc 类型

```ts
export interface JsonRpcRequest<TParams = unknown> {
  jsonrpc: "2.0";
  id: string | number;
  method: string;
  params?: TParams;
}

export interface JsonRpcSuccess<TResult = unknown> {
  jsonrpc: "2.0";
  id: string | number | null;
  result: TResult;
}

export interface JsonRpcError {
  jsonrpc: "2.0";
  id: string | number | null;
  error: {
    code: number;
    message: string;
    data?: {
      errorCode?: string;
      [key: string]: unknown;
    };
  };
}

export type JsonRpcResponse<TResult = unknown> =
  | JsonRpcSuccess<TResult>
  | JsonRpcError;
```

## FR-004：新增 callCatalogSkill

### 目标

新增统一调用 API：

```ts
callCatalogSkill(input: CallCatalogSkillInput): Promise<CallCatalogSkillResult>
```

### IPC

新增：

```text
hermes-experts:call-catalog-skill
```

### Preload

新增：

```ts
callCatalogSkill(input: {
  slug: string;
  catalogKind: "expert" | "expert_team";
  skillName: string;
  prompt: string;
  context?: Record<string, unknown>;
  sessionId?: string;
}): Promise<CallCatalogSkillResult>;
```

### Main Process 行为

```text
1. 根据 slug 创建本地 run。
2. 插入 mcp.call.started event。
3. 调用 ExpertMcpClient.callSkill。
4. 读取 result.content 中的 text。
5. 读取 structuredContent。
6. 更新 run completed 或 failed。
7. 插入 artifact source=expert_mcp_response。
8. 返回 runId / responseText / structuredContent。
```

### 验收

```text
1. expert 和 expert_team 都走 callCatalogSkill。
2. expert_team 不再走本地 delegation。
3. 调用完成后 Run 立即 completed。
4. Run Detail 可见 Markdown 响应。
5. Artifact 可见 expert_mcp_response。
```

## FR-005：summonExpert / summonTeam 兼容改造

### 目标

保持现有 UI 不大面积崩溃，同时将旧召唤 API 内部改为新统一调用。

### summonExpert 新逻辑

```text
1. getExpert(input.expertId)
2. slug = input.slug || expert.catalogSlug || expert.slug || expert.expertId
3. skillName = input.skillName || resolveDefaultCallableSkill(slug, "expert_skill")
4. callCatalogSkill({ slug, catalogKind: "expert", skillName, prompt, context })
```

```text
1. getExpertTeam(input.teamId)
2. slug = input.slug || team.catalogSlug || team.slug || team.teamId
3. skillName = input.skillName || resolveDefaultCallableSkill(slug, "expert_team_skill")
4. callCatalogSkill({ slug, catalogKind: "expert_team", skillName, prompt, context })
```


## FR-007：统一 Call Drawer

### 目标

新增或改造统一调用抽屉：

```text
ExpertCatalogCallDrawer
```

```text
ExpertCallDrawer
ExpertTeamCallDrawer
```

### Props

```ts
interface ExpertCatalogCallDrawerProps {
  open: boolean;
  catalogItem: {
    slug: string;
    kind: "expert" | "expert_team";
    displayName: string;
    callableSkillCount?: number;
```

### 行为

```text

8. 失败后展示友好错误。
```

### 13.1 ExpertMcpClient

```ts
export class ExpertMcpClient {
  constructor(
    private readonly baseUrl: string,
    private readonly getToken: () => Promise<string>
  ) {}

  private async postJsonRpc<TResult>(
    path: string,
    body: JsonRpcRequest
  ): Promise<TResult> {
    const token = await this.getToken();

    const res = await fetch(`${this.baseUrl}${path}`, {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        "Authorization": `Bearer ${token}`
      },
      body: JSON.stringify(body)
    });

    if (!res.ok) {
      throw new HermesExpertsError(
        res.status === 401 ? "NODESKCLAW_UNAUTHORIZED" : "NODESKCLAW_BACKEND_UNREACHABLE",
        `HTTP ${res.status}: ${await res.text()}`
      );
    }

    const payload = (await res.json()) as JsonRpcResponse<TResult>;

    if ("error" in payload) {
      throw normalizeExpertJsonRpcError(payload.error);
    }

    return payload.result;
  }

  async health(): Promise<ExpertHealthResponse> {
    const token = await this.getToken();

    const res = await fetch(`${this.baseUrl}/expert/health`, {
      headers: {
        "Authorization": `Bearer ${token}`
      }
    });

    if (!res.ok) {
      throw new HermesExpertsError(
        res.status === 401 ? "NODESKCLAW_UNAUTHORIZED" : "EXPERT_GATEWAY_HEALTH_FAILED",
        `HTTP ${res.status}: ${await res.text()}`
      );
    }

    return (await res.json()) as ExpertHealthResponse;
  }

  async initializeRoot(): Promise<unknown> {
    return this.postJsonRpc("/expert/mcp", {
      jsonrpc: "2.0",
      id: `init-${Date.now()}`,
      method: "initialize",
      params: {
        clientInfo: {
          name: "copilot-desktop",
          version: process.env.npm_package_version ?? "v1.1.1"
        }
      }
    });
  }

  async listCatalog(): Promise<RemoteCatalogItem[]> {
    const result = await this.postJsonRpc<ToolsListResult<ExpertCatalogAnnotations>>(
      "/expert/mcp",
      {
        jsonrpc: "2.0",
        id: `catalog-${Date.now()}`,
        method: "tools/list"
      }
    );

    return result.tools
      .map(mapCatalogTool)
      .filter((item): item is RemoteCatalogItem => item != null);
  }

  async listSkills(slug: string): Promise<RemoteExpertSkill[]> {
    const result = await this.postJsonRpc<ToolsListResult<ExpertSkillAnnotations>>(
      `/expert/mcp/${encodeURIComponent(slug)}`,
      {
        jsonrpc: "2.0",
        id: `skills-${Date.now()}`,
        method: "tools/list"
      }
    );

    return result.tools
      .map(mapSkillTool)
      .filter((item): item is RemoteExpertSkill => item != null);
  }

  async callSkill(input: {
    slug: string;
    skillName: string;
    arguments: ExpertCallArguments;
  }): Promise<ToolsCallResult> {
    assertNoRouteOverride(input.arguments);

    return this.postJsonRpc<ToolsCallResult>(
      `/expert/mcp/${encodeURIComponent(input.slug)}`,
      {
        jsonrpc: "2.0",
        id: `call-${Date.now()}`,
        method: "tools/call",
        params: {
          name: input.skillName,
          arguments: input.arguments
        }
      }
    );
  }
}
```

### 13.2 Route Guard

```ts
const FORBIDDEN_ROUTE_KEYS = new Set([
  "_routing",
  "_execution",
  "route_config",
  "profile",
  "agent_alias",
  "agentAlias",
  "runtime",
  "runtime_name",
  "runtimeName",
  "api_server_url",
  "apiServerUrl",
  "workspace_override",
  "workspaceOverride"
]);

export function assertNoRouteOverride(args: Record<string, unknown>): void {
  const rootKeys = Object.keys(args).filter((key) => FORBIDDEN_ROUTE_KEYS.has(key));

  const context = args.context;
  const contextKeys =
    context && typeof context === "object" && !Array.isArray(context)
      ? Object.keys(context as Record<string, unknown>).filter((key) =>
          FORBIDDEN_ROUTE_KEYS.has(key)
        )
      : [];

  const found = [...rootKeys, ...contextKeys];

  if (found.length > 0) {
    throw new HermesExpertsError(
      "EXPERT_ROUTE_OVERRIDE_FORBIDDEN",
      `Forbidden route override keys: ${found.join(", ")}`
    );
  }
}
```

### 13.3 callCatalogSkill

```ts
export async function callCatalogSkill(
  input: CallCatalogSkillInput
): Promise<CallCatalogSkillResult> {
  const run = createExpertRun({
    runType: input.catalogKind === "expert_team" ? "team" : "single_expert",
    expertId: input.catalogKind === "expert" ? input.slug : undefined,
    teamId: input.catalogKind === "expert_team" ? input.slug : undefined,
    profileId: "remote",
    sessionId: input.sessionId,
    title: input.slug,
    userPrompt: input.prompt,
    status: "running",
    catalogSlug: input.slug,
    catalogKind: input.catalogKind,
    skillName: input.skillName,
    executionMode: "remote_mcp"
  });

  insertRunEvent({
    runId: run.runId,
    eventType: "mcp.call.started",
    payload: {
      slug: input.slug,
      catalogKind: input.catalogKind,
      skillName: input.skillName
    }
  });

  try {
    const result = await getExpertMcpClient().callSkill({
      slug: input.slug,
      skillName: input.skillName,
      arguments: {
        prompt: input.prompt,
        context: {
          source: "copilot-desktop",
          ...input.context
        }
      }
    });

    const responseText = extractTextContent(result);
    const structuredContent = result.structuredContent;

    updateExpertRunStatus(run.runId, "completed", {
      responseText,
      structuredContentJson: structuredContent
        ? JSON.stringify(structuredContent)
        : undefined,
      invocationId: structuredContent?.invocationId
    });

    insertRunEvent({
      runId: run.runId,
      eventType: "mcp.call.completed",
      payload: {
        slug: input.slug,
        catalogKind: input.catalogKind,
        skillName: input.skillName,
        structuredContent
      }
    });

    insertArtifact({
      runId: run.runId,
      profileId: "remote",
      title: `${input.slug} - ${input.skillName}`,
      artifactType: "markdown",
      previewText: responseText.slice(0, 4000),
      source: "expert_mcp_response"
    });

    return {
      ok: true,
      runId: run.runId,
      responseText,
      structuredContent
    };
  } catch (err) {
    const errorCode = isHermesExpertsError(err)
      ? err.code
      : "EXPERT_MCP_CALL_FAILED";

    const message = err instanceof Error ? err.message : String(err);

    updateExpertRunStatus(run.runId, "failed", {
      error: {
        code: errorCode,
        message
      }
    });

    insertRunEvent({
      runId: run.runId,
      eventType: "mcp.call.failed",
      payload: {
        slug: input.slug,
        catalogKind: input.catalogKind,
        skillName: input.skillName,
        errorCode,
        message
      }
    });

    return {
      ok: false,
      runId: run.runId,
      errorCode,
      message
    };
  }
}
```
