# PRD v7.4.4：Chat Expert Gateway 参数修复与 Work Popover 样式修复

版本：v7.4.4
产品线：Copilot Desktop / Work Expert Workspace
实施对象：`ai-os-desktop`
实施范围：Chat 页面 Expert Gateway 分支参数修复 + Work Controls Popover 样式修复
不包含范围：Expert Gateway SSE / Timeline 接入，放到 v7.5
语言要求：本版本新增 UI 文案继续只使用 English，不新增 i18n，不修改 locale 文件

---

## 1. 背景

nodeskclaw Expert MCP Gateway v6.2 已经按 `event_stream` 方向设计：`tools/call` 默认创建 HermesTask，并立即返回 `taskId / eventSseUrl / artifactUrl`，Desktop 后续应订阅 `GET /api/v1/hermes/tasks/{task_id}/events` 获取任务过程。v6.2 计划中也明确说明 copilot-desktop TaskWindow UI 对接不在 nodeskclaw 本仓实现范围内，而是另仓实现。

但本次 v7.4.4 先不接 SSE。原因是当前 Chat 页面还有两个必须先修的阻塞问题：

```text
1. Expert Gateway 提交时报错：
   Forbidden route override keys: profile

2. Work Controls 的 Expert / Skill / Permission 下拉浮层样式与原选择框差异大，
   且弹出位置过远。
```

当前 `useWorkExpertGatewaySend.ts` 在 Expert Gateway 分支调用时，把 `profile: input.profile ?? HERMES_DEFAULT_PROFILE` 放进了 `context`。([GitHub][1]) `workExpertGatewayApi.ts` 又继续把 `input.context?.profile` 传给 `callCatalogSkill` 的 `context`。([GitHub][2]) 但 Main 侧 `expert-route-guard.ts` 明确禁止 `profile / runtime / agent_alias / route_config / _routing / _execution` 等 route override 字段。([GitHub][3])

所以本版目标很明确：**先修掉 Expert Gateway context 里的非法 `profile`，并把弹框样式修到可用；SSE / Timeline 在 v7.5 做。**

## 4. 问题一：Chat 提交报错修复

### 4.1 当前错误链路

```text
ComposerBar Send
  → selectedExpert + selectedSkill
  → useWorkExpertGatewaySend
  → workExpertGatewayApi.callExpertSkill
  → window.hermesExperts.callCatalogSkill
  → Main hermes-experts runtime
  → ExpertMcpClient.callSkill
  → assertNoRouteOverride
  → Forbidden route override keys: profile
```

原因：

```text
profile 被放入了 MCP arguments.context。
```

这是非法的。`profile` 只应该存在于本地 Hermes Chat / Profile Runtime 链路中，不应该作为远端 Expert Gateway MCP arguments 的 context 字段。

### 6.4 返回字段兼容

当前 v7.4.4 不接 SSE，但需要避免同步返回字段差异导致空响应。

在 `workExpertGatewayApi.ts` 中兼容：

```ts
const responseText =
  result.responseText ??
  (result as { contentText?: string }).contentText ??
  "(empty response)";
```

返回：

```ts
return {
  ok: true,
  responseText,
};
```

注意：

```text
v7.4.4 不处理 taskId / eventSseUrl / artifactUrl。
如果 v6.2 返回 async accepted 结构但没有 responseText，本版只允许显示一个 readable accepted message；
真正 SSE 订阅放 v7.5。
```

可选增强：

```ts
const structured = result.structuredContent as Record<string, unknown> | undefined;

if (
  structured &&
  (structured.taskId || structured.eventSseUrl || structured.streaming === true)
) {
  return {
    ok: true,
    responseText:
      `**Expert task accepted**\n\n` +
      `- Task: ${String(structured.taskNo ?? structured.taskId ?? "")}\n` +
      `- Status: ${String(structured.status ?? "accepted")}\n\n` +
      `Streaming timeline will be available in v7.5.`,
  };
}
```

这个增强不等于接 SSE，只是避免 v6.2 accepted result 被误判为空。

## 8. 修改方案二：Popover 定位与样式

### 8.1 修改 `WorkPopoverSelect.tsx`

文件：

```text
src/renderer/src/screens/Hermes/pages/Chat/components/work/WorkPopoverSelect.tsx
```

新增类型：

```ts
export type WorkPopoverPlacement = "top" | "bottom";

type FloatingRect = {
  left: number;
  width: number;
  maxHeight: number;
  top?: number;
  bottom?: number;
};
```

新增 Props：

```ts
type Props = {
  value?: string;
  options: WorkPopoverOption[];
  placeholder?: string;
  disabled?: boolean;
  searchable?: boolean;
  placement?: WorkPopoverPlacement;
  menuWidth?: number;
  maxMenuHeight?: number;
  onChange: (id: string | undefined) => void;
};
```

### 8.2 定位算法

关键规则：**top 模式不要用 `top = rect.top - maxHeight - gap`**。
要用 `bottom` 锚定触发按钮上沿。

```ts
const updatePosition = () => {
  const trigger = triggerRef.current;
  if (!trigger) return;

  const rect = trigger.getBoundingClientRect();
  const gap = 8;
  const width = menuWidth ?? Math.max(rect.width, 220);

  if (placement === "top") {
    const availableTop = Math.max(rect.top - gap - 8, 120);

    setFloatingRect({
      left: rect.left,
      bottom: window.innerHeight - rect.top + gap,
      width,
      maxHeight: Math.min(maxMenuHeight, availableTop),
    });

    return;
  }

  const availableBottom = window.innerHeight - rect.bottom - gap - 8;

  setFloatingRect({
    left: rect.left,
    top: rect.bottom + gap,
    width,
    maxHeight: Math.min(maxMenuHeight, Math.max(120, availableBottom)),
  });
};
```

原因：

```text
用 bottom 锚定可以让菜单底边始终贴近触发控件上方 8px。
菜单实际高度由内容决定，maxHeight 只控制滚动，不决定菜单起点。
```

### 8.3 Portal 渲染

```tsx
createPortal(
  <div
    ref={panelRef}
    className="hermes-work-popover-panel"
    style={{
      left: floatingRect.left,
      top: floatingRect.top,
      bottom: floatingRect.bottom,
      width: floatingRect.width,
      maxHeight: floatingRect.maxHeight,
    }}
  >
    ...
  </div>,
  document.body,
)
```

### 8.4 Outside Click

必须同时判断：

```text
1. trigger root
2. portal panel
```

```ts

  const target = event.target as Node;

```

