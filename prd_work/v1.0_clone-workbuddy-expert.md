# Copilot Desktop Hermes Experts Workspace PRD

版本：v1.0
模块：`src/renderer/src/screens/Hermes`
主题：基于 Hermes Agent 多 Profile Runtime 实现 WorkBuddy 风格“召唤专家 / 专家团队”
范围：仅实现 copilot-desktop / screen/Hermes 侧能力；nodeskclaw 本阶段只定义接口与报文，不实现服务端功能。
状态：方案 PRD / Cursor Spec 前置文档

---

# 1. 背景

当前 copilot-desktop 的 Hermes 模块已经具备基础 Agent 管理能力，包括 Chat、Sessions、Skills、GeneHub Skill Center、MCP、MCP Gateway、Tools、Memory、Providers、Models 等页面。

但当前产品形态仍偏向：

```text
Hermes Agent 管理控制台
+ 单 default profile 聊天入口
+ Skill / MCP / Model / Memory 配置页面
```

目标 v1.0 是将 Hermes 模块升级为：

```text
Hermes Experts Workspace
= 专家广场
+ 专家团队
+ 专家召唤
+ 多 Profile 运行时
+ 团队任务委派
+ Run Timeline
+ Artifact 成果交付
+ Skill / MCP / Memory 依赖治理
```

该版本不处理 WebOperator / Hermes Panel 深度联动，也不实现 nodeskclaw 后端逻辑。nodeskclaw 在本 PRD 中只作为远端专家目录、团队目录、安装计划、MCP Gateway、业务能力中心的接口提供方。

# 2. 产品目标

## 2.1 总目标

在 `screen/Hermes` 中实现类似 WorkBuddy 的“召唤专家 / 专家团队”产品能力，使用户可以：

```text
1. 浏览专家
2. 查看专家详情
3. 安装专家 Profile
4. 召唤专家进入 Chat
5. 浏览专家团队
6. 安装团队成员 Profile
7. 召唤专家团队执行任务
8. 查看每个专家执行状态
9. 查看团队汇总结果
10. 查看 Run Timeline / Artifacts / Audit
```

## 2.2 产品定位

```text
专家 Expert
  = 一个可安装、可启动、可召唤的 Hermes Profile

专家团队 Expert Team
  = Team Manifest + Leader Profile + Member Profiles + Collaboration Policy

召唤 Summon
  = 将当前 Chat 运行上下文绑定到某个 Expert Profile 或 Team Run

专家运行 Expert Run
  = 一次可追踪、可审计、可恢复的专家任务执行记录
```

## 2.3 不做内容

v1.0 暂不做：

```text
1. 不实现 nodeskclaw 后端。
2. 不实现 WebOperator + Hermes Panel 的跨系统自动化联动。
3. 不实现多 BrowserWindow。
4. 不实现 Electron Plugin Host。
5. 不实现复杂 DAG 多智能体图编排。
6. 不做企业组织级权限后台。
7. 不做云端记忆同步。
8. 不让 Renderer 直接访问 SQLite / 文件系统 / hermes-agent CLI。
```


暴露：

```ts
export const hermesExpertsApi = {
  listExpertCatalog: (input?: ExpertCatalogQuery) =>
    ipcRenderer.invoke("hermes-experts:list-catalog", input),

  getExpert: (expertId: string) =>
    ipcRenderer.invoke("hermes-experts:get-expert", expertId),

  listExpertTeams: (input?: ExpertTeamCatalogQuery) =>
    ipcRenderer.invoke("hermes-experts:list-teams", input),

  getExpertTeam: (teamId: string) =>
    ipcRenderer.invoke("hermes-experts:get-team", teamId),

  previewInstallExpert: (expertId: string) =>
    ipcRenderer.invoke("hermes-experts:preview-install-expert", expertId),

  installExpert: (expertId: string, options?: InstallOptions) =>
    ipcRenderer.invoke("hermes-experts:install-expert", expertId, options),

  previewInstallTeam: (teamId: string) =>
    ipcRenderer.invoke("hermes-experts:preview-install-team", teamId),

  installTeam: (teamId: string, options?: InstallOptions) =>
    ipcRenderer.invoke("hermes-experts:install-team", teamId, options),

  summonExpert: (input: SummonExpertInput) =>
    ipcRenderer.invoke("hermes-experts:summon-expert", input),

  summonTeam: (input: SummonTeamInput) =>
    ipcRenderer.invoke("hermes-experts:summon-team", input),

  listExpertRuns: (filter?: ExpertRunFilter) =>
    ipcRenderer.invoke("hermes-experts:list-runs", filter),

  getExpertRun: (runId: string) =>
    ipcRenderer.invoke("hermes-experts:get-run", runId),

  cancelExpertRun: (runId: string) =>
    ipcRenderer.invoke("hermes-experts:cancel-run", runId),

  retryExpertRun: (runId: string) =>
    ipcRenderer.invoke("hermes-experts:retry-run", runId),

  onExpertRuntimeEvent: (callback: (event: ExpertRuntimeEvent) => void) => {
    const handler = (_event, payload) => callback(payload);
    ipcRenderer.on("hermes-experts:event", handler);
    return () => ipcRenderer.removeListener("hermes-experts:event", handler);
  },
};
```

# 10. 专家召唤流程

## 10.1 单专家召唤

```text
用户点击“召唤专家”
  ↓
Renderer 调 hermesExperts.summonExpert
  ↓
Main 检查 Expert Manifest
  ↓
Main 检查 expert_instance 是否已安装
  ↓
未安装则返回 EXPERT_NOT_INSTALLED 或引导安装
  ↓
检查 Profile Runtime 状态
  ↓
未运行则 startProfile(profileId)
  ↓
创建 expert_run
  ↓
返回 activeProfileId / runId / sessionId
  ↓
Renderer 切换 Chat active context
```

返回：

```ts
interface SummonExpertResult {
  ok: boolean;
  expertId: string;
  profileId: string;
  runId: string;
  sessionId?: string;
  runtimeStatus: "running" | "starting" | "failed";
  message?: string;
}
```

## 10.2 专家团队召唤

v1.0 第一版采用 `leader_dispatch`。

```text
用户点击“召唤专家团队”
  ↓
检查 TeamManifest
  ↓
检查 Leader / Members 是否已安装
  ↓
未安装则提示安装团队
  ↓
启动 leader profile
  ↓
按需要启动 member profiles
  ↓
创建 team expert_run
  ↓
Chat active context 绑定 leader profile + teamRunId
  ↓
用户输入任务
  ↓
Leader 拆解并 delegate 到成员 profiles
  ↓
成员返回结果
  ↓
Leader 汇总最终答案
  ↓
Artifacts / Run Events 入库
```

# 19. 总体验收标准

```text
1. Hermes 左侧导航新增“专家 / 专家团队 / 专家运行”。
2. 专家页面可展示专家目录、分类、详情、推荐问题。
3. 专家团队页面可展示团队目录、成员、职责、编排模式。
4. 支持从 nodeskclaw API 拉取专家/团队配置。
5. 支持查看 Expert InstallPlan 和 Team InstallPlan。
6. 支持安装单个专家为 Hermes Profile。
7. 支持安装专家团队为多个 Hermes Profile。
8. 支持召唤单专家进入 Chat。
9. 支持召唤专家团队进入 Chat。
10. Chat 可根据 activeProfileId 发送到不同 Profile。
11. 支持 Team Leader Dispatch 到成员专家。
12. Expert Runs 页面可查看运行记录。
13. Run Timeline 可展示 profile start、delegate、tool、mcp、artifact、error 等事件。
14. Artifacts 可展示专家/团队输出成果。
15. Risk Report / Trust 状态可展示并阻断高风险动作。
16. 所有新增 IPC 均经过 Preload 暴露，不允许 Renderer 直接访问 Node / SQLite / 文件系统。
17. 所有 Profile 路径必须通过 profileHome 或现有 Profile Runtime 路径机制处理。
18. 所有 DB 变更必须有 migration。
19. 所有失败状态必须有可操作错误提示。
20. nodeskclaw 不可用时，已安装专家仍可本地召唤。
```

# 20. 最终产品形态

v1.0 完成后，Hermes 模块从：

```text
默认 Agent 聊天 + Runtime 配置
```

升级为：

```text
Hermes Experts Workspace
  ├─ Chat
  ├─ Experts
  ├─ Expert Teams
  ├─ Expert Runs
  ├─ Sessions
  ├─ Skills
  ├─ Skill Center
  ├─ MCP
  ├─ MCP Gateway
  ├─ Tools
  ├─ Memory
  ├─ Providers
  └─ Models
```

核心使用路径：

```text
浏览专家
  ↓
查看专家详情
  ↓
预览安装计划
  ↓
安装为 Hermes Profile
  ↓
召唤专家
  ↓
进入 Chat 执行任务
  ↓
查看 Run Timeline
  ↓
获取 Artifacts

浏览专家团队
  ↓
查看团队成员
  ↓
安装团队 Profile
  ↓
召唤团队
  ↓
Leader Dispatch
  ↓
成员专家执行
  ↓
Leader 汇总
  ↓
查看团队运行记录与成果
```

v1.0 的关键不是复制 WorkBuddy 的 UI，而是把 WorkBuddy 的“专家/专家团队”产品体验落到 Hermes Agent 的多 Profile Runtime、Skill、MCP、Memory、Session、Audit 体系中，形成 copilot-desktop 自有的专家办公工作台。
