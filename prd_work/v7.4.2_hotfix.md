# PRD v7.4.2 Hotfix：Chat-first Work Controls for MCP Expert Gateway

版本：v7.4.2 hotfix
产品线：Copilot Desktop / Work Expert Workspace
实施对象：ai-os-desktop / copilot-desktop
核心范围：恢复 Chat 主入口、ComposerBar 增加 Work 控件插槽、Chat ContextBar、发送链路增强、调用 MCP Expert Gateway
语言要求：本版本页面只提供 English UI，不新增多语言翻译，不新增 i18n key，不修改 locale 文件
实施工具：Cursor / Spec Driver

---

## 1. 本版结论

v7.4.2 不再做独立 Work Task 页面。

本版只做：

```text
Chat Page
  + Work Controls
  + Expert Selector
  + Expert Skill Selector
  + Chat ContextBar
  + Expert Gateway Send Path
```

核心原则：

```text
1. Chat is the main entry.
2. ComposerBar is the only input composer.
3. Work controls are embedded into ComposerBar.
4. ContextBar is embedded into Chat page.
5. Send is still triggered by the original Chat send button.
6. No pre-send from a separate task page.
7. No new TaskStream.
8. No second Composer.
9. No new i18n translation files.
10. New UI labels must be English literals or English constants.
```

---

## 2. 背景问题

v7.4.1 的问题是把 Chat 隐藏后，默认进入了任务首页，导致：

```text
1. 原 Hermes Chat 无法直接进入。
2. 调试发送链路变复杂。
3. 用户必须先创建任务才能看到 Chat。
4. WorkTaskStartComposer 抢占了原 ComposerBar 的职责。
5. 发送变成“先发送，再进入 Chat”，而不是“在 Chat 中发送”。
6. Chat、Task、Session 的关系被反向依赖。
```

v7.4.2 立即修正为：

```text
Chat = Task Window
Work Controls = Chat enhancement
Expert Gateway = optional send route
```

## 13. 发送链路设计

### 13.1 默认 Hermes Chat 发送

未选择专家或技能时：

```text
User clicks Send
  → stream.send(text, attachmentIds, modelId, attachmentMetas)
  → existing Hermes Chat path
  → Main
  → Hermes API Server 8642
  → SSE
  → ChatScrollArea
```

### 13.2 Expert Gateway 发送

选择 Expert + Skill 后：

```text
User clicks Send
  → handleSend detects workContext.useExpertGateway
  → append local user message
  → call workExpertGatewayApi.callExpertSkill
  → append expert response as assistant message
  → show error in StatusToast if failed
```

### 13.3 本版显示策略

v7.4.2 只要求核心调用跑通：

```text
1. 用户消息显示在 ChatScrollArea。
2. Expert Gateway 返回内容显示为 assistant message。
3. Expert Gateway 错误显示为 error message / StatusToast。
4. 不强制写入 Hermes state.db。
5. 不强制生成 Artifact。
6. 不强制 SSE。
```

### 13.4 后续版本

```text
v7.4.3:
  Expert Gateway result persistence into session history.

v7.5:
  Expert Gateway SSE streaming and Team events.

v7.6:
```

## 15. useWorkExpertGatewaySend

### 15.1 新增文件

```text
src/renderer/src/screens/Hermes/pages/Chat/hooks/useWorkExpertGatewaySend.ts
```

### 15.2 输入

```ts
export function useWorkExpertGatewaySend(input: {
  workContext: WorkChatContext;
  appendLocalMessage: (message: { role: "user" | "assistant"; content: string }) => void;
  setExternalRunState: (state: "creating" | "streaming" | "completed" | "error") => void;
}) {
  return {
    sendToExpertGateway,
  };
}
```

### 15.3 sendToExpertGateway

```ts
async function sendToExpertGateway(input: {
  text: string;
  attachmentIds: string[];
  modelId: string | null;
  sessionId?: string | null;
  profile?: string;
}): Promise<void>
```

### 15.4 执行步骤

```text
1. 校验 selectedExpert。
2. 校验 selectedSkill。
3. append local user message。
4. setExternalRunState("creating")。
5. call workExpertGatewayApi.callExpertSkill。
6. 成功后 append local assistant message。
7. 失败后 append local assistant error message。
8. setExternalRunState("completed" or "error")。
```
