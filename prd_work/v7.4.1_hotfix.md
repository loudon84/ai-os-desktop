# PRD v7.4.1 Hotfix：Work 工作台任务体验回归 Hermes Agent 会话底座

版本：v7.4.1 hotfix
产品线：Copilot Desktop / Work 专家工作台
主题：修复 Work 工作台任务首页与任务窗口体验
核心原则：WorkBuddy 只作为产品能力参考；Copilot Desktop 的智能体底座仍然是 hermes-agent；任务窗口必须复用 Hermes 原生会话与 SSE 能力，不允许另起一套低质量 Chat UI。
实施对象：`https://github.com/loudon84/ai-os-desktop`
实施工具：Cursor / Spec Driver

---

## 1. 问题结论

当前截图 1 的 Work 任务页面不可继续作为产品主线。主要问题：

```text
1. 它没有复用 Hermes 原有会话能力。
2. 它把任务首页做成了一个静态大输入框，缺少真实 Agent 会话区。
3. 它没有体现 Hermes Agent 的模型状态、连接状态、会话历史、消息搜索、附件、流式输出等成熟能力。
4. 它用原生 select、裸按钮、错位布局，交互质感低。
5. 左侧导航混乱，“任务 / 工作台 / 对话 / 专家运行 / 成果中心 / 能力管理”同时存在，信息架构不收敛。
6. 首页出现 `????`，说明 i18n / 文案 / 字符编码或 mock 数据存在明显问题。
7. “最近任务”只是卡片占位，没有与 Hermes session / task metadata 形成真实映射。
8. 输入框发送后如果不是进入 Hermes 会话流，那就不是智能体任务窗口。
```

正确方向：

```text
Work 工作台 = Hermes Agent 会话底座 + WorkBuddy 式任务启动能力 + 专家/专家团队/MCP 能力选择。

任务窗口 = Hermes 原生 Chat Session 的增强壳。
不是重新做一个 Chat。
不是把 Hermes 会话藏起来。
不是把任务做成一个表单。
```

## 2. v7.4.1 Hotfix 目标

### 2.1 产品目标

```text
1. 立即修复 Work 工作台首页体验。
2. 任务入口必须复用 Hermes 原有会话窗口。
3. Work 的“任务”本质上绑定 Hermes sessionId。
4. Work 首页只做任务创建、专家团队选择、场景推荐，不替代会话窗口。
5. 点击发送后必须进入 Hermes Chat 会话窗口，并带入 Work task metadata。
6. 左侧“对话”不应与“任务”形成两个割裂入口。
7. 专家 / 专家团队 / 技能 / MCP 作为任务上下文选择器进入 Hermes 会话。
8. 保留并强化原 Hermes 会话能力：模型、Provider、消息搜索、附件、历史、SSE、tool progress。
```

### 2.2 技术目标

```text
1. 不重写 hermes-agent 聊天链路。
2. 不绕过 IPC。
3. 不让 Renderer 直接访问 8642。
4. 不让 Renderer 直接访问 nodeskclaw token。
5. 继续通过 Preload API 调 Main Process。
6. Main Process 继续负责调用 Hermes API Server 8642。
7. Work Task 层只增加 task metadata、expert/team/skill context，不替代 Hermes session。
8. 优先复用现有 Hermes Chat 组件，而不是新建低质量 Task Chat 组件。
```

## 4. 核心设计修正

### 4.1 原错误方案

```text
Work 任务页
  ├─ 自己做任务首页
  ├─ 自己做输入框
  ├─ 自己做最近任务
  ├─ 自己做任务窗口

```

问题：

```text
1. 破坏 Hermes 既有会话能力。
2. 新 UI 没有成熟 Chat 能力。
3. 用户需要在“任务”和“对话”之间来回跳。
4. 任务不绑定 Hermes session。
5. 运行结果无法自然进入 Hermes 消息历史。
```

```text
Work 任务
  ├─ 任务首页：只做任务启动面
  └─ 任务窗口：直接复用 Hermes Chat Session 面板

  ├─ 原有消息流
  ├─ 原有模型状态
  ├─ 原有附件
  ├─ 原有 SSE
  ├─ 原有会话历史
  └─ 增强 Work Task Context Bar
```

任务和会话关系：

```text
WorkTask
  └─ sessionId: Hermes Session ID
```

任务窗口不是一个新页面，而是：

```text
HermesChatView + WorkTaskHeader + WorkTaskContextBar + ExpertTeamRuntimePanel + OutputPreviewPanel
```


## 9. API 调用链路

### 9.1 正确链路

```text
Renderer WorkTaskStartComposer
  → workTaskApi.startTask(input)
  → window.hermesDefaultChat 或 window.hermesAPI
  → Preload
  → ipcMain.handle(...)
  → src/main/hermes.ts
  → Hermes API Server 127.0.0.1:8642 /v1/chat/completions
  → SSE
  → Main event.sender.send(...)
  → Renderer Hermes Chat 组件
```

### 9.2 禁止链路

```text
Renderer → fetch("http://127.0.0.1:8642")
Renderer → localStorage token
Renderer → Node fs
Renderer → direct nodeskclaw API
Renderer → fake mock task result
```

### 9.3 v7.4.1 API 适配

新增：

```text
src/renderer/src/screens/Work/api/workTaskApi.ts
```

职责：

```ts
export const workTaskApi = {
  async startTask(input: WorkTaskStartInput): Promise<WorkTaskStartResult> {
    // 1. build first message
    // 2. call existing Hermes chat API
    // 3. return sessionId + task metadata
  },

  async resumeTask(taskId: string): Promise<WorkTaskResumeResult> {
    // load task metadata + sessionId
  },

  async listRecentTasks(): Promise<WorkTask[]> {
    // merge task metadata with existing Hermes sessions
  }
};
```
