# Copilot Desktop + NoDeskClaw Remote Experts 产品规划与技术实现导向文档

版本：v1.0
主题：NoDeskClaw Remote Experts / Hermes WorkBuddy Mode
适用范围：copilot-desktop、nodeskclaw-backend、Remote Hermes Expert Runtimes、GeneHub Skill Hub
状态：产品规划 + 技术实现导向文档
核心结论：专家 / 专家团队全部放在 nodeskclaw MCP Gateway；copilot-desktop 不再本地安装专家 Profile，只做专家工作台、MCP Client、任务追踪、产物消费与本地上下文桥接。

---

# 1. 总体目标

## 1.1 产品目标

将 copilot-desktop 的 `screen/Hermes` 从“本地 Hermes Agent 配置台”升级为“企业远端专家办公工作台”。

用户在 Desktop 中完成：

```text
1. 连接 nodeskclaw MCP Gateway
2. 浏览远端专家
3. 浏览远端专家团队
4. 召唤专家 / 专家团队
5. 查看任务运行状态
6. 查看执行时间线
7. 预览 / 下载 / 导入中心产物
8. 提交 Skill 到公司 GeneHub
9. 追踪 Skill Pull / 注册 / 发布状态
```

## 1.2 技术目标

形成统一链路：

```text
copilot-desktop
  ↓ JSON-RPC MCP + REST/SSE
nodeskclaw-backend
  ↓ HermesTask / route_snapshot
Remote Hermes Expert Runtime
  ↓ result / workspace artifacts
nodeskclaw Artifact Bridge
  ↓ server_artifacts
copilot-desktop preview / download / import
```

## 1.3 产品边界

明确不做：

```text
1. copilot-desktop 不安装专家 Profile。
2. copilot-desktop 不复制专家团队成员 Profile。
3. copilot-desktop 不决定 Runtime Skill 执行实例。
4. copilot-desktop 不向 MCP arguments 传 _routing / _execution / route_config。
5. copilot-desktop 不直接读取远端 Hermes workspace 文件。
6. copilot-desktop 不把 server_artifacts 伪装成本地文件。
7. 专家团队编排不在 desktop 做，由 nodeskclaw 后端完成。
```

# 3. 架构职责边界

## 3.1 copilot-desktop 职责

```text
1. nodeskclaw 连接配置
2. MCP initialize / tools/list / tools/call
3. 专家 / 团队卡片展示
4. 专家召唤交互
5. Runs 状态展示
6. Timeline 展示
7. Artifact 预览 / 下载 / 导入
8. GeneHub Skill List / Push / Pull 状态展示
9. screen / WebOperator / 本地文件上下文采集
10. 用户确认与审批提示
11. 本地最近使用缓存
```

## 3.2 nodeskclaw-backend 职责

```text
1. 用户 / 组织 / 权限 / 审计
2. GeneHub Skill 供应链
3. Skill 注册 / 安装 / 授权
4. Runtime Skill 注册到组织级 MCP
5. MCP tools/list / tools/call
6. 任务创建 / 队列 / Worker 执行
7. 执行路由锁定
8. 任务事件 / Timeline
9. 中心产物物化
10. Artifact 预览 / 下载 / 分享 / 入库审核
11. 专家团队后端编排
12. Remote Hermes Runtime 绑定与健康管理
```

## 3.3 Remote Hermes Runtime 职责

```text
1. 承载具体 Hermes runtime skill
2. 暴露 API_SERVER
3. 执行指定 skill
4. 生成结果文本
5. 可选生成 workspace 文件
6. 返回结果给 nodeskclaw worker
```

## 3.4 GeneHub Skill Hub 职责

```text
1. 公司内部 Skill 包管理
2. Skill 提交
3. Skill 审核
4. Skill 发布
5. Skill 版本管理
6. Skill Pull 到 Remote Hermes Runtime
7. Skill 注册为组织级 MCP Tool
```

# 5. 端到端核心流程

## 5.1 Desktop 连接 nodeskclaw

```text
用户输入 backendBaseUrl / token
  ↓
GET /api/v1/system/info
  ↓
读取 mcp descriptor
  ↓
GET /api/v1/hermes/mcp/health
  ↓
POST /api/v1/hermes/mcp initialize
  ↓
保存连接状态
  ↓
进入 Workbench
```

本地保存：

```ts
interface NodeskclawConnectionConfig {
  backendBaseUrl: string;
  tokenRef: string;
  deviceId: string;
  clientName: "copilot-desktop";
  clientVersion: string;
  lastConnectedAt?: string;
}
```

token 必须由 Main Process 保存，Renderer 不直接接触。

---

## 5.2 专家发现

```text
Workbench / Experts 加载
  ↓
POST /api/v1/hermes/mcp tools/list
  ↓
返回 MCP tools
  ↓
过滤 annotations.kind = expert
  ↓
转换为 RemoteExpert
  ↓
渲染专家卡片
```

若后端补充专家目录 API，则优先：

```text
GET /api/v1/hermes/client/experts
GET /api/v1/hermes/client/expert-teams
```

MCP `tools/list` 作为兜底。

## 5.3 召唤单专家

```text
用户点击专家卡片
  ↓

  ↓

  ↓
```

```json
        "device_id": "desktop-001",
        "workspace_id": "default",
        "conversation_id": "local-session-001"

    }
  }
}
```

禁止字段：

```text
_routing
_execution
route_config
```


UI 处理：

```text

GRANT_EXPIRED / REVOKED

  → 提示权限或连接问题

INTERNAL_ERROR
  → 展示错误详情与 retry
```

状态：

```text

mcp_unhealthy
unauthorized
permission_denied
task_timeout
artifact_not_found
```

UI 原则：

```text

```

# 10. 权限与安全

## 10.1 Desktop 安全要求

```text
1. token 仅 Main Process 保存。
2. Renderer 不直接访问 token。
3. Renderer 不直接访问 nodeskclaw API。
4. 所有请求通过 Preload API。
5. 下载文件必须校验目标路径。
6. import artifact 不能覆盖用户文件，默认创建副本。
7. MCP arguments 禁止包含 route override 字段。
```

## 10.2 nodeskclaw 安全要求

```text
1. skill:view 控制 tools/list 可见。
2. skill:invoke 控制 tools/call。
3. hermes_task:view 控制任务查看。
4. hermes_artifact:view 控制产物查看。
5. 写工具走 approval / grant。
6. Runtime Skill 执行路由只能来自 installation.routing_metadata。
7. 产物下载走权限校验。
8. KB 入库走审核。
```

# 14. 最终产品形态

完成 v1.1 后，Copilot Desktop 的 Hermes 模块变为：

```text
企业远端专家工作台
  ├─ 连接 nodeskclaw
  ├─ 发现组织专家
  ├─ 召唤专家执行任务
  ├─ 查看远端任务过程
  ├─ 消费中心产物
  ├─ 提交内部 Skill
  └─ 连接 screen / WebOperator 上下文
```

nodeskclaw 变为：

```text
企业专家能力中心
  ├─ Skill 供应链
  ├─ Runtime Skill 注册
  ├─ MCP Gateway
  ├─ Task / Artifact / KB
  ├─ 权限 / 审批 / 审计
  └─ Remote Hermes Runtime 管理
```

核心闭环：

```text
Skill 开发
  → GeneHub 提交
  → 管理员审核
  → 发布
  → Pull 到 Remote Hermes Runtime
  → 注册为组织级 MCP Tool
  → Desktop 显示为专家
  → 用户召唤
  → nodeskclaw 创建任务
  → Remote Hermes 执行
  → 中心产物物化
  → Desktop 预览 / 下载 / 导入
  → 可选进入知识库审核
```
