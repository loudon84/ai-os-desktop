# PRD v7.5：Expert Gateway v6.2 Stream Adapter

版本：v7.5
目标：将 Chat 页面 Expert Gateway 分支升级为 **OpenAI-compatible payload + event_stream accepted result + Main Process SSE 代理 + Chat Timeline + Artifact Preview/Download**。
范围：copilot-desktop / `screen/Hermes` / Expert Gateway 分支。
不做：完整 TaskWindow 重构、Expert Team 成员复杂 Timeline、Artifact Import、KB 入库、Cancel/Retry。

---

## 1. 背景与边界

nodeskclaw Expert Gateway v6.2 已经从同步 RPC 改为 `event_stream`：`tools/call` 创建 HermesTask，并立即返回 `taskId / eventSseUrl / artifactUrl`，Desktop 再订阅任务 SSE 展示过程。这个方向与当前远端专家规划一致：copilot-desktop 负责专家工作台、MCP Client、任务追踪、产物消费，不负责远端 Runtime 路由，也不向 MCP arguments 传 `_routing / _execution / route_config`。

copilot-desktop 当前架构要求 Renderer 不能直接访问 Node / token / 文件系统，所有能力必须经 Preload 暴露，再由 Main Process 处理 IPC 和远端请求。 所以 v7.5 的 SSE 必须由 Main Process 代理，Renderer 只接收事件。

---

## 2. v7.5 核心目标

```text
1. Chat 页面选择 Expert + Skill 后提交任务给 Expert MCP Gateway。
2. 提交 payload 必须是 OpenAI-compatible Chat Completion 格式。
3. MCP tools/call 返回 event_stream accepted result。
4. Main Process 根据 eventSseUrl 订阅 Task SSE。
5. Preload 暴露 expert task event subscription。
6. Chat 页面展示 task.started / task.progress / task.completed / task.failed。
7. task.artifact.ready 后显示 Artifact 卡片。
8. Artifact 支持 Preview / Download。
9. 普通 Hermes Chat IPC → 8642 链路不受影响。
```

---

## 3. 最终链路

```text
ComposerBar Send
  → if selectedExpert + selectedSkill
  → useWorkExpertGatewaySend
  → workExpertGatewayApi.callExpertSkill
  → OpenAI-compatible payload
  → window.hermesExperts.callCatalogSkill
  → Main hermes-experts runtime
  → ExpertMcpClient.callSkill
  → remote Expert MCP Gateway tools/call
  → accepted result: taskId / eventSseUrl / artifactUrl
  → Main subscribe Task SSE
  → Preload push task events
  → Chat Timeline render
  → Artifact Preview / Download
```

普通 Chat 未选择 Expert + Skill 时仍走原 Hermes Chat 流式链路，不进入该分支。

## 4. OpenAI-compatible Payload 规范

### 4.1 MCP JSON-RPC 外壳

`tools/call` 仍使用 MCP JSON-RPC：

```json
{
  "jsonrpc": "2.0",
  "id": "call_xxx",
  "method": "tools/call",
  "params": {
    "name": "customer-profiling",
    "arguments": {}
  }
}
```

### 4.2 `params.arguments` 必须是 OpenAI-compatible

`params.arguments` 不再使用：

```json
{
  "prompt": "...",
  "context": {}
}
```

必须改为：

```json
{
  "model": "expert-gateway",
  "messages": [
    {
      "role": "user",
      "content": "分析陕西天基通信科技有限责任公司的客户画像"
    }
  ],
  "stream": true,
  "metadata": {
    "source": "copilot-desktop",
    "conversation_id": "local-session-id",
    "device_id": "desktop-device-id",
    "permission_mode": "default",
    "attachment_ids": []
  }
}
```

### 4.3 允许字段

```ts
export interface OpenAICompatibleExpertPayload {
  model: string;
  messages: Array<{
    role: "system" | "developer" | "user" | "assistant" | "tool";
    content: string;
    name?: string;
    tool_call_id?: string;
  }>;
  stream: true;
  temperature?: number;
  top_p?: number;
  max_tokens?: number;
  metadata?: {
    source: "copilot-desktop";
    conversation_id?: string;
    workspace_id?: string;
    device_id?: string;
    model_id?: string;
    permission_mode?: "default" | "ask_each_time";
    attachment_ids?: string[];
  };
}
```

### 4.4 禁止字段

以下字段禁止出现在 `arguments` 任意层级：

```text
profile
runtime
runtime_name
runtimeName
agent_alias
agentAlias
route_config
_routing
_execution
api_server_url
workspace_override
workspaceOverride
```

## 5. Accepted Result 建模

新增文件：

```text
src/shared/hermes-experts/expert-task-stream-contract.ts
```

定义：

```ts
export type ExpertGatewayCallResult =
  | ExpertGatewayAcceptedResult
  | ExpertGatewaySyncResult
  | ExpertGatewayCallError;

export interface ExpertGatewayAcceptedResult {
  ok: true;
  mode: "event_stream";
  taskId: string;
  taskNo?: string;
  eventSseUrl: string;
  artifactUrl?: string;
  status: "accepted" | "queued" | "running";
  streaming: true;
  rawStructuredContent?: Record<string, unknown>;
}

export interface ExpertGatewaySyncResult {
  ok: true;
  mode: "sync_result";
  responseText: string;
  structuredContent?: Record<string, unknown>;
}

export interface ExpertGatewayCallError {
  ok: false;
  mode?: "event_stream" | "sync_result";
  error: string;
  errorCode?: string;
}
```

识别规则：

```ts
if (
  structured.taskId ||
  structured.eventSseUrl ||
  structured.streaming === true
) {
  return mode: "event_stream";
}

if (result.responseText || result.contentText) {
  return mode: "sync_result";
}
```

## 7. Event Contract

```ts
export type ExpertTaskEvent =
  | ExpertTaskStartedEvent
  | ExpertTaskProgressEvent
  | ExpertTaskArtifactReadyEvent
  | ExpertTaskCompletedEvent
  | ExpertTaskFailedEvent
  | ExpertTaskStreamClosedEvent;

export interface ExpertTaskStartedEvent {
  type: "task.started";
  taskId: string;
  taskNo?: string;
  message?: string;
  createdAt: string;
}

export interface ExpertTaskProgressEvent {
  type: "task.progress";
  taskId: string;
  stage?: string;
  message: string;
  progress?: number;
  createdAt: string;
}

export interface ExpertTaskArtifactReadyEvent {
  type: "task.artifact.ready";
  taskId: string;
  artifactId?: string;
  artifactUrl?: string;
  name?: string;
  mimeType?: string;
  createdAt: string;
}

export interface ExpertTaskCompletedEvent {
  type: "task.completed";
  taskId: string;
  message?: string;
  resultText?: string;
  createdAt: string;
}

export interface ExpertTaskFailedEvent {
  type: "task.failed";
  taskId: string;
  error: string;
  errorCode?: string;
  createdAt: string;
}

export interface ExpertTaskStreamClosedEvent {
  type: "task.stream.closed";
  taskId: string;
  reason?: string;
  createdAt: string;
}
```

事件映射规则：

```text
raw event task.started          → task.started
raw event task.progress         → task.progress
raw event task.artifact.ready   → task.artifact.ready
raw event task.completed        → task.completed
raw event task.failed           → task.failed
unknown event                   → task.progress(message=raw summary)
```

```ts
  permissionMode: "default" | "ask_each_time";
```

```ts
const payload: OpenAICompatibleExpertPayload = {
  model: input.modelId ?? "expert-gateway",

    },
```

## 10. Chat Timeline 显示

`useWorkExpertGatewaySend` 流程：

```text
1. append user message
2. callExpertSkill
3. 如果 mode=sync_result：
   append assistant responseText
4. 如果 mode=event_stream：
   append ExpertTaskAccepted block
   subscribeExpertTaskEvents
   后续事件进入 timeline
5. 如果 failed：
   append assistant error block
```

显示规则：

```text
task.started:
  显示 “Expert task started”

task.progress:
  显示阶段消息，可多条追加

task.artifact.ready:
  显示 Artifact card

task.completed:
  显示完成状态；如果 resultText 存在，显示最终结果

task.failed:
  显示错误卡片
```
