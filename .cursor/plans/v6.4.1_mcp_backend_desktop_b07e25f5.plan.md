# v6.4.1 MCP Backend Desktop 实施计划

## 当前状态总结

- 默认 `backendUrl` = `http://127.0.0.1:8000`（需改为 `http://192.168.0.118:4510`）
- 登录用 `POST /login` + `{ email, password }`（需改为 `POST /account-login` + `{ account, password }`）
- `McpSkillGatewayRuntimeConfig` 保留独立 `backendBaseUrl` 字段（需废弃，统一从 AuthEndpointConfig 读）
- `testRemoteMcpSkillGateway()` 直接请求远程 backend（需改为通过 Local Proxy）
- Login UI 只支持 email 格式验证（需改为 account 通用输入）

## 数据流（改造后）

```mermaid
flowchart TD
  User["用户输入 account + password"] --> LoginScreen
  LoginScreen --> |"desktopAuth.login"| AuthIPC["auth:login IPC"]
  AuthIPC --> AuthClient["HttpAuthClient"]
  AuthClient --> |"POST /account-login"| NodeskClaw["nodeskclaw :4510"]
  NodeskClaw --> |"unwrap response"| FetchMe["/auth/me"]
  FetchMe --> |"assertMember"| TokenStore["writeStoredSession"]
  TokenStore --> McpProxy["startMcpSkillGatewayProxy"]
  McpProxy --> Register["registerToHermes (default)"]

  subgraph proxyFlow ["MCP Proxy 转发"]
    HermesAgent["Hermes Agent"] --> |"POST /mcp"| LocalProxy["Local Proxy :48742"]
    LocalProxy --> |"inject Bearer"| RemoteMcp["nodeskclaw /api/v1/hermes/mcp"]
  end

  subgraph configSource ["配置单一源"]
    AuthConfig["AuthEndpointConfig.backendUrl"] --> LocalProxy
    AuthConfig --> McpProxy
    AuthConfig --> GatewayPage["HermesMcpGatewayPage 展示"]
  end
```

## 关键约束

- `backendUrl` 唯一源: `AuthEndpointConfig`（用户登录时选定）
- 测试期默认: `http://192.168.0.118:4510`
- Hermes `config.yaml` 只写 `http://127.0.0.1:48742/mcp`，禁止写 token/远程 URL
- Test remote MCP 必须经 Local Proxy，禁止 Renderer 直连 backend
- `email` 字段保留为兼容，新流程使用 `account`
