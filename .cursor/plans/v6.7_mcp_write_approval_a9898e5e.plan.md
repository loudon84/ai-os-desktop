# v6.7 MCP Write Tools Approval 实施计划

## 核心定位

Desktop **不做本地审批**，只负责：
1. MCP Proxy 转发时注入 Desktop/Profile 上下文 Header
2. 展示 nodeskclaw 返回的服务端授权状态
3. 感知 grant 撤销并刷新 UI
4. Profile-scoped Proxy URL 注册

---

## 阶段 1：Shared Contract 扩展

**文件**: [`src/shared/mcp-skill-gateway-runtime/mcp-gateway-operations-contract.ts`](src/shared/mcp-skill-gateway-runtime/mcp-gateway-operations-contract.ts)

- 扩展 `McpGatewayToolPreview` 新增 7 个授权字段：

```ts
approvalMode?: "none" | "server";
requiresApproval?: boolean;
authorized?: boolean;
grantStatus?: "active" | "missing" | "pending" | "revoked" | "expired" | "disabled";
grantId?: string;
approvalRequestId?: string;
expiresAt?: string | null;
```

- 扩展 `McpGatewayInvokeTestResult` 新增 3 个授权字段：

```ts
approvalRequired?: boolean;
approvalRequestId?: string;
grantStatus?: string;
```

- 扩展 `McpGatewayProxyLogEntry` 新增 4 个授权日志字段：

```ts
toolName?: string;
approvalRequestId?: string;
grantId?: string;
grantStatus?: string;
```

- 新增 4 个错误码到 `McpGatewayOperationsErrorCode`：
  - `MCP_OP_TOOL_APPROVAL_REQUIRED`
  - `MCP_OP_TOOL_GRANT_REVOKED`
  - `MCP_OP_TOOL_GRANT_EXPIRED`
  - `MCP_OP_TOOL_CONSTRAINT_VIOLATION`

**文件**: [`src/shared/mcp-skill-gateway-runtime/mcp-skill-gateway-runtime-contract.ts`](src/shared/mcp-skill-gateway-runtime/mcp-skill-gateway-runtime-contract.ts)

- 扩展 `McpSkillGatewayRuntimeConfig` 新增 3 个 v6.7 配置字段：

```ts
profileScopedProxyUrl?: boolean;       // default: true
showServerAuthorizationPanel?: boolean; // default: true
allowWriteToolInvokeTest?: boolean;     // default: false
```

---

## 阶段 2：Proxy 上下文 Header 注入

**文件**: [`src/main/mcp-skill-gateway-runtime/mcp-skill-gateway-proxy.ts`](src/main/mcp-skill-gateway-runtime/mcp-skill-gateway-proxy.ts)

核心改造：`authHeaders()` 和 `handleRequest()`

- 从 `/mcp` 请求的 URL query 中解析 `?profile=xxx`（兼容无 profile 时默认 `default`）
- `forwardJsonRpc()` 时在 Authorization 之外增加 4 个上下文 header：
  - `X-NoDeskClaw-Desktop-Device-Id` -- 来自 `getDeviceIdentity().deviceFingerprint`（`src/main/genehub/device-identity.ts`）
  - `X-NoDeskClaw-Hermes-Profile` -- 从 URL query `profile` 解析
  - `X-NoDeskClaw-Client: copilot-desktop`
  - `X-NoDeskClaw-MCP-Proxy-Version: v6.7`
- `tools/call` 响应中识别 `error.data.errorCode` 为 `MCP_TOOL_APPROVAL_REQUIRED` / `MCP_TOOL_GRANT_REVOKED` / `MCP_TOOL_GRANT_EXPIRED` / `MCP_TOOL_CONSTRAINT_VIOLATION`，写入结构化日志（含 toolName/approvalRequestId/grantId/grantStatus）

## 阶段 3：Profile-scoped Proxy URL 注册

**文件**: [`src/main/mcp-skill-gateway-runtime/mcp-skill-gateway-register.ts`](src/main/mcp-skill-gateway-runtime/mcp-skill-gateway-register.ts)

- `buildProxyUrl()` 支持 profile query 参数：
  - 当 `profileScopedProxyUrl === true` 时生成 `http://127.0.0.1:<port>/mcp?profile=<profileName>`
  - 当 `false` 时保持旧 URL `http://127.0.0.1:<port>/mcp`
- `listMcpSkillGatewayProfileRegistrations()` 兼容读取旧 URL（无 `?profile`）和新 URL
- 注册 URL 变化时设置 `hermesRestartRequired: true`

**文件**: [`src/main/mcp-skill-gateway-runtime/mcp-skill-gateway-config.ts`](src/main/mcp-skill-gateway-runtime/mcp-skill-gateway-config.ts)

- `resolveLocalMcpUrl()` 增加可选 `profile` 参数
- `DEFAULT_MCP_SKILL_GATEWAY_CONFIG` 增加 `profileScopedProxyUrl: true`、`showServerAuthorizationPanel: true`、`allowWriteToolInvokeTest: false`

## 阶段 7：UI — Server Authorization Panel

**新文件**: `src/renderer/src/screens/Hermes/pages/McpGateway/McpGatewayServerAuthorizationPanel.tsx`

展示内容（按 PRD 第 10 节）：
- 工具授权状态表格：toolName / permission / riskLevel / requiresApproval / authorized / grantStatus / grantId / expiresAt / approvalRequestId
- 三个操作按钮：刷新授权状态 / 打开审批中心 / 复制诊断报告
- **不提供** Approve/Reject/Revoke 按钮

按工具 `grantStatus` 显示不同状态：
- `read` 工具 → "可用 / 无需审批"
- `write` + `grantStatus === "active"` → "可用 / 已通过 nodeskclaw 审批"（显示 grantId/expiresAt）
- `write` + `grantStatus === "missing" | "pending"` → "待服务端审批"（操作：打开审批中心）
- `grantStatus === "revoked"` → "已撤销 / 不可用"

## 关键约束重申

- Desktop **不做** approve/reject/revoke
- Desktop **不缓存** grant 作为最终鉴权依据（缓存仅用于 UI 展示，标记 stale）
- Token 不进入 Renderer、不写入 config.yaml
- `X-NoDeskClaw-Desktop-Device-Id` 由 Main Process `device-identity.ts` 提供
- 撤销后 UI 必须显示 revoked（缓存 stale 时提示）
- 旧 URL 无 `?profile` 时兼容为 `default`
