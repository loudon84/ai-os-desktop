# v5.7.3 SidePanel Action 实施计划（不改 WebOperator 外壳布局）

## 约束（已确认）
- **不改 WebOperator 外壳布局**：保持当前 `WebOperatorScreen` 的网格结构与交互（SideRail 永久在线、SectionPanel 可折叠、focusedPanel 控制显示）。
- 允许在 `PageStructurePanel` 内新增子组件与交互入口（PRD 的 v5.7.3 需求主要落在 PageStructurePanel）。
- `getFrameHtml` UI 默认展示：**frame 元信息 + html**（已确认）。

## 目标能力（来自 PRD v5.7.3）
- **A. CRM 按钮验证增强**：在 `PageStructurePanel` 提供手工 selector 的 Find / Test Click / Test Type（优先实现 Test Click，Find/Type 按 PRD 逐步补齐）。
- **B. 获取指定 iframe 的 HTML**：实现端到端 `browser:get-frame-html`，并在 `PageStructurePanel` 增加 Frame HTML Inspector（支持 selector / outer|inner / maxLength / Copy）。

## 代码改动清单（按层）

### 1) Shared contracts
- 修改 `src/shared/browser/browser-frame-contract.ts`
  - 新增 `BrowserFrameHtmlRequest extends BrowserFrameTarget`：`selector?`、`outer?`、`maxLength?`
  - 新增 `BrowserFrameHtmlResult`：`ok`、`frameId/url/title/selector/html/text/capturedAt/truncated/error`
- 修改 `src/shared/browser/browser-action-contract.ts`
  - `BrowserActionType` 增加 `"getFrameHtml"`

### 2) Main：WebContentsView 核心与控制器
- 修改 `src/main/browser/browser-v57-core.ts`
  - 新增 `getFrameHtml(target: BrowserFrameHtmlRequest): Promise<BrowserFrameHtmlResult>`
  - 通过 `frameInspector.listFrames(wc)` + `resolveFrameByPath` 定位 frame
  - 通过 `WebFrameMain.executeJavaScript(GET_FRAME_HTML_SCRIPT(...))` 抽取 html/text
  - 复用现有错误模型（`WEB_CONTENTS_NOT_READY/FRAME_NOT_FOUND/FRAME_SCRIPT_BLOCKED/ELEMENT_NOT_FOUND/...`）并写入 `BrowserActionLog`（action=`getFrameHtml`）
- 修改 `src/main/browser/browser-controller.ts`
  - 透传：`getFrameHtml(...) { return this.v57.getFrameHtml(target); }`
- 修改 `src/main/browser/browser-ipc.ts`
  - 注册 `ipcMain.handle("browser:get-frame-html", ...)`
  - `unregister()` 中增加该 channel

### 3) Preload：`window.aiosBrowser` API
- 修改 `src/preload/browser-api.ts`
  - 新增 `getFrameHtml: (target) => ipcRenderer.invoke("browser:get-frame-html", target)`
- 修改 preload 类型声明（若项目有 `src/preload/index.d.ts` 或 `aiosBrowser` 声明文件）
  - 为 `window.aiosBrowser.getFrameHtml()` 添加类型签名，引用 shared contract。

### 4) Renderer：PageStructurePanel 交互增强
- 新增 `src/renderer/src/screens/WebOperator/panels/PageSelectorActionBar.tsx`
  - 输入：`selectedFrameId`、`onClickTestClick(selector)`
  - 调用：`window.aiosBrowser.clickElement({ selector, frame: { frameId: selectedFrameId } })`
  - 验收：BrowserActionLog 出现 clickElement；CRM 若触发上报，CrmEventPanel 可看到 lastEvent。
- 新增 `src/renderer/src/screens/WebOperator/panels/PageFrameHtmlInspector.tsx`
  - 输入：`selectedFrameId`、`frames`（用于展示元信息：frameId/url/title/origin 等）
  - 控件：selector（可空）、outer/inner 切换、maxLength（默认 100000）、Get HTML、Copy
  - 调用：`window.aiosBrowser.getFrameHtml({ frameId: selectedFrameId, selector, outer, maxLength })`
  - 展示：**frame 元信息 + html**（按确认默认），并显示错误 code/message。
- 修改 `src/renderer/src/screens/WebOperator/PageStructurePanel.tsx`
  - 在 `FrameTreePanel` 与 `ElementListPanel` 之间插入：
    - `<PageSelectorActionBar selectedFrameId={...} />`
    - `<PageFrameHtmlInspector selectedFrameId={...} frames={snapshot?.frames ?? []} />`
  - 复用现有 state：`selectedFrameId` / `snapshot` / `externalRefreshTrigger`。

## 验收标准（PRD 对齐）
- **Selector Test Click**：选择 CRM iframe → 输入 selector → Test Click → `BrowserActionLog` 有记录且页面实际触发。
- **getFrameHtml**：选择 iframe → Get HTML → 返回包含 `frameId/url/title/selector/html/capturedAt`；超长截断标记 `truncated`。
- **错误可诊断**：Frame 不存在、脚本被拦、selector 找不到元素时返回明确 error code。
- **外壳布局不变**：不修改现有 WebOperator 外壳结构与 SideRail/SectionPanel 折叠交互。

- `npm run typecheck`
- `npm run build`
- （可选）为 `getFrameHtml` 写 1-2 个单测（若 browser v57 core 有既有测试框架可复用）。
