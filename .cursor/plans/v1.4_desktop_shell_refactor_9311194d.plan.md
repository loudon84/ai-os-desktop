## Phase 1：Installer Precheck 增强（已完成）

### 1.1 任务（PRD §8 Phase 1）

```text
1. 在 build/installer.nsh 增加 customInit 检查逻辑
2. 抽离 VC Runtime 检查到 build/nsis/Include/VCRuntimeCheck.nsh
3. 抽离 runtime precheck 到 build/nsis/Include/RuntimePrecheck.nsh
4. 生成 $INSTDIR\runtime\installer-precheck.json
5. 保留现有 customInstall 和 customUnInstall 主逻辑
```

### 1.2 实施要点

**阻断项（customInit）**

- `!macro customInit` 调用 `EnsureVCRuntime`
- VC++ 2015–2022 x64 缺失时提示下载安装 `vc_redist.x64.exe`；用户拒绝或安装失败则 `Abort`

**非阻断项（customInstall → RunRuntimePrecheck）**

- Git / Python / uv：`where` 检测，结果写入 JSON（`pass` / `missing`），不阻断安装
- 端口 8642：`netstat` 检测（`available` / `occupied`），仅记入 JSON
- 综合 `result`：`git|python|uv` 任一 missing 或端口 occupied → `warning`；`vcRuntime` missing → `error`（customInit 已阻断，正常路径为 pass）

**保留项（customInstall / customUnInstall）**

- 目录：`bin/`、`runtime/`、`runtime/hermes-agent|logs|cache|downloads`
- Shim：`smc-copilot.cmd`、`hermes-desktop.cmd`、`hermes.cmd`
- `desktop-runtime.json`、HKCU `Software\SMC\Copilot`、`AddToPathSafe` / `RemoveFromPathSafe`、`WM_SETTINGCHANGE`、legacy shortcut 清理

### 1.3 产出文件

| 文件 | 职责 |
|------|------|
| [build/installer.nsh](build/installer.nsh) | `!include RuntimePrecheck.nsh`；`customInit` → `EnsureVCRuntime`；`customInstall` → `RunRuntimePrecheck` |
| [build/nsis/Include/VCRuntimeCheck.nsh](build/nsis/Include/VCRuntimeCheck.nsh) | `CheckVCRuntime` / `EnsureVCRuntime` |
| [build/nsis/Include/RuntimePrecheck.nsh](build/nsis/Include/RuntimePrecheck.nsh) | `RunRuntimePrecheck`；写出 `installer-precheck.json` 与 precheck 日志行 |

### 1.4 installer-precheck.json 契约

路径：`$INSTDIR\runtime\installer-precheck.json`（供 Phase 5 `installer-precheck-reader.ts` 读取）

```json
{
  "schemaVersion": "1.3.1",
  "windowsVersion": "<NSIS __NSD_VERSION>",
  "vcRuntime": "pass",
  "git": "pass",
  "python": "missing",
  "uv": "missing",
  "port8642": "available",
  "installDir": "...",
  "runtimeRoot": "...\\runtime",
  "binDir": "...\\bin",
  "result": "warning"
}
```

附加日志：`$INSTDIR\runtime\logs\nsis-install.log`（precheck 行 + install 完成行）

### 1.5 验收（PRD §8 Phase 1）

```text
pnpm build
pnpm package:win

安装器：
- 显示 assisted installer
- 可选择安装目录
- VC Runtime 缺失时提示安装（阻断）
- Git/Python/uv 缺失不阻断
- 安装后 PATH 正常
- 卸载后 PATH 清理
- 安装后存在 runtime/installer-precheck.json
```

### 1.6 与 PRD §5.3 范围说明

PRD 5.3.1 列出的「Windows 版本检查」「目录可写性检查」未纳入本次 NSIS 实现；Git/Python/uv/端口按 PRD 5.3 归入非阻断，由 `RunRuntimePrecheck` 在 `customInstall` 阶段写入 JSON，供 Electron Runtime Doctor / Phase 5 UI 承接。

## 关键约束

- Renderer 禁止导入 `electron`/`fs`/`path` — 只通过 `window.desktopWindow` / `window.hermesAPI`
- 所有事件监听必须返回 unsubscribe 函数，组件 unmount 时清理
- 不引入 Mantine/MUI/TanStack Router
- 保持 Chat `display:none` 挂载、Office lazy mount、Providers `visible` prop、Settings 保持挂载
- macOS 保留现有 drag-region 和 traffic lights 行为
- NSIS 不在安装阶段执行 git clone / venv / pip（属 Electron Bootstrap / Runtime Doctor）
