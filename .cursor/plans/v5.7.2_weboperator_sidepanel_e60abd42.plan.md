# v5.7.2 WebOperator SidePanel 重构计划

## 目标与边界
- **目标**：将 WebOperator 右侧面板从 `WebOperatorScreen.tsx` 直接拼装，改为 `screens/WebOperator/panels/` 独立模块统一组织，且 **同时显示 5 个独立面板**（非 Tab 切换）。
- **停用 ScreenshotPanel**：不删除文件，但确保运行链路/导航链路/聚焦滚动链路都不再引用 `ScreenshotPanel` 与 `"screenshot"` panel。
- **不改**：Main / Preload / Browser IPC / WebContentsView 控制链路（仅 Renderer 重构）。

## 面板顺序（固定）
1. `BrowserStatePanel`
2. `CrmEventPanel`
3. `HermesTaskPanel`
4. `PageStructurePanel`
5. `BrowserActionLog`

## 主要改动文件
- 新增 panels 模块（全新目录）：
  - `src/renderer/src/screens/WebOperator/panels/web-operator-panels-contract.ts`
  - `src/renderer/src/screens/WebOperator/panels/WebOperatorPanelCard.tsx`
  - `src/renderer/src/screens/WebOperator/panels/WebOperatorPanels.tsx`
  - `src/renderer/src/screens/WebOperator/panels/web-operator-panels.css`
  - `src/renderer/src/screens/WebOperator/panels/index.ts`
- 重构：
  - `src/renderer/src/screens/WebOperator/WebOperatorScreen.tsx`：删除各面板 import/refs/scroll 分支，改为渲染 `<WebOperatorPanels ... />`
  - `src/renderer/src/screens/WebOperator/WebOperatorSideRail.tsx`：移除 screenshot 图标入口；新增 `hermes-task` 图标入口
- Shared workspace 导航契约：
  - `src/shared/workspace/workspace-contract.ts`：`WorkspaceSecondaryPanel` **移除** `"screenshot"`、**新增** `"hermes-task"`
  - `src/shared/workspace/workspace-secondary-nav.ts`：`web-operator` 二级面板列表替换为 5 项，并补 `"hermes-task": "navigation.hermesTask"`
- i18n：
  - `src/shared/i18n/locales/en/navigation.ts`：新增 `navigation.hermesTask`
  - `src/shared/i18n/locales/zh-CN/navigation.ts`：新增 `navigation.hermesTask`
  - 如项目存在其它语言（PRD 指定 en/zh-CN/es/pt-BR），逐个补齐同 key（可先英文兜底）。
- CSS 拆分：
  - `src/renderer/src/screens/WebOperator/web-operator.css`：移除/迁移 `.web-operator-layout__side-panels`、`.web-operator-layout__panel*` 等旧面板样式；保留布局/handle/rail/toolbar 等。

## 关键实现要点
- `WebOperatorPanels.tsx` 负责：
  - 将 `focusedPanel` 归一化（兼容历史 `"screenshot"` → `"browser-state"`），并在侧栏展开时 `scrollIntoView` 到对应 card。
  - 将 `onRefreshSnapshot` 从 `CrmEventPanel` 透传到 `WebOperatorScreen` 既有的 `handleRefreshSnapshot()`，确保 CRM 面板的 Refresh 能驱动 `PageStructurePanel` 刷新。
  - 避免在 `useMemo` 内创建 `useRef`（PRD 已提示），采用显式 `const stateRef = useRef(...)`。
- `WebOperatorSideRail.tsx`：
  - 图标映射更新为：`browser-state/crm-context/hermes-task/page-structure/action-log`，删除 Camera/screenshot。
  - 行为保持：点击 rail → `onFocusedPanelChange(panel)` + `onExpand()`。

## 验证清单
- 代码搜索：
  - `rg "ScreenshotPanel" src/renderer/src`：仅允许命中 `ScreenshotPanel.tsx` 自身。
  - `rg "\"screenshot\"" src/shared src/renderer/src/screens/WebOperator`：不应再出现在 WebOperator 二级导航/面板代码。
  - `rg "hermes-task" src/shared src/renderer/src/screens/WebOperator`：应覆盖 contract/nav/rail/panels。
- 构建验证：
  - `npm run typecheck`
  - `npm run lint`
  - `npm run build`
- 手工 UI 验收：
  - WebOperator 侧栏展开后同时显示 5 个卡片面板、顺序正确、各自独立边框/滚动。
  - rail 点击 Hermes Task 能展开并滚动到 HermesTaskPanel。
  - CRM Context 点击 Refresh snapshot 后 PageStructurePanel 正常刷新。

## 回滚策略
- 如出现布局/滚动问题，可临时只回滚 `WebOperatorPanels.tsx` 的滚动逻辑；面板拆分与导航变更保持不回滚，避免 screenshot 复活。
