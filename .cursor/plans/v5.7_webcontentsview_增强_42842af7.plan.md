# V5.7 WebContentsView 浏览器核心能力增强 — 实施计划

## 现状分析

### 已有能力

- **Main Process**：[`src/main/browser/browser-controller.ts`](src/main/browser/browser-controller.ts) 已实现 `open/back/forward/reload/getState/screenshot/click/type/extractTable`，但全部**只操作 main frame**，通过 `executeJavaScript` 注入脚本。
- **IPC 层**：[`src/main/browser/browser-ipc.ts`](src/main/browser/browser-ipc.ts) 13 个 handler（`browser.*` channel）；[`src/preload/browser-api.ts`](src/preload/browser-api.ts) 暴露 `window.aiosBrowser` 13 方法。
- **契约类型**：[`src/shared/browser/browser-contract.ts`](src/shared/browser/browser-contract.ts) 定义了 `BrowserPageState`（inputs/buttons/links 扁平列表）和 `BrowserAuditRecord` 审计。
- **Renderer**：[`WebOperatorScreen.tsx`](src/renderer/src/screens/WebOperator/WebOperatorScreen.tsx) 左右分栏，右侧含 `BrowserStatePanel`（展示 title/url/inputs/buttons/links）和 `BrowserActionLog`（审计日志）。`ScreenshotPanel` 已有但被注释。
- **WebContentsHost**：[`src/renderer/src/components/shell/WebContentsHost.tsx`](src/renderer/src/components/shell/WebContentsHost.tsx) 纯 bounds/show/hide 管理，符合 PRD 要求不再扩展。

### Phase 1：Shared DTO + 错误码（Task 01）

**目标**：定义 PRD §7 全部数据结构和 §11 错误码，为后续 Main/Preload/Renderer 提供类型基础。

**新增文件**：

- `src/shared/browser/browser-frame-contract.ts` — `BrowserFrameSnapshot`, `BrowserFrameTarget`, `BrowserRect`
- `src/shared/browser/browser-snapshot-contract.ts` — `BrowserElementSnapshot`, `BrowserPageSnapshot`, `BrowserSnapshotOptions`, `BrowserSnapshotError`
- `src/shared/browser/browser-action-contract.ts` — `BrowserActionType`, `BrowserActionResult`(v2), `BrowserActionLog`, `BrowserActionError`, `BrowserActionErrorCode`, `BrowserElementTarget`, `BrowserTypeOptions`, `BrowserScrollOptions`, `BrowserScreenshotOptions`, `BrowserScreenshotResult`(v2), `BrowserState`(v2)

**修改文件**：

- [`src/shared/browser/browser-contract.ts`](src/shared/browser/browser-contract.ts) — 保留现有类型不破坏，新类型放独立文件；可添加 re-export

**关键决策**：
- 新 DTO 与旧 `BrowserPageState` / `BrowserActionResult` 共存，不做破坏性重命名
- 新 `BrowserState`（含 `loading/canGoBack/canGoForward/frameCount`）是 PRD §FR-002 的结构，与旧 `BrowserPageState` 并行

### Phase 4：Main Process 核心模块（Task 04-09）

每个模块对应一个 Task，按依赖顺序实施：

**Task 04 — BrowserFrameInspector**

- 新增 `src/main/browser/browser-frame-inspector.ts`
- 通过 `webContents.mainFrame` 和 `webFrame.framesInSubtree` 遍历 frame tree
- 输出 `BrowserFrameSnapshot[]`，含 `frameId / url / origin / depth / path / sameOriginWithParent`
- 跨域 frame 仅返回元信息（url/origin），不注入脚本

**Task 05 — BrowserDomSnapshot**

- 新增 `src/main/browser/browser-dom-snapshot.ts`
- 在每个可执行 frame 内 `executeJavaScript` 采集可交互元素（a / button / input / textarea / select / [role] / [aria-label] / [contenteditable]）
- 输出 `BrowserPageSnapshot`（含 frames + elements + errors）
- 单 frame 失败记录 `BrowserSnapshotError`，不中断整体

**Task 06 — BrowserElementLocator**

- 新增 `src/main/browser/browser-element-locator.ts`
- 按 PRD §FR-005 优先级（elementId > selector+frame > selector > role+text > ariaLabel > placeholder > text > hrefIncludes）查找元素
- 未指定 frame 时从 main frame 向 child frames 顺序搜索
- 多候选时优先可见可用元素

**Task 07 — BrowserCoordinateResolver**

- 新增 `src/main/browser/browser-coordinate-resolver.ts`
- 计算 `rectInMainFrame`：递归累加 iframe 偏移
- 使用 `webContents.executeJavaScript` 获取 iframe 在父 frame 中的 `getBoundingClientRect`
- 点击前重新计算坐标（不复用旧值）

**Task 08 — BrowserController 增强**

- 修改 [`src/main/browser/browser-controller.ts`](src/main/browser/browser-controller.ts)
- 新增 `clickElement / typeElement / selectOption / scroll / scrollToElement` 方法
- 点击策略：resolve target → resolve frame → collect fresh rect → attempt DOM click → fallback coordinate click → wait idle → update state → write log
- 输入支持 `clear` 模式（默认）和 `append` 模式
- 所有动作写入 `BrowserActionLogStore`

**Task 09 — BrowserActionLogStore**

- 新增 `src/main/browser/browser-action-log-store.ts`
- 内存存储（环形队列，最大 500 条）
- 每条记录 `BrowserActionLog`（id / action / params / result / createdAt）
- 支持 `getAll(limit) / clear() / onLogged(callback)`
- 动作后通过 `mainWindow.webContents.send` 推送 `browser:action-logged` 事件

## 关键约束

- `WebContentsHost` 不增加任何 DOM/iframe/动作逻辑（PRD §6.1）
- Renderer 禁止 Node.js / Electron / ipcRenderer 直接访问，只用 `window.aiosBrowser`
- 新旧 IPC channel 共存（旧 `browser.xxx` 保留，新 `browser:xxx` 并行）
- 所有 IPC 参数/返回值严格 TypeScript DTO，禁止 `any`
- 不修改 Layout / MainPage / 全局样式
- 不引入 Hermes Agent / Gateway / Skill / Profile Runtime 逻辑
