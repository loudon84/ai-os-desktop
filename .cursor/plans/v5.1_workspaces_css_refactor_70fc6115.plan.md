# Workspaces 样式补齐与重构 (v5.1)

## 问题根因

`main.tsx` 只加载 `assets/main.css`（手写 CSS 变量设计系统），**没有 Tailwind CSS 编译产物**。Workspaces 全模块 ~12 个组件文件大量使用 Tailwind utility class（`flex`、`grid`、`h-full`、`bg-gray-*`、`border-gray-800`、`text-xs` 等），运行时全部失效，页面退化为浏览器默认文档流堆叠。

## 修复策略

新增 `Workspaces.css` 独立样式文件，使用 `workspaces-` 前缀的 BEM-like class，复用 `main.css` 的 CSS 变量（`--bg-primary`、`--border`、`--text-muted` 等）。**不改** `main.tsx`、`assets/main.css`、全局 Shell 布局。

## 三栏 Grid 布局目标结构

```
workspaces-screen (flex col, 100% height)
 └─ workspaces-shell (flex col, 100% height)
     ├─ workspaces-status (flex row, 76px, 固定顶部)
     │   ├─ workspaces-status-profiles (flex row, 滚动)
     │   │   └─ workspaces-profile-card * N
     │   └─ workspaces-status-actions (flex row, 按钮组)
     └─ workspaces-body (CSS grid 三栏, flex-1)
         ├─ workspaces-sidebar (220px / 48px)
         │   ├─ workspaces-sidebar-toggle
         │   └─ workspaces-nav-list
         ├─ workspaces-center (minmax(0, 1fr))
         │   └─ workspaces-center-scroll
         │       └─ PageLoader (lazy pages)
         └─ workspaces-right-panel (340px) / workspaces-right-rail (48px)
             ├─ workspaces-right-tabs
             └─ panel body (scroll)
```

## 验收标准

1. 页面打开后顶部为 Status Cards 横向排列
2. Status Cards 下方稳定三栏：左 Sidebar + 中 Main + 右 Panel
3. 左 Sidebar 折叠 48px / 展开 220px，中间自动扩展
4. 右 Panel 折叠为 48px rail / 展开 340px
5. Settings 页面仅在 Center 内滚动，不撑开整个 Shell
6. 无浏览器默认 button 样式
7. DevTools 检查 `.workspaces-body` display 为 grid
8. 不修改 `main.tsx`、`assets/main.css`
9. `npm run typecheck` 通过
