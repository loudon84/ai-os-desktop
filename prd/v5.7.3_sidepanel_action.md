# v5.7.2 panels 首批实现边界

## 1. 操作 CRM 某个按钮：放在 PageStructurePanel 验证

结论：

```text
按钮定位、按钮点击、iframe 内按钮操作验证：
  放在 PageStructurePanel

CRM 业务事件是否上报成功：
  放在 CrmEventPanel

点击动作执行记录：
  看 BrowserActionLog
```

原因：

当前 `PageStructurePanel` 已经具备基于 `frameId + selector + elementId` 执行元素点击的能力，内部调用的是：

```ts
window.aiosBrowser.clickElement({
  elementId: element.elementId,
  selector: element.selector,
  frame: { frameId: element.frameId },
});
```

这说明它已经是页面结构、iframe、元素选择器验证的正确入口。

`ElementListPanel` 也已经有 `Test Click` 和 `Test Type` 操作按钮，适合作为 CRM 按钮验证入口。

`CrmEventPanel` 当前职责是展示 CRM 上报事件、复制上下文、刷新 snapshot、发送 CRM command，它适合作为“CRM JSSDK 事件是否收到”的业务验证面板，不适合做底层 selector / iframe / click 调试。

# 2. panels 内的推荐验证流程

在 v5.7.2 中不要新增第 6 个顶层面板，仍保持 5 个面板：

```text
BrowserStatePanel
CrmEventPanel
HermesTaskPanel
PageStructurePanel
BrowserActionLog
```

按钮验证流程：

```text
1. 在 PageStructurePanel 点击 Refresh Snapshot
2. 在 FrameTreePanel 选择 CRM iframe
3. 在 ElementListPanel 查看该 iframe 内的 buttons / inputs / links
4. 选择目标按钮
5. 点击 Test Click
6. 在 BrowserActionLog 查看 clickElement 执行结果
7. 如果该按钮触发 CRM JSSDK 上报，再到 CrmEventPanel 查看 lastEvent
```

`HermesTaskPanel` 不作为按钮调试入口。它主要用于敏感动作确认队列，当前内部使用 `usePendingActions()`，并提供 Confirm / Reject。

# 3. 获取 CRM 指定 iframe 内 HTML：放在 PageStructurePanel 内实现

结论：

```text
获取指定 iframe HTML：
  在 PageStructurePanel 中新增 FrameHtmlInspector 子组件

不要放到 CrmEventPanel：
  CrmEventPanel 只看 CRM 上报结果，不做 DOM 抽取
```

```text

window.aiosBrowser.snapshot()
window.aiosBrowser.findElement()

window.aiosBrowser.typeElement()
```

但没有 `getFrameHtml()`。现有 preload API 只暴露到 `screenshotV2 / getActionLogs / clearActionLogs`，没有 iframe HTML 获取接口。

当前 frame DTO 已经有：

```ts

parentFrameId
url
origin
name
depth
path
sameOriginWithParent
childCount
```

并且 `BrowserFrameTarget` 支持 `frameId / framePath / urlIncludes / name / title / index`。

Main 侧已经有 `BrowserFrameInspector.findFrame()` 和 `resolveFrameByPath()`，可以按 `frameId` 或 `framePath` 找到指定 `WebFrameMain`。

## 4.3 BrowserV57Core 新增 getFrameHtml

修改：

```text
src/main/browser/browser-v57-core.ts
```

新增脚本：

```ts
const GET_FRAME_HTML_SCRIPT = (
  selector: string | undefined,
  outer: boolean,
  maxLength: number,
) => `
(function() {
  var node = ${selector ? `document.querySelector(${JSON.stringify(selector)})` : "document.documentElement"};
  if (!node) {
    return {
      ok: false,
      code: "ELEMENT_NOT_FOUND",
      message: "Element not found"
    };
  }

  var html = ${selector ? (outer ? "node.outerHTML" : "node.innerHTML") : "document.documentElement.outerHTML"};
  var text = document.body ? document.body.innerText : "";
  var truncated = html.length > ${maxLength};

  return {
    ok: true,
    url: location.href,
    title: document.title,
    html: html.slice(0, ${maxLength}),
    text: text.slice(0, ${Math.min(maxLength, 20000)}),
    truncated: truncated
  };
})()
`;
```

新增方法：

```ts
async getFrameHtml(
  target: BrowserFrameHtmlRequest,
): Promise<BrowserFrameHtmlResult> {
  const startedAt = Date.now();
  const action: BrowserActionType = "getFrameHtml";
  const wc = this.getWc();

  if (!wc || !this.viewManager.isReady()) {
    const result: BrowserFrameHtmlResult = {
      ok: false,
      capturedAt: new Date().toISOString(),
      error: {
        code: "WEB_CONTENTS_NOT_READY",
        message: "External web view is not ready",
      },
    };
    this.log(action, target, wrapResult(action, startedAt, { ok: false }));
    return result;
  }

  const frames = this.frameInspector.listFrames(wc);
  const frameMeta =
    target.frameId
      ? frames.find((frame) => frame.frameId === target.frameId)
      : target.framePath
        ? frames.find((frame) => JSON.stringify(frame.path) === JSON.stringify(target.framePath))
        : target.urlIncludes
          ? frames.find((frame) => frame.url.includes(target.urlIncludes!))
          : target.name
            ? frames.find((frame) => frame.name === target.name)
            : frames[0];

  if (!frameMeta) {
    return {
      ok: false,
      capturedAt: new Date().toISOString(),
      error: {
        code: "FRAME_NOT_FOUND",
        message: "Frame not found",
      },
    };
  }

  const frame = this.frameInspector.resolveFrameByPath(wc, frameMeta.path);
  if (!frame) {
    return {
      ok: false,
      frameId: frameMeta.frameId,
      capturedAt: new Date().toISOString(),
      error: {
        code: "FRAME_NOT_FOUND",
        message: "Frame not found for path",
      },
    };
  }

  try {
    const maxLength = target.maxLength ?? 100000;
    const raw = await frame.executeJavaScript(
      GET_FRAME_HTML_SCRIPT(target.selector, target.outer !== false, maxLength),
    );

    if (!raw.ok) {
      return {
        ok: false,
        frameId: frameMeta.frameId,
        url: frameMeta.url,
        selector: target.selector,
        capturedAt: new Date().toISOString(),
        error: {
          code: raw.code ?? "UNKNOWN_BROWSER_ERROR",
          message: raw.message ?? "Get frame html failed",
        },
      };
    }

    return {
      ok: true,
      frameId: frameMeta.frameId,
      url: raw.url,
      title: raw.title,
      selector: target.selector,
      html: raw.html,
      text: raw.text,
      truncated: raw.truncated,
      capturedAt: new Date().toISOString(),
    };
  } catch (err) {
    return {
      ok: false,
      frameId: frameMeta.frameId,
      url: frameMeta.url,
      selector: target.selector,
      capturedAt: new Date().toISOString(),
      error: {
        code: "FRAME_SCRIPT_BLOCKED",
        message: (err as Error).message,
      },
    };
  }
}
```

说明：现有 `BrowserDomSnapshot` 已经是逐 frame 执行 `executeJavaScript`，并把失败记录为 `FRAME_SCRIPT_BLOCKED`，所以 `getFrameHtml` 也沿用同一错误模型。


组件职责：


```tsx
<PageFrameHtmlInspector
```

```text

  ├─ Header
  ├─ Error
```

# 6. CRM 按钮验证增强

在 `ElementListPanel` 当前 `Test Click` 基础上补一个“指定 selector 验证区”，因为 CRM 页面按钮可能不是 interactive snapshot 里容易识别的按钮。

新增子组件：

```text
src/renderer/src/screens/WebOperator/panels/PageSelectorActionBar.tsx
```

功能：

```text
selectedFrameId: 当前 FrameTreePanel 选中的 frameId
selector: 手工输入 CSS selector
action:
  - Find
  - Test Click
  - Test Type
```

调用方式：

```ts
await window.aiosBrowser.clickElement({
  selector,
  frame: { frameId: selectedFrameId },
});
```

验收：

```text
1. 选择 CRM iframe
2. 输入按钮 selector，例如 #submitBtn 或 button[data-action="save"]
3. 点击 Test Click
4. BrowserActionLog 出现 clickElement 记录
5. CRM 页面按钮被实际触发
6. 若 CRM JSSDK 上报事件，CrmEventPanel 出现 lastEvent
```

# 7. 最终面板职责表

| 面板                 | 做什么                                          | 不做什么                         |
| ------------------ | -------------------------------------------- | ---------------------------- |
| BrowserStatePanel  | 当前页面 URL、title、runtime state                 | 不做按钮调试                       |
| CrmEventPanel      | CRM JSSDK 上报事件、业务上下文、CRM command 验证          | 不做 selector / iframe HTML 调试 |
| HermesTaskPanel    | Hermes 发起的敏感动作确认                             | 不做普通按钮定位                     |
| PageStructurePanel | iframe、DOM snapshot、元素列表、按钮测试、iframe HTML 获取 | 不存储业务事件                      |
| BrowserActionLog   | 所有 browser action 执行日志                       | 不发起操作                        |
