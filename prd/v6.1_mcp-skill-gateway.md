# PRD：v6.1_hermes-desktop-mcp

版本：`v6.1_hermes-desktop-mcp`
项目：`copilot-desktop`
模块：`src\renderer\src\screens\Hermes`
主题：Hermes Desktop MCP 服务管理 + MCP Skills 启用 + 本地 Hermes Agent MCP Skill 支持

---

## 1. 背景与目标

现有 `writer`、`finance`、`coding` 等专用 Hermes Agent 已通过 `MCP Skills Gateway` 对外暴露能力。Gateway 已具备：

```text
- MCP tools/list
- MCP tools/call
- REST Task API
- SSE Task Events
- Artifact 输出
- Skill Registry
- Agent/Profile/Workspace/Skill 映射
```

Copilot Desktop 的 Hermes 模块需要新增 MCP 管理能力，实现类似截图中的：

```text
管理
├── MCP
│   ├── Node Repl
│   └── Writer / Finance / Coding MCP Gateway
├── 技能
│   ├── Deploy Model
│   ├── Image Gen
│   ├── writer_article_generate
│   ├── finance_cashflow_analyze
│   └── coding_prd_generate
└── 市场
```

Hermes Desktop 的 Renderer 不能直接访问 Node.js 或本地文件，只能通过 `window.hermesAPI` 调用 Main Process；现有架构已经通过 `contextBridge.exposeInMainWorld("hermesAPI", hermesAPI)` 暴露安全 API。 新能力必须继续走 `preload -> ipcMain -> main service` 链路，不能让 React 页面直接读写本地配置或直接持有密钥。主进程当前已集中注册 IPC handler，新增能力也应挂到统一 IPC 注册入口。

## 2. 本版本目标

### 2.1 产品目标

```text
1. Hermes 页面新增 MCP 服务管理能力。
2. 支持注册、启用、停用、测试 MCP Server。
3. 支持从 MCP Server 同步 tools/list，并映射为 Desktop Skills。
4. 支持按 profile 启用 MCP Skill。
5. 支持本地 Hermes Agent 调用这些 MCP Skills。
6. 支持 tools/call 后展示 task_id、SSE 进度、artifact 下载。
7. 支持 writer / finance / coding 等远程 MCP Skills Gateway 接入。
8. 支持 Node Repl 等本地 stdio MCP Server 接入。
```

### 2.2 工程目标

```text
1. Renderer 只负责 UI 展示，不直接访问 token、文件系统、网络敏感调用。
2. MCP 连接、工具同步、调用测试统一由 Main Process 处理。
3. MCP 工具启用状态按 profile 隔离。
4. 本地 Hermes Agent 通过 MCP Skill Bridge 使用 MCP Skill。
5. 所有配置变更可审计、可恢复、可诊断。
```

```text
  ├── mcp-server-registry.ts
  ├── mcp-client-service.ts
  ├── mcp-tool-sync-service.ts
  ├── mcp-skill-binding-service.ts
  ├── mcp-runtime-proxy.ts
  ├── hermes-mcp-bridge-installer.ts
  └── mcp-audit-service.ts
  │
  ├── Remote MCP Skills Gateway
  │     ├── writer MCP Gateway
  │     ├── finance MCP Gateway
  │     └── coding MCP Gateway
  │
  ├── Local MCP Server
  │     └── Node Repl / stdio MCP
  │
  ▼
Local Hermes Agent
  └── mcp-skill-bridge skill

```

现有聊天链路中，Main Process 已负责在本地 gateway 未启动时启动 Hermes Gateway，并通过 `sendMessage` 处理流式消息、工具进度、usage、done、error 等事件。 本次 MCP 功能应沿用该模式：所有 MCP 调用事件统一由 Main Process 转发给 Renderer。

## 5. 核心设计结论

### 5.1 MCP Server 与 MCP Skill 分离

```text
MCP Server
  表示一个外部能力提供方。
  示例：
  - Node Repl
  - Writer MCP Skills Gateway
  - Finance MCP Skills Gateway
  - Coding MCP Skills Gateway

MCP Skill
  表示从 MCP tools/list 同步出来的可调用 tool。
  示例：
  - writer_article_generate
  - finance_cashflow_analyze
  - coding_prd_generate
```

### 5.2 Desktop 是 MCP 管理面

Copilot Desktop 负责：

```text
- MCP Server 注册
- MCP Server 启用 / 停用
- tools/list 同步
- Skill 展示
- Skill 启用 / 停用
- profile 绑定
- 调用测试
- 事件展示
- Artifact 展示
- 本地 Hermes Agent Bridge 配置生成
```

### 5.3 Local Hermes Agent 通过 Bridge 调用 MCP

本地 Hermes Agent 不直接管理 MCP Server。Desktop 安装一个系统级 Hermes Skill：

```text
mcp-skill-bridge
```

该 Bridge 负责把本地 Hermes Agent 的 skill 调用转发到 Desktop MCP Runtime Proxy：

```text
Hermes Agent
  ↓
mcp-skill-bridge
  ↓
127.0.0.1:{desktop_mcp_proxy_port}
  ↓
Desktop MCP Runtime Proxy
  ↓
MCP Server tools/call
```

这样可以统一处理：

```text
- Token
- Server enable 状态
- Skill enable 状态
- input_schema 校验
- 调用审计
- SSE 事件
- Artifact
- 本地 stdio MCP Server 生命周期
```

## 6. MCP Server 类型

### 6.1 Remote Streamable HTTP

用于 writer / finance / coding MCP Skills Gateway。

```yaml
id: writer-gateway
name: Writer MCP Gateway
transport: streamable_http
url: https://agent.superic.com/api/v1/hermes/mcp
auth:
  type: bearer
  token_ref: credential:writer_gateway_token
enabled: true
profile_scope:
  - writer
  - default
```

### 6.2 Local stdio

用于 Node Repl 等本地 MCP Server。

```yaml
id: node-repl
name: Node Repl
transport: stdio
command: node
args:
  - ./mcp-servers/node-repl/server.js
env: {}
enabled: true
profile_scope:
  - default
```

### 6.3 SSE Task Events

MCP `tools/call` 本身只返回 task handle。对于长任务，Desktop 通过返回的 `event_url` 订阅 SSE：

```text
tools/call
  ↓
返回 task_id / event_url / artifact_url
  ↓
Desktop 订阅 event_url
  ↓
UI 展示 task.accepted / task.routed / hermes.run.started / artifact.created / task.completed
```

## 7. 数据模型

### 7.1 本地存储建议

新增本地 SQLite：


原因：

```text
1. MCP Server、Tool、Binding、Invocation 需要查询、过滤、状态恢复。

3. 可配合 migration 做版本升级。
```

```sql
  last_error TEXT,

  updated_at TEXT NOT NULL,
```

```sql
CREATE TABLE desktop_mcp_tools (
```

## 9. Hermes Skill Materialization

启用 MCP Skill 后，需要在目标 profile 下生成薄封装 Skill。

路径：

```text
~/.hermes/profiles/{profile}/skills/mcp/{server_id}/{tool_name}/
├── SKILL.md
└── mcp-skill.json
```

### 9.1 SKILL.md 示例

```markdown
---
id: mcp.writer-gateway.writer_article_generate
title: Writer Article Generate
source: mcp
server_id: writer-gateway
tool_name: writer_article_generate
category: writer
enabled: true
---

# Writer Article Generate

Use this skill when the user requests article generation through the Writer MCP Gateway.

The runtime call must be delegated to `mcp-skill-bridge`.

Input schema is defined in `mcp-skill.json`.
```

### 9.2 mcp-skill.json 示例

```json
{
  "server_id": "writer-gateway",
  "tool_name": "writer_article_generate",
  "input_schema": {
    "type": "object",
    "properties": {
      "requirement": { "type": "string" },
      "platform": {
        "type": "string",
        "enum": ["wechat", "zhihu", "technical_blog", "internal_report"]
      },
      "length": { "type": "integer", "default": 2500 }
    },
    "required": ["requirement"]
  },
  "output_schema": {
    "type": "object",
    "properties": {
      "task_id": { "type": "string" },
      "status": { "type": "string" },
      "event_url": { "type": "string" },
      "artifact_url": { "type": "string" }
    }
  }
}
```

### 9.3 禁用策略

禁用时不直接删除目录，改写：

```yaml
enabled: false
```

并同步更新：

```text
desktop_mcp_skill_bindings.enabled = 0
```

## 13. 用户流程

### 13.1 添加 Writer MCP Gateway

```text
1. 用户进入 Hermes > MCP。
2. 点击“添加 MCP 服务”。
3. 选择 transport = streamable_http。
4. 输入：
   name = Writer MCP Gateway
   url = https://agent.superic.com/api/v1/hermes/mcp
   auth = bearer token
5. 点击测试连接。
6. 测试通过后保存。
7. 点击同步 Skills。
8. Desktop 调用 tools/list。
9. Skills Tab 出现 writer_article_generate 等技能。
```

### 13.2 启用 MCP Skill

```text
1. 用户进入 Hermes > 技能。
2. 搜索 writer_article_generate。
3. 打开启用开关。
4. 选择绑定 profile：default / writer。
5. Desktop 写入 desktop_mcp_skill_bindings。
6. Desktop 生成 profile 下的 MCP Skill wrapper。
7. Desktop 更新 mcp_skill_bindings.json。
8. 如果 profile gateway 正在运行，提示需要重启或自动重启。
```

### 13.3 本地 Hermes Agent 调用 MCP Skill

```text
1. 用户在本地 Hermes Chat 中提出写作任务。
2. Hermes Agent 识别 writer_article_generate skill。
3. mcp-skill-bridge 读取 profile 的 mcp_skill_bindings.json。
4. 调用 Desktop MCP Runtime Proxy。
5. Runtime Proxy 调用 Writer MCP Gateway tools/call。
6. 返回 task_id / event_url / artifact_url。
7. Desktop 订阅 SSE。
8. Chat 显示任务状态与 artifact。
```


```text

```

### 15.2 文件结构

```text
mcp-skill-bridge/
├── SKILL.md
├── bridge.py
├── bridge_config.schema.json
└── README.md
```

### 15.3 职责

```text
1. 读取当前 profile 的 mcp_skill_bindings.json。
2. 校验 skill 是否启用。
3. 组装调用请求。
4. 调用 Desktop MCP Runtime Proxy。
5. 返回 task handle 或最终结果。
```

### 15.4 调用模式

```text
async:
  默认模式。立即返回 task_id / event_url / artifact_url。

wait:
  适合短任务。等待 task.completed，最长等待 max_wait_seconds。

artifact:
  适合文档生成。返回 artifact 列表。
```

## 16. 权限与安全

### 16.1 Renderer 限制

```text
1. Renderer 不保存 token。
2. Renderer 不直接请求 MCP Gateway。
3. Renderer 不读取 ~/.hermes。
4. Renderer 不启动 stdio MCP Server。
5. Renderer 只通过 hermesAPI 调用 Main Process。
```

### 16.2 Token 存储

```text
1. token_ref 写入 mcp-registry.db。
2. token 明文不进入 Renderer。
3. Windows 优先使用系统凭据或 Electron safeStorage。
4. 日志中禁止输出 token。
```

### 16.3 stdio MCP Server 限制

```text
1. command 必须经过白名单或用户确认。
2. 禁止拼接 shell 字符串执行。
3. 使用 args 数组启动进程。
4. 限制工作目录。
5. 记录启动、停止、异常日志。
```

### 16.4 Profile 隔离

```text
1. Skill 启用状态按 profile 保存。
2. 一个 profile 启用的 MCP Skill 不自动影响其它 profile。
3. default 可单独绑定 writer / finance / coding 工具。
4. finance 默认不允许使用 coding 的 MCP Skill，除非手工绑定。
```
