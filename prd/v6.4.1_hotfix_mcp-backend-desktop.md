# 子 PRD B：ai-os-desktop

## v6.4.1_hotfix_mcp-desktop

## 1. 目标

修复 Desktop 执行：

```text
mcp:sync-tools
```

时出现：

```text
McpServiceError: fetch failed
```

的问题。

Desktop 只负责：

```text
1. 从 backend 获取 MCP descriptor
2. 读取 Desktop 登录态 access token
3. 配置 Local MCP Proxy
4. 执行 MCP initialize + tools/list
5. 缓存 tools/list 结果
6. UI 显示 MCP Gateway 状态与诊断
```

不处理 backend 用户权限过滤，不实现 backend MCP Gateway。

---

## 2. 当前源码边界

Desktop 当前 Main Process 已集中注册认证、Profile Runtime、Workspace Chat、Hermes Chat、AIOS、Copilot Serve 等 IPC 能力。MCP 新能力必须进入 Main Process，不允许 Renderer 直接访问 backend MCP endpoint。

当前公开分支中 MCP 相关能力主要是旧的：

```text
list-mcp-servers
```

它只用于发现 Hermes profile MCP server 配置，不足以完成新的 MCP Skill Gateway 同步。

验收：

### FR-DE-002：从 backend system/info 获取 MCP descriptor

调用：

```text
GET {backendBaseUrl}/api/v1/system/info
```

读取：

```json
{
  "mcp": {
    "enabled": true,
    "name": "Coding MCP Gateway",
    "transport": "streamable_http",
    "endpoint": "/api/v1/mcp",
    "healthEndpoint": "/api/v1/mcp/health",
    "requiresAuth": true,
    "protocolVersion": "2025-06-18"
  }
}
```

Desktop 合成：

```text
upstreamUrl = backendBaseUrl + mcp.endpoint
```

验收：

```text
1. endpoint 缺失时返回 MCP_DESCRIPTOR_MISSING。
2. backendBaseUrl 缺失时返回 MCP_BACKEND_URL_MISSING。
3. endpoint 不允许由 Renderer 输入。
```

---

### FR-DE-003：Main Process token provider

新增：

```text
src/main/mcp/mcp-token-provider.ts
```

职责：

```text
1. 从 Desktop 当前登录态读取 access token。
2. 判断 token 是否存在。
3. 提供 tokenPresent / tokenPreview 给 diagnostics。
4. 不向 Renderer 返回完整 token。
```

返回：

```ts
interface McpAuthState {
  authenticated: boolean;
  tokenPresent: boolean;
  tokenPreview?: string;
  userId?: string;
  organizationId?: string;
}
```

验收：

```text
1. 登录后 tokenPresent=true。
2. 未登录时 tokenPresent=false。
3. Renderer 无法读取完整 token。
```

### FR-DE-004：Local MCP Proxy 配置刷新

Desktop Main Process 调用：

```text
POST http://127.0.0.1:48742/admin/config
```

请求：

```json
{
  "upstreamUrl": "http://192.168.102.247:4510/api/v1/mcp",
  "transport": "streamable_http",
  "authorizationMode": "desktop_token",
  "protocolVersion": "2025-06-18",
  "clientInfo": {
    "name": "smc-copilot-desktop",
    "version": "v6.4.1_hotfix_mcp-desktop"
  }
}
```

验收：

```text
1. Local Proxy 可接收新的 upstreamUrl。
2. 配置刷新失败时返回 MCP_LOCAL_PROXY_UNREACHABLE。
3. /admin/config 只允许 127.0.0.1。
```

---

### FR-DE-005：Local MCP Proxy 转发 Authorization

Local MCP Proxy 转发 backend MCP 请求时必须带：

```text
Authorization: Bearer <access_token>
Content-Type: application/json
Accept: application/json, text/event-stream
```

验收：

```text
1. Desktop 登录后 tools/list 成功。
2. Desktop 未登录时返回 MCP_UNAUTHORIZED。
3. Proxy 日志不输出完整 token。
```

---

### FR-DE-006：tools/list 前自动 initialize

Local MCP Proxy 收到 `tools/list` 时：

```text
1. 如果未初始化，先向 backend MCP Gateway 发送 initialize。
2. initialize 成功后再发送 tools/list。
3. 保存 session 状态。
```

验收：

```text
1. 直接 curl Local Proxy tools/list 可自动 initialize。
2. initialize 失败时返回 MCP_INITIALIZE_FAILED。
3. tools/list 失败时保留 backend 原始 errorCode。
```

---

### FR-DE-007：mcp:sync-tools 返回结构化结果

返回类型：

```ts
interface McpSyncToolsResult {
  ok: boolean;
  server: {
    id: string;
    name: string;
    transport: "streamable_http";
    upstreamUrl: string;
    localProxyUrl: string;
  };
  status:
    | "connected"
    | "degraded"
    | "unauthorized"
    | "forbidden"
    | "offline"
    | "misconfigured";
  toolCount: number;
  tools: Array<{
    name: string;
    description?: string;
    inputSchema?: unknown;
  }>;
  diagnostics: {
    backendReachable: boolean;
    localProxyReachable: boolean;
    tokenPresent: boolean;
    initialized: boolean;
    lastSyncAt: string;
  };
  error?: {
    code: string;
    message: string;
    upstreamUrl?: string;
    localProxyUrl?: string;
    httpStatus?: number;
    responseBody?: string;
    cause?: string;
  };
}
```

验收：

```text
1. 不再向 Renderer 抛裸 fetch failed。
2. 401 映射 unauthorized。
3. 403 映射 forbidden。
4. 404 映射 misconfigured。
5. backend 不可达映射 offline。
6. tools/list 失败映射 degraded。
```

### FR-DE-010：Renderer MCP Gateway 卡片

UI 显示：

```text
Coding MCP Gateway
Transport: streamable_http
Status: connected / degraded / unauthorized / forbidden / offline / misconfigured
Tools: N
Last Sync: yyyy-MM-dd HH:mm:ss
Error: MCP_UNAUTHORIZED
```

操作：

```text
1. 重新探测
2. 同步 tools
3. 查看诊断
```

验收：

```text
1. 成功时显示 connected + toolCount。
2. 未登录时显示 unauthorized。
3. backend 不可达时显示 offline。
4. endpoint 错误时显示 misconfigured。
5. tools/list 失败时显示 degraded。
6. 不允许继续显示 unknown。
```

## 4. Desktop 错误码

| code                        | UI 状态         | 说明                             |
| --------------------------- | ------------- | ------------------------------ |
| MCP_BACKEND_URL_MISSING     | misconfigured | backendBaseUrl 缺失              |
| MCP_DESCRIPTOR_MISSING      | misconfigured | system/info 未返回 mcp descriptor |
| MCP_LOCAL_PROXY_UNREACHABLE | offline       | Local Proxy 不可达                |
| MCP_BACKEND_UNREACHABLE     | offline       | backend 不可达                    |
| MCP_ENDPOINT_NOT_FOUND      | misconfigured | MCP endpoint 404               |
| MCP_UNAUTHORIZED            | unauthorized  | 未登录或 token 无效                  |
| MCP_FORBIDDEN               | forbidden     | 无权限                            |
| MCP_INITIALIZE_FAILED       | degraded      | initialize 失败                  |
| MCP_TOOLS_LIST_FAILED       | degraded      | tools/list 失败                  |
| MCP_GATEWAY_REQUEST_FAILED  | degraded      | backend gateway 请求失败           |
| MCP_INVALID_RESPONSE        | degraded      | 响应格式错误                         |
