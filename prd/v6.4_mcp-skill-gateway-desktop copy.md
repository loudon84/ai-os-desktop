## 2. Desktop 产品定位

本版本 Desktop 的定位限定为：

```text
1. 用户登录
2. 配置读取
3. Hermes 本地部署管理
4. Hermes 本地运行状态管理
5. 嵌入管理窗口
6. 将 mcp-skill-gateway 注册为 Hermes Agent 的 MCP Server
```

Desktop 不承担以下职责：

```text
1. 不直接执行远程 Skill
2. 不做远程 Tool 调用中心
3. 不做 Task Center 主流程
4. 不直接展示 Skill 执行结果
5. 不绕过 Hermes Agent 调用 mcp-skill-gateway
6. 不把 nodeskclaw accessToken 写入 Hermes config.yaml
```

## 3. 总体目标

在用户登录 Desktop 后，Desktop 从 nodeskclaw 读取运行配置，在本机启动一个受控的本地 MCP Proxy，并把该 Proxy 注册到本地 Hermes Agent 的 `mcp_servers` 配置中。

最终链路：

```text
Hermes Agent
  → http://127.0.0.1:<local_mcp_proxy_port>/mcp
  → Desktop Main Process MCP Proxy
  → nodeskclaw /api/v1/hermes/mcp
  → mcp-skill-gateway
```

Desktop 负责：

```text
1. 登录 nodeskclaw
2. 保存和刷新 Token
3. 读取远程 MCP Skill Gateway 配置
4. 启动本地 MCP Proxy
5. 写入 Hermes profile config.yaml
6. 重启或提示重启 Hermes Gateway
7. 在嵌入窗口中打开 nodeskclaw 管理页面
8. 显示本地注册状态、代理状态、Hermes 状态
```

Hermes Agent 负责：

```text
1. MCP tools/list
2. MCP tools/call
3. 将 Tool 能力纳入自身工具调用链
4. 与 mcp-skill-gateway 进行 JSON-RPC 通讯
```

## 7. 本地 MCP Proxy 设计

### 7.1 模块目录

新增目录：

```text
src/main/mcp-skill-gateway-runtime/

  mcp-skill-gateway-register.ts
  mcp-skill-gateway-health.ts
  mcp-skill-gateway-errors.ts
  mcp-skill-gateway-log.ts
```

### 7.2 Proxy 服务职责

`mcp-skill-gateway-proxy.ts` 负责：

```text
1. 在 127.0.0.1:<port> 启动 HTTP Server
2. 暴露 POST /mcp
3. 校验请求必须来自本机
4. 读取 Desktop 当前登录 Token
5. 将 JSON-RPC 请求转发到 nodeskclaw /api/v1/hermes/mcp
6. 注入 Authorization: Bearer <accessToken>
7. 返回原始 JSON-RPC 响应
8. 不修改 tools/list 返回结构
9. 不修改 tools/call 返回结构
10. 对日志做脱敏
```

### 7.3 Proxy HTTP 接口

#### `GET /health`

返回：

```json
{
  "ok": true,
  "service": "mcp-skill-gateway-proxy",
  "loggedIn": true,
  "backendBaseUrl": "https://example.com",
  "target": "/api/v1/hermes/mcp"
}
```

#### `POST /mcp`

请求体：

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/list",
  "params": {}
}
```

转发到：

```text
{backendBaseUrl}/api/v1/hermes/mcp
```

请求头：

```text
Content-Type: application/json
Authorization: Bearer <desktop_access_token>
```

### 7.4 Proxy 错误返回

当 Desktop 未登录：

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "error": {
    "code": -32010,
    "message": "Desktop login required"
  }
}
```

当 backendUrl 未配置：

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "error": {
    "code": -32011,
    "message": "Backend endpoint not configured"
  }
}
```

当远程请求失败：

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "error": {
    "code": -32012,
    "message": "MCP gateway request failed"
  }
}
```

当 Token 过期：

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "error": {
    "code": -32013,
    "message": "Desktop session expired"
  }
}
```

## 8. Hermes MCP 注册设计

### 8.1 注册函数

新增：

```ts
export async function registerMcpSkillGatewayToHermes(input: {
  profile: string;
  localProxyPort: number;
  enabled: boolean;
}): Promise<{
  changed: boolean;
  configPath: string;
}>;
```

### 8.2 读取目标路径

必须使用现有 profile 路由规则：

```text
default:
  ~/.hermes/config.yaml

named profile:
  ~/.hermes/profiles/<profile>/config.yaml
```

禁止硬编码 profile 路径。

### 8.3 YAML 合并规则

写入时必须保留现有配置，只更新：

```yaml
mcp_servers.mcp_skill_gateway
```

不能覆盖：

```text
model 配置
memory 配置
platform_toolsets
已有其他 mcp_servers
用户自定义配置
```

### 8.4 注册策略

```text
1. 用户登录成功后：
   - 如果 autoStartProxy=true，启动 Proxy
   - 如果 autoRegisterToHermes=true，注册 default profile

2. 用户切换 profile 后：
   - 检查该 profile 是否在 registeredProfiles
   - 如果在，确保 config.yaml 已注册 mcp_skill_gateway

3. 用户点击“注册到当前 Profile”：
   - 写入当前 profile config.yaml
   - 若 autoRestartHermesGateway=true，重启 Gateway
   - 否则提示需要重启 Hermes Gateway

4. 用户点击“取消注册”：
   - 设置 enabled=false 或删除 mcp_skill_gateway 节点
   - 默认采用 enabled=false
```

## 12. 生命周期

### 12.1 App 启动

```text
1. hydrateTokenStore
2. 读取 mcp-skill-gateway-runtime-config
3. 如果已登录且 enabled=true 且 autoStartProxy=true：
   - 启动 MCP Proxy
4. 如果 autoRegisterToHermes=true：
   - 检查 registeredProfiles
   - 修复缺失注册
5. 不自动重启 Hermes Gateway，除非 autoRestartHermesGateway=true
```

### 12.2 用户登录成功

```text
1. 保存 session
2. 刷新 endpoint config
3. 启动 MCP Proxy
4. testRemoteMcp
5. 注册 default profile
6. 刷新 UI 状态
```

### 12.3 用户退出登录

```text
1. 停止 MCP Proxy
2. 保留 Hermes config.yaml 中的 mcp_skill_gateway 配置
3. Proxy 停止后 Hermes 调用 tools/list 会失败
4. UI 显示 Desktop login required
```

不自动删除 Hermes 配置，避免用户配置丢失。

### 12.4 Hermes Gateway 启动前

```text
1. 检查 MCP Proxy 是否运行
2. 检查当前 profile 是否已注册
3. 若未注册，提示用户注册
4. 若已注册但 Proxy 停止，提示启动 Proxy
```

### 12.5 App 退出

```text
1. 停止 MCP Proxy
2. 写入最后状态
3. 不修改 Hermes config.yaml
```

## 13. 安全要求

### 13.1 Token

```text
1. Token 只允许存在 Desktop Auth Token Store
2. Token 不允许写入 config.yaml
3. Token 不允许输出到日志
4. Token 不允许暴露给 Renderer
5. Proxy 请求日志必须脱敏 Authorization
```

### 13.2 Proxy 访问限制

```text
1. Proxy 只监听 127.0.0.1
2. 不允许监听 0.0.0.0
3. 不允许跨机器访问
4. 只接受 POST /mcp 与 GET /health
5. 请求体最大 2MB
6. 超时时间默认 60 秒
```

### 13.3 CORS

```text
1. /mcp 不需要浏览器 CORS
2. 不设置 Access-Control-Allow-Origin: *
3. 如果必须返回 CORS，只允许 127.0.0.1
```

### 13.4 日志

日志路径：

```text
<userData>/logs/mcp-skill-gateway-proxy.log
```

日志内容：

```text
time
level
method
jsonrpc_id
remote_status
duration_ms
error_code
```

禁止记录：

```text
Authorization
accessToken
refreshToken
password
完整 arguments 中的敏感字段
```

敏感字段：

```text
password
token
api_key
secret
authorization
cookie
```

## 17. 验收标准

### 17.1 登录与配置

```text
1. 用户登录 Desktop 后，系统可读取 backendUrl。
2. Token 不出现在 Renderer、config.yaml、日志中。
3. 退出登录后 Proxy 自动停止。
```

### 17.2 Proxy

```text
1. Proxy 只监听 127.0.0.1。
2. GET /health 返回 ok。
3. POST /mcp 能转发 tools/list。
4. 未登录时 POST /mcp 返回 JSON-RPC error -32010。
5. Token 过期时返回 JSON-RPC error -32013。
```

### 17.3 Hermes 注册

```text
1. default profile 可一键注册 mcp_skill_gateway。
2. named profile 可一键注册 mcp_skill_gateway。
3. 重复注册不会破坏 config.yaml。
4. 已有 mcp_servers 不被覆盖。
5. 取消注册后 enabled=false。
```

### 17.4 本地部署管理

```text
1. Hermes 未安装时，MCP Gateway 页面提示先安装 Hermes。
2. Hermes 已安装但 Gateway 未启动时，可提示启动。
3. 注册配置变化后，提示需要重启 Hermes Gateway。
4. 如果 autoRestartHermesGateway=true，注册后自动重启。
```

### 17.5 嵌入窗口

```text
1. 用户可从 Desktop 打开 nodeskclaw 管理窗口。
2. 窗口复用登录态。
3. 退出登录后窗口登录态清理。
```
