# PRD 1：copilot-desktop v6.6.2_genehub-mcp-registration

## 1. 版本信息

版本号：v6.6.2_genehub-mcp-registration
项目：copilot-desktop
模块范围：

```text
src/main/genehub/
src/shared/genehub/
src/preload/genehub-api.ts
src/renderer/src/screens/Hermes/pages/GeneHub/
src/renderer/src/screens/Hermes/pages/McpGateway/
src/main/mcp-skill-gateway-runtime/
```

依赖版本：

```text
v6.6.0：MCP Skill Gateway E2E
v6.6.1：MCP Gateway 运营化与安全治理
v6.5.x：GeneHub Desktop Runtime 基础能力
```

本版本目标：

```text
让 Hermes Agent 通过 nodeskclaw MCP Gateway 发起 GeneHub Skill 注册请求，
由 copilot-desktop 拉取 install job，经用户确认后安装到本地 Hermes profile。
```

---

## 2. 背景

当前 copilot-desktop 已完成：

```text
1. MCP Skill Gateway 本地 Proxy。
2. MCP Gateway 一键诊断、tools preview、invoke test。
3. GeneHub Runtime API。
4. GeneHub device/profile 注册。
5. pending install job 拉取。
6. install job claim。
7. bundle download。
8. bundle validate。
9. Hermes skill 写入。
10. Hermes reload/restart。
11. installed skills sync。
```

当前缺口：

```text
1. MCP Gateway 创建的 GeneHub registration job 与 Desktop 安装 UI 未形成闭环。
2. Desktop 对 MCP 发起的 GeneHub job 缺少“待确认安装”视图。
3. Install job 缺少来源标记：UI 手动创建 / MCP 创建 / 管理员分配。
4. Desktop 安装日志与 MCP job 状态缺少聚合展示。
5. 用户无法在 Desktop 侧明确处理 Agent 发起的 GeneHub skill 安装请求。
```

---

## 3. 产品目标

### 3.1 核心目标

```text
1. Desktop 能识别由 MCP Gateway 创建的 GeneHub install job。
2. Desktop 能展示待确认安装任务。
3. Desktop 默认不自动安装 MCP 发起的 install job。
4. 用户确认后，Desktop 执行下载、校验、安装、重启、同步。
5. 安装状态回传 nodeskclaw。
6. Hermes Agent 可通过 genehub.registration.status 查询安装状态。
7. Desktop 页面能看到完整安装链路与错误原因。
```

### 3.2 用户价值

```text
1. 业务人员可以在 Hermes Agent 对话中请求安装某个 GeneHub skill。
2. IT 管理人员可以在 Desktop 中确认安装动作。
3. Agent 不直接获得本机文件写入权限。
4. nodeskclaw 不直接写用户本地 ~/.hermes。
5. Skill 分发进入可审计、可确认、可回滚的流程。
```

---

## 4. 非目标

本版本不做：

```text
1. 不做 GeneHub skill 发布。
2. 不做 skill 审核后台。
3. 不做 install_git / install_zip / uninstall 等通用写操作审批。
4. 不开放 Hermes Agent 静默安装 skill。
5. 不让 Renderer 直接写本地 Hermes skills。
6. 不让 nodeskclaw 直接写用户本地 ~/.hermes。
7. 不做多级审批流。
8. 不做 skill marketplace 全量重构。
```

---

## 5. 核心链路

```text
Hermes Agent
  ↓ MCP tools/call
nodeskclaw genehub.skill.register_to_hermes
  ↓ create install job
nodeskclaw pending install job
  ↓ Desktop scheduler / manual refresh
copilot-desktop pending jobs panel
  ↓ user confirm
Desktop install worker
  ↓ claim job
download bundle
  ↓ validate
write SKILL.md
  ↓ reload / restart Hermes
sync installed skills
  ↓ update job status
Hermes Agent 查询 genehub.registration.status
```

## 6.2 Pending Job 自动发现

当前 GeneHub Runtime 已有 pending job 查询能力，需要在 v6.6.2 明确行为：

```text
1. Desktop 登录成功后启动 GeneHub scheduler。
2. scheduler 每 pendingJobsIntervalMs 拉取 pending jobs。
3. source=mcp_agent_request 的 job 写入本地 pending cache。
4. 若 autoInstallAssignedJobs=false，只通知 UI，不自动安装。
5. 若 autoInstallAssignedJobs=true，可自动执行安装，但默认关闭。
```

本地缓存建议：

```text
~/.hermes/desktop/genehub/pending-jobs.json
```

缓存内容：

```ts
interface GeneHubPendingJobCache {
  updatedAt: string;
  jobs: InstallJob[];
}
```

## 6.5 Bundle 校验增强

现有校验流程继续保留，v6.6.2 需要明确校验项：

```text
1. geneSlug 不为空。
2. skillName 符合 GENEHUB_SKILL_NAME_PATTERN。
3. files 至少包含 SKILL.md。
4. relativePath 不允许绝对路径。
5. relativePath 不允许 ../ 路径穿越。
6. scripts 仅允许写入受控 scripts 目录。
7. verifySignature=true 时必须校验 signature。
8. bundleHash / manifestHash 不匹配时阻断安装。
9. compatibility.profiles 不匹配时阻断或警告。
```

失败状态：

```text
status=failed
errorCode=GENEHUB_BUNDLE_INVALID | GENEHUB_SIGNATURE_INVALID | GENEHUB_PATH_NOT_ALLOWED
```

## 6.10 用户确认策略

配置项沿用：

```ts
autoInstallAssignedJobs: boolean
verifySignature: boolean
```

默认：

```text
autoInstallAssignedJobs=false
verifySignature=true
```

行为：

```text
1. false：所有 MCP 发起的安装请求进入待确认。
2. true：允许自动安装，但只适用于 permission.canInstall=true 且 bundle 校验通过的 job。
3. failed job 不自动重试。
4. 重试必须用户手动触发。
```

## 9. 安全要求

```text
1. Renderer 不直接访问本地文件系统。
2. Renderer 不传任意本地路径。
3. Bundle 下载、校验、写入全部在 Main Process。
4. 默认不自动安装 MCP 发起的 job。
5. verifySignature 默认开启。
6. 不允许路径穿越。
7. 不允许覆盖非 GeneHub 管理的 scripts，除非 provenance 匹配。
8. 安装失败必须回传 nodeskclaw。
9. accessToken 只由 Main 注入。
10. install log 不记录 token、password、Authorization。
```
