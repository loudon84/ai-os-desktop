# PRD 1：copilot-desktop v6.6.1_mcp-gateway-operations

## 1. 版本信息

版本号：v6.6.1_mcp-gateway-operations
模块范围：copilot-desktop
主题：MCP Gateway 运营化与安全治理
依赖版本：v6.6 MCP Skill Gateway E2E 已完成并测试成功
目标对象：Copilot Desktop、本地 Hermes Agent、nodeskclaw MCP Skill Gateway

---

## 2. 背景

v6.6 已经完成 Copilot Desktop 与 nodeskclaw MCP Skill Gateway 的基础打通：

1. Copilot Desktop 可以登录 nodeskclaw。
2. Desktop 本地 MCP Proxy 可以启动。
3. Proxy 可以访问 nodeskclaw `/api/v1/hermes/mcp`。
4. Hermes profile 可以注册 `mcp_skill_gateway`。
5. Hermes Agent 可以通过 MCP Gateway 调用 nodeskclaw 侧只读 tools。

当前问题是：链路虽然可用，但缺少运营化能力。

主要缺口：

1. 出错时无法快速判断是登录、Proxy、远端 MCP、Profile 注册还是 Hermes Gateway 生效问题。
2. Desktop 页面缺少完整 tools/list 展示。
3. 缺少 invoke test，无法在 UI 中直接验证某个 MCP tool 是否能调用。
4. 缺少 profile 注册一致性检测。
5. Hermes Gateway 是否需要重启提示不够明确。
6. 缺少调用日志、诊断报告、错误码映射。
7. 对写操作工具缺少安全分级和用户确认机制。

---

## 3. 产品目标

v6.6.1 的目标是把 MCP Gateway 从“功能能通”升级为“可诊断、可审计、可治理、可长期运行”。

### 3.1 核心目标

1. 提供 MCP Gateway 一键诊断。
2. 展示 nodeskclaw 远端 MCP tools 列表。
3. 支持只读 tool invoke test。
4. 检查本地 Proxy、远端 MCP、Hermes profile 注册一致性。
5. 明确 Hermes Gateway 是否需要重启。
6. 展示 Proxy 日志、远端错误、工具调用结果。
7. 为后续写操作工具与 GeneHub Skill Registration 提供安全治理基础。

---

## 4. 非目标

本版本不做以下内容：

1. 不做 GeneHub Marketplace UI。
2. 不做 GeneHub skill 下载落盘。
3. 不做 skill 发布、审核、版本管理。
4. 不开放静默写操作。
5. 不让 Renderer 直接请求 nodeskclaw MCP。
6. 不让 Hermes Agent 直接访问远端 nodeskclaw MCP。
7. 不把 accessToken 写入 Hermes `config.yaml`。
8. 不改变现有 Hermes Agent skill 目录结构。

## 7. 功能需求

## 7.1 MCP Gateway 一键诊断

新增按钮：

```text
一键检查 MCP Gateway
```

执行流程：

```text
1. 检查 desktopAuth.getState()
2. 检查 backendBaseUrl
3. 检查 localProxyStatus
4. 若 Proxy 未启动，尝试 startProxy()
5. 执行 testProxy()
6. 执行 testRemoteMcp()
7. 执行 tools/list
8. 检查 default profile registration
9. 检查 Hermes Gateway 是否需要重启
10. 输出诊断报告
```

诊断结果结构：

```ts
interface McpGatewayDiagnosticsResult {
  ok: boolean;
  checkedAt: string;
  auth: DiagnosticCheck;
  backend: DiagnosticCheck;
  localProxy: DiagnosticCheck;
  remoteMcp: DiagnosticCheck;
  toolsList: DiagnosticCheck;
  defaultProfileRegistration: DiagnosticCheck;
  hermesGateway: DiagnosticCheck;
  toolCount: number;
  hermesRestartRequired: boolean;
  errors: DiagnosticError[];
}
```

## 7.2 Tools Preview

新增区域：

```text
MCP Tools Preview
```

展示字段：

```ts
interface McpGatewayToolPreview {
  name: string;
  description: string;
  category: "hermes" | "genehub" | "system" | "unknown";
  permission: "read" | "write" | "admin";
  riskLevel: "low" | "medium" | "high";
  inputSchema: Record<string, unknown>;
  enabled: boolean;
  source: "nodeskclaw";
  lastSyncedAt: string;
}
```

展示规则：

1. 默认按 category 分组。
2. read 工具可直接 invoke test。
3. write / admin 工具只展示，不允许直接调用。
4. inputSchema 可折叠查看。
5. tools/list 失败时展示结构化错误。

## 7.3 Invoke Test

新增区域：

```text
Invoke Test
```

默认 tool：

```text
hermes.skills.list
```

默认参数：

```json
{
  "instance_ref": "zhang-zhen"
}
```

请求结构：

```ts
interface McpGatewayInvokeTestInput {
  toolName: string;
  arguments: Record<string, unknown>;
}
```

返回结构：

```ts
interface McpGatewayInvokeTestResult {
  ok: boolean;
  toolName: string;
  permission: "read" | "write" | "admin";
  durationMs: number;
  result?: unknown;
  errorCode?: string;
  errorMessage?: string;
}
```

安全规则：

1. 本版本只允许 invoke read 工具。
2. write/admin 工具点击测试时提示“v6.9 审批流后开放”。
3. 输入 JSON 必须校验。
4. 调用超时时间默认 60 秒。
5. 结果区域最多显示 256KB。

注册逻辑：

3. 不写入 accessToken。
4. 如果 Hermes Gateway 正在运行，返回 `hermesRestartRequired=true`。
5. UI 显示“一键重启 Hermes Gateway”入口。

## 7.5 Proxy Logs 结构化

当前 logs 可读性不足，需要增加结构化展示。

日志字段：

```ts
interface McpGatewayProxyLogEntry {
  time: string;
  level: "info" | "warn" | "error";
  method?: string;
  jsonrpcId?: string | number | null;
  remoteStatus?: number;
  durationMs?: number;
  errorCode?: string | number;
  message?: string;
}
```

UI 展示：

1. 支持按 level 过滤。
2. 支持按 method 过滤。
3. 支持搜索 errorCode。
4. 支持复制诊断报告。
5. 不展示 Authorization header。

## 7.6 Hermes Gateway 重启提示

当以下情况发生时，必须提示重启：

1. 首次注册 MCP Gateway 到 profile。
2. 修改 Proxy port。
3. 取消注册后需要使配置生效。
4. config.yaml 与运行中 Gateway 状态不一致。

提示文案：

```text
MCP Gateway 已写入 Hermes profile 配置。当前 Hermes Gateway 需要重启后才能加载 mcp_skill_gateway。
```

按钮：

```text
稍后重启
立即重启 Hermes Gateway
```

## 9. 安全要求

1. Renderer 不直接访问 nodeskclaw MCP。
2. Renderer 不接触 accessToken。
3. accessToken 不写入 Hermes `config.yaml`。
4. 本地 Proxy 只监听 `127.0.0.1`。
5. 本地 Proxy 拒绝非 localhost 请求。
6. read tool 可 invoke test。
7. write/admin tool 不允许直接 invoke。
8. 日志不得输出 Authorization header。
9. JSON-RPC body 最大 2MB。
10. invoke test 结果最大显示 256KB。
