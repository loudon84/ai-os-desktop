# ai-os-desktop 源码重构拆解方案

## 0. 本次确定的产品口径

```txt id="l18j2f"
主应用入口：
  aios-home -> http://127.0.0.1:3000

用户配置来源：
  Portal backend 自带 desktop bootstrap API

登录策略：
  首次登录直接覆盖本地配置
  后续配置变更必须 diff 确认后再覆盖

Settings 入口：
  TopBar Settings Drawer

Hermes 页面路由：
  彻底删除 Renderer 页面路由
  保留 Main / Preload / IPC / Runtime 能力

Token 存储：
  Electron Main Process 侧 safeStorage 加密存储
  Renderer 不接触 accessToken / refreshToken
```

# 1. 当前源码状态判断

当前仓库已经不是原始 Hermes Desktop，而是一个混合形态的桌面控制面：

```txt id="n10sf7"
Electron Shell
├─ Hermes Runtime 运维页面
├─ Portal Home / Workspace
├─ Web Operator
├─ Multi Profile Runtime
├─ Enterprise Installer
└─ Shell View / WebContentsView 管理
```

原 Hermes Desktop 的基本边界是：Renderer 无 Node.js 权限，Preload 是唯一安全桥，Main Process 统一注册 IPC，Python Gateway 作为外部进程运行。这个边界应继续保留，不要让 Renderer 直接读写文件、token 或进程。

当前问题集中在 Renderer 层：

1. `Layout.tsx` 的主导航还混有大量 Hermes 运维页面：`chat`、`sessions`、`agents`、`models`、`providers`、`skills`、`soul`、`memory`、`tools`、`gateway`、`runtime-setup`、`settings`。
2. `WorkspaceOutlet.tsx` 直接 import 并渲染 Hermes 页面，导致 Portal 主应用和 Hermes 运维功能耦合。
3. `desktop-shell.ts` 的 `View` 类型仍包含旧 Hermes 页面路由和 `profile-workspace:*`。
4. `MainTopBar.tsx` 的 Settings 按钮仍然执行 `onNavigate("settings")`，不是打开 Drawer。
5. `shell-view-ipc.ts` 当前 Portal Home 默认加载 `http://127.0.0.1:${frontendPort}/zh`，而需求要求主应用打开根路径 `http://127.0.0.1:3000`。 
6. Preload 已有庞大的 `hermesAPI`、`aiosRuntime`、`shellView`、`profileRuntime` 等接口，说明底层能力可以保留并迁移到 Drawer 使用，不需要重写。

function isValidRestoredView(
  view: string | undefined,
  externalTabIds: string[],
): view is View {
  if (!view) return false;

  const known: View[] = [
    "aios-home",
    "aios-workspace",
    "web-operator",
    "office",
  ];

  if (known.includes(view as View)) return true;

  if (view.startsWith("external-browser:") && externalTabIds.includes(view)) {

  }

  return false;
}

## 11.3 Token 存储

```txt id="fcfim5"
%APPDATA%/ai-copilot-desktop/auth/session.enc
```

实现：

```ts id="tfqkxi"
import { app, safeStorage } from "electron";
import { existsSync, mkdirSync, readFileSync, writeFileSync, rmSync } from "fs";
import { dirname, join } from "path";
import type { InternalAuthSession } from "../../shared/auth/auth-contract";

const SESSION_FILE = join(app.getPath("userData"), "auth", "session.enc");

export function saveEncryptedSession(session: InternalAuthSession): void {
  if (!safeStorage.isEncryptionAvailable()) {
    throw new Error("safeStorage encryption is not available");
  }

  mkdirSync(dirname(SESSION_FILE), { recursive: true });
  const encrypted = safeStorage.encryptString(JSON.stringify(session));
  writeFileSync(SESSION_FILE, encrypted);
}

export function readEncryptedSession(): InternalAuthSession | null {
  if (!existsSync(SESSION_FILE)) return null;
  if (!safeStorage.isEncryptionAvailable()) return null;

  const encrypted = readFileSync(SESSION_FILE);
  const plain = safeStorage.decryptString(encrypted);
  return JSON.parse(plain) as InternalAuthSession;
}

export function clearEncryptedSession(): void {
  if (existsSync(SESSION_FILE)) {
    rmSync(SESSION_FILE, { force: true });
  }
}
```

## 12.5 首次覆盖 / 后续 Diff

```ts id="c85cl9"
export async function bootstrapUserConfig(): Promise<BootstrapResult> {
  const state = readBootstrapState();
  const remote = await fetchRemoteBootstrapConfig();

  if (!state.initialized) {
    await applyUserConfig(remote, { mode: "overwrite" });
    writeBootstrapState({
      initialized: true,
      lastConfigHash: remote.configHash,
      lastConfigVersion: remote.configVersion,
      lastAppliedAt: new Date().toISOString(),
    });

    return {
      ok: true,
      firstLogin: true,
      applied: true,
      config: remote,
    };
  }

  const local = readLocalBootstrapConfig();
  const diff = diffBootstrapConfig(local, remote);

  if (diff.length > 0) {
    return {
      ok: true,
      firstLogin: false,
      applied: false,
      config: remote,
      diff,
    };
  }

  return {
    ok: true,
    firstLogin: false,
    applied: false,
    config: remote,
    diff: [],
  };
}
```

# 24. 结论

本次不是重写 `ai-os-desktop`，而是做 **Renderer 路由收敛 + Hermes 运维 Drawer 化 + Auth Bootstrap 补齐**。

最终形态：

```txt id="wac35d"
aios-home
  = 主应用
  = http://127.0.0.1:3000
  = WebContentsView

aios-workspace
  = 本地 Portal 控制台

web-operator
  = Agent Web 自动化浏览器

office
  = 内置办公模块

external-browser:*
  = 临时网页 Tab

Hermes Runtime
  = 不再是页面路由
  = TopBar Settings Drawer 管理

Auth / User Config
  = Portal backend 提供
  = Main 侧加密 token
  = 首次覆盖
  = 后续 diff 确认
```

这套拆解完成后，`ai-copilot-desktop` 的产品边界会变清晰：**Portal 是主应用，Hermes Agent 是本地 Runtime 能力，Electron Shell 是安全桌面容器与运维控制面。**
