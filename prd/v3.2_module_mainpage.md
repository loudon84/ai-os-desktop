## 结论

阶段五不再继续围绕“页面路由”扩展，而是把现有 MainPage 改成 **Workspace Module Host**。

目标结构：

```txt
MainPage
  ├─ MainTopBar
  │   ├─ Workspace Tabs
  │   ├─ External Browser Tab Controls
  │   └─ Settings Drawer Trigger
  ├─ Sidebar
  │   └─ 二级功能导航，不再主导一级布局
  ├─ WorkspaceOutlet
  │   ├─ aios-home              WebContentsView
  │   ├─ aios-workspace         React Local Console
  │   ├─ web-operator           React Shell + WebContentsView
  │   ├─ office                 React/Local Module
  │   └─ external-browser:*     Dynamic WebContentsView
  └─ SettingsDrawer
      ├─ Auth / User Config
      ├─ Hermes Runtime
      ├─ Runtime Setup
      ├─ Gateway Status
      └─ Profiles / Skills / Providers
```

现有代码已经具备 Phase 5 的基础：`MainPage` 已经成为一级容器，并把 `sidebar / outlet / statusBar / modalLayer / drawerLayer` 作为组合入口传入。
`Layout.tsx` 当前负责组合 MainPage、DesktopSidebar、WorkspaceOutlet、StatusBar、ModalLayer、DrawerLayer，并已经持久化 `sidebarMode / tabOrder / externalTabs / lastActiveView`。

# Phase 5：Workspace Module 化 + Auth/User Config + Hermes Runtime Drawer

## 1. 现有代码状态判断

### 1.1 MainPage 已经具备 Mattermost 式一级窗口骨架

Mattermost 的 MainPage 思路是：主窗口只负责 TopBar、TabBar、ServerDropdown、Downloads、错误占位等 Desktop Chrome 状态，真正的 WebView 生命周期由主进程管理。Mattermost MainPage 内部维护 activeServer、activeTab、tabs、downloads、developerMode、load status，并通过 desktop API 响应 server/tab/download 事件。
它的 TabBar 支持选择、关闭、新增、DnD 排序、错误状态映射。

当前 ai-os-desktop 已经按这个方向演进：

```txt
Layout.tsx
  ├─ useDesktopNavigation()
  ├─ useShellViewMetadata()
  ├─ useExternalBrowserTabs()
  ├─ useKeepAliveRegistry()
  └─ MainPage(...)
```

`MainTopBar` 目前已经包含：

```txt
Sidebar toggle
Profile Switch
Runtime Indicator
MainViewTabs
New External Tab
Back / Forward / Stop / Reload / Close
Portal Home / Web Operator / Settings shortcut
WindowControls
```

这些能力已经接近 Mattermost 顶栏模型。

### 1.2 Tab 模型已经可复用

当前 `MainViewTabs` 已经使用 `@dnd-kit` 实现可拖拽排序，并把 system tabs、profile tabs、external tabs 合并渲染。

当前固定 tab 是：

```ts

aios-workspace
web-operator
```

profile workspace 会从 `profileEntries` 动态生成：

```ts

```

external browser 会从 `externalTabs` 动态生成。

### 1.3 WebContentsView 生命周期已经在 Main 侧

当前 `ShellViewManager` 已经支持：

```txt

activateView
deactivateView
destroyView
loadUrl
focusView
reloadView
stopLoadingView
goBackView
goForwardView
recoverView
getAllViewSnapshots
```

并且 `createView` 已经具备 keep-alive 语义：如果 view 已存在，不重新创建，显式 recover 才 destroy + recreate。

`ManagedWebContentsView` 已经封装了 WebContentsView 的 title、favicon、loading、canGoBack、canGoForward、load failed、crashed 等状态。

`WebContentsHost` 通过 DOM bounds 同步到 `window.shellView.setBounds()`，避免 React 页面直接管理 Electron View。

### 1.4 Phase 4.5 KeepAlive 已经完成，但还有边界需要收口

当前 `KeepAliveView` 是通过 `display: flex / none` 保留 React 子树，不卸载组件。

`WorkspaceOutlet` 已经对多数 React 页面做了 KeepAlive，但 `profile-workspace:*` 和 `external-browser:*` 仍是条件渲染。

这在 WebContentsView 层面不会丢内容，因为 native view 没被 destroy；但 React Host 会卸载，切换时会重新计算 bounds。这个行为可接受，但 Phase 5 要统一成模块生命周期模型。

  if (workspaceId === "aios-home") {

    );

    );

# 11. 高风险点

| 风险                          | 说明                                            | 处理                                        |
| --------------------------- | --------------------------------------------- | ----------------------------------------- |
| `View` 类型大改                 | 会影响 Layout、Sidebar、WorkspaceOutlet、MainTopBar | Phase 5 不直接删除旧 view，只隐藏入口                 |
| Settings 从路由变 Drawer        | 可能影响现有 Settings 组件依赖 profile                  | 先复用组件，Drawer 内包一层适配器                      |
| profile-workspace 移出 TopBar | 当前 `MainViewTabs` 会自动生成 profile tab           | registry 中先关闭 profile tab 生成逻辑            |
| WebContentsView bounds      | Drawer 打开时不能挤压主内容导致 WebView 频繁 resize         | Drawer 使用 overlay，不改 MainPage__content 尺寸 |
| token 注入                    | 误注入到 external-browser 有安全风险                   | Main 侧按 partition + origin 白名单拦截          |
| 首次覆盖配置                      | 可能覆盖用户本地修改                                    | 仅 firstSyncCompleted=false 时执行，之后全部 diff  |
| external tab 恢复             | 当前 state 已支持 externalTabs restore             | V2 迁移必须兼容 version=1                       |
