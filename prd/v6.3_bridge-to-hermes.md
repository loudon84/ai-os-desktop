## 2. 背景

当前 `HermesTaskStartDialog.tsx` 内部直接加载 skills，并在弹框中维护技能下拉选项。该逻辑后续会被 HostBridge、WebOperator 任务、普通 Hermes Panel 多处复用，因此需要抽成独立组件。

同时，HostBridge 外部传入的 `skillName` 是业务系统指定的执行技能。该 skill 必须存在于当前 hermes-agent 的已安装 skills 中。不存在时，不允许用户选择其它 skill 替代执行，避免业务请求被错误 skill 处理。

新增能力：

1. `HermesPanelSkill`：当前 hermes-agent 可用 skills 下拉组件。
2. `HermesPanelSession`：当前 hermes-agent 最近 7 天 sessions 下拉组件。
3. `HermesTaskStartDialog` 集成上述两个组件。
4. 外部传入 `skillName` 时，必须由 `HermesPanelSkill` 校验。
5. `skillName` 不存在时，不允许提交任务。
6. 用户可选择新建 session 或继续最近 7 天内已有 session。
7. Dialog 确认后，把 `skill`、`sessionId`、`userPrompt` 一起提交给 `HermesTaskPanel`。

## 3. 产品目标

### 3.1 HermesPanelSkill

用于统一显示当前 hermes-agent 已安装 skills。

能力：

```text
1. 从 window.hermesAPI.listInstalledSkills(profile) 获取 skills。
2. 支持 default profile，默认 profile = "default"。
3. 支持外部传入 requiredSkillName。
4. requiredSkillName 存在时自动选中该 skill。
5. requiredSkillName 不存在时进入 invalid 状态。
6. invalid 状态下禁止选择其它 skill 替代。
7. 支持 refresh。
8. 支持 loading / error / empty 状态。
```

### 3.2 HermesPanelSession

用于统一显示当前 hermes-agent 最近 7 天内创建的 sessions。

能力：

```text
1. 先调用 window.hermesAPI.syncSessionCache() 刷新本地 session cache。
2. 再调用 window.hermesAPI.listCachedSessions(limit, offset) 获取 sessions。
3. 只保留 startedAt 在最近 7 天内的 sessions。
4. 默认提供“新建会话”选项。
5. 支持外部传入 selectedSessionId。
6. 支持 loading / error / empty 状态。
7. 选中已有 session 后，HermesTaskPanel 应使用该 sessionId 继续会话。
```

### 3.3 HermesTaskStartDialog

改造后 Dialog 不再直接读取 skills 和 sessions。

职责调整：

```text
1. 只负责弹框布局。
2. 只负责组合 HermesPanelSkill / HermesPanelSession。
3. 只负责收集 userPrompt。
4. 只负责提交用户确认数据。
5. 不再直接调用 window.hermesAPI.listInstalledSkills。
6. 不再自行校验外部 skillName。
```

### 4.1.1 Props

```ts
export interface HermesPanelSkillOption {
  label: string
  value: string
  category?: string
  description?: string
  path?: string
}

export interface HermesPanelSkillValidation {
  status: "idle" | "loading" | "valid" | "invalid" | "error"
  requiredSkillName?: string
  message?: string
  installedSkills: HermesPanelSkillOption[]
}

export interface HermesPanelSkillProps {
  profile?: string
  value: string
  requiredSkillName?: string
  allowDefault?: boolean
  disabled?: boolean
  className?: string

  onChange: (skillName: string) => void
  onValidationChange?: (validation: HermesPanelSkillValidation) => void
}
```

### 4.1.2 行为规则

```text
1. profile 默认值为 "default"。
2. allowDefault 默认 false。
3. requiredSkillName 存在时：
   - 只允许选择 requiredSkillName。
   - 如果 installedSkills 中存在 requiredSkillName，则选中并 status=valid。
   - 如果不存在，则 status=invalid。
4. requiredSkillName 不存在时：
   - 允许用户从全部 installedSkills 中选择。
   - allowDefault=true 时增加 default 选项。
5. installedSkills 为空时 status=invalid。
6. listInstalledSkills 调用失败时 status=error。
```

### 4.1.3 skill 匹配规则

必须兼容以下形式：

```text
skill.name
category/name
name
```

匹配函数：

```ts
function matchSkillName(
  requiredSkillName: string,
  skill: HermesPanelSkillOption,
): boolean {
  const target = requiredSkillName.trim()
  const name = skill.value.trim()
  const categoryName = skill.category
    ? `${skill.category}/${skill.value}`
    : skill.value

  return target === name || target === categoryName
}
```

### 4.1.4 渲染规则

valid：

```text
显示 skills 下拉
选中 requiredSkillName
允许提交
```

invalid：

```text
显示错误提示：
Skill 未安装：{requiredSkillName}
请先在 Local Hermes / Skills 中安装后重新触发业务页面按钮。
禁用下拉
通知父组件不可提交
```

error：

```text
显示错误提示：
无法读取当前 Hermes skills，请检查 hermes-agent runtime。
禁用提交
```

empty：

```text
显示错误提示：
当前 Hermes profile 未安装任何 skill。
禁用提交
```

### 4.1.5 示例实现结构

```tsx
export function HermesPanelSkill({
  profile = "default",
  value,
  requiredSkillName,
  allowDefault = false,
  disabled = false,
  className,
  onChange,
  onValidationChange,
}: HermesPanelSkillProps): React.JSX.Element {
  const [loading, setLoading] = useState(false)
  const [error, setError] = useState<string | null>(null)
  const [options, setOptions] = useState<HermesPanelSkillOption[]>([])

  useEffect(() => {
    let cancelled = false

    async function load(): Promise<void> {
      setLoading(true)
      setError(null)

      try {
        const installed = await window.hermesAPI.listInstalledSkills(profile)

        if (cancelled) return

        const nextOptions = installed.map((item) => ({
          label: item.category ? `${item.category}/${item.name}` : item.name,
          value: item.name,
          category: item.category,
          description: item.description,
          path: item.path,
        }))

        setOptions(nextOptions)

        const matched = requiredSkillName
          ? nextOptions.find((item) => matchSkillName(requiredSkillName, item))
          : null

        if (requiredSkillName) {
          if (matched) {
            onChange(matched.value)
            onValidationChange?.({
              status: "valid",
              requiredSkillName,
              installedSkills: nextOptions,
            })
          } else {
            onChange("")
            onValidationChange?.({
              status: "invalid",
              requiredSkillName,
              message: `Skill 未安装：${requiredSkillName}`,
              installedSkills: nextOptions,
            })
          }
        } else {
          onValidationChange?.({
            status: nextOptions.length > 0 ? "valid" : "invalid",
            installedSkills: nextOptions,
            message: nextOptions.length > 0 ? undefined : "当前未安装任何 skill",
          })
        }
      } catch (e) {
        if (cancelled) return

        const message = e instanceof Error ? e.message : String(e)
        setError(message)
        onValidationChange?.({
          status: "error",
          requiredSkillName,
          message,
          installedSkills: [],
        })
      } finally {
        if (!cancelled) setLoading(false)
      }
    }

    void load()

    return () => {
      cancelled = true
    }
  }, [profile, requiredSkillName, onChange, onValidationChange])

  const selectableOptions = allowDefault
    ? [{ label: "default", value: "" }, ...options]
    : options

  const invalidRequiredSkill =
    requiredSkillName &&
    !options.some((item) => matchSkillName(requiredSkillName, item))

  return (
    <div className={className}>
      <label>
        <span>技能</span>
        <select
          value={value}
          disabled={disabled || loading || !!invalidRequiredSkill || !!error}
          onChange={(event) => onChange(event.target.value)}
        >
          {selectableOptions.map((item) => (
            <option key={item.category ? `${item.category}/${item.value}` : item.value} value={item.value}>
              {item.label}
            </option>
          ))}
        </select>
      </label>

      {loading ? <p>正在读取 skills…</p> : null}
      {error ? <p>{error}</p> : null}
      {invalidRequiredSkill ? (
        <p>Skill 未安装：{requiredSkillName}</p>
      ) : null}
    </div>
  )
}
```

### 4.2.2 行为规则

```text
1. profile 默认 "default"。
2. days 默认 7。
3. limit 默认 100。
4. 组件挂载后先 syncSessionCache。
5. sync 后调用 listCachedSessions(limit, 0)。
6. startedAt >= nowSeconds - days * 86400 的 session 才展示。
7. 默认第一项为“新建会话”。
8. 选择“新建会话”时返回 null。
9. 选择已有 session 时返回 session.id。
```

### 4.2.3 时间处理规则

当前 session cache 的 `startedAt` 按 Unix seconds 处理。

```ts
const nowSeconds = Math.floor(Date.now() / 1000)
const since = nowSeconds - days * 24 * 60 * 60
const recent = sessions.filter((item) => item.startedAt >= since)
```

### 4.2.4 示例实现结构

```tsx
export function HermesPanelSession({
  value,
  days = 7,
  limit = 100,
  disabled = false,
  className,
  onChange,
  onLoaded,
}: HermesPanelSessionProps): React.JSX.Element {
  const [loading, setLoading] = useState(false)
  const [error, setError] = useState<string | null>(null)
  const [sessions, setSessions] = useState<HermesPanelSessionOption[]>([])

  useEffect(() => {
    let cancelled = false

    async function load(): Promise<void> {
      setLoading(true)
      setError(null)

      try {
        await window.hermesAPI.syncSessionCache()
        const cached = await window.hermesAPI.listCachedSessions(limit, 0)

        if (cancelled) return

        const nowSeconds = Math.floor(Date.now() / 1000)
        const since = nowSeconds - days * 24 * 60 * 60

        const recent = cached
          .filter((item) => item.startedAt >= since)
          .map((item) => ({
            id: item.id,
            title: item.title || item.id,
            startedAt: item.startedAt,
            source: item.source,
            messageCount: item.messageCount,
            model: item.model,
          }))

        setSessions(recent)
        onLoaded?.(recent)
      } catch (e) {
        if (!cancelled) {
          setError(e instanceof Error ? e.message : String(e))
        }
      } finally {
        if (!cancelled) setLoading(false)
      }
    }

    void load()

    return () => {
      cancelled = true
    }
  }, [days, limit, onLoaded])

  return (
    <div className={className}>
      <label>
        <span>会话</span>
        <select
          value={value ?? ""}
          disabled={disabled || loading}
          onChange={(event) => onChange(event.target.value || null)}
        >
          <option value="">新建会话</option>
          {sessions.map((session) => (
            <option key={session.id} value={session.id}>
              {session.title} · {session.messageCount}条
            </option>
          ))}
        </select>
      </label>

      {loading ? <p>正在读取最近 7 天 sessions…</p> : null}
      {error ? <p>{error}</p> : null}
      {!loading && !error && sessions.length === 0 ? (
        <p>最近 7 天没有可继续的会话</p>
      ) : null}
    </div>
  )
}
```

## 10. HostBridgePanel 接入规则

文件：

```text
src/renderer/src/screens/WebOperator/HostBridgePanel.tsx
```

## 10.1 HostBridgePanel 不再做最终 skill 阻断

HostBridgePanel 可保留初步提示，但最终校验以 `HermesPanelSkill` 为准。

HostBridgePanel 收到事件后只负责传参：

```ts
setHostBridgeTaskRequest({
  requestId,
  taskId,
  formType,
  action,
  callbackUrl,
  skillName,
  requiredSkillName: skillName,
  pageUrl: uniqueUrl,
  uniqueUrl,
  pageContext,
  rawEvent: event,
  createdAt: new Date().toISOString(),
})
```

## 10.2 Dialog 负责最终校验

```text
1. HostBridgePanel 不读取 bridge-config.template.json.allowedSkills。
2. HostBridgePanel 不直接判定 skill 是否安装。
3. HostBridgePanel 只把 skillName 作为 requiredSkillName 传给 Dialog。
4. Dialog 内部 HermesPanelSkill 完成安装状态校验。
5. 校验失败时 Dialog 禁止提交。
```

这样可以保证：

```text
HostBridge 业务事件入口
  ↓
HermesTaskStartDialog
  ↓
HermesPanelSkill 统一校验
  ↓
HermesTaskPanel 统一执行
```

## 14. 错误处理

| 场景                      | 处理                  |
| ----------------------- | ------------------- |
| listInstalledSkills 失败  | 禁止提交，显示错误           |
| requiredSkillName 不存在   | 禁止提交，提示安装           |
| sessions 加载失败           | 允许提交，但只能新建会话        |
| 最近 7 天无 session         | 显示“新建会话”            |
| selected session 不存在    | 自动置空为新建会话           |
| HostBridge 未传 skillName | 禁止提交，提示缺少 skillName |
| 用户取消                    | 不创建任务，不写入 session   |
