结论：**不是三栏 React 结构没有写，而是 Workspaces 模块仍在大量使用 Tailwind utility class，但当前 desktop 全局只加载 `assets/main.css` 的手写设计系统样式，并没有 Tailwind 产物，所以 `flex / grid / h-full / border-* / bg-*` 这些 class 在运行时基本失效。** 页面因此退化成浏览器默认文档流：Status Cards、Sidebar、Settings 内容全部从上往下堆叠，右侧 Panel 也会被挤到下方。

代码证据：

WorkspacesShell 里已经有「StatusCards + grid 三列」的设计，`gridTemplateColumns` 也已经按 left / center / right 计算。问题在于它依赖 `className="flex ... grid ..."` 这一类 Tailwind 样式。

`WorkspacesScreen` 根节点同样只写了 Tailwind class，没有独立 CSS class 承接布局。

`WorkspacesSidebar` 和 `WorkspaceStatusCards` 也全部依赖 Tailwind utility class，所以截图里按钮回退成了浏览器默认按钮样式。 

`main.tsx` 当前只导入 `./assets/main.css`，没有导入 Tailwind 入口或 Workspaces 独立样式。 `main.css` 是自定义设计系统变量与全局布局，不是 Tailwind utility 生成文件。 全局 `.app-content` 本身是 flex 容器并支持子页面占满空间，这部分不用改。

---

## 修复原则

这次不要再改 `main.tsx`，也不要把 Workspaces 样式塞进 `assets/main.css`。

正确做法是：

```text
src/renderer/src/screens/Workspaces/
├── index.tsx
├── Workspaces.css              ← 新增，Workspaces 独立样式入口
├── panels/WorkspacesShell.tsx
├── components/WorkspaceStatusCards.tsx
├── components/WorkspacesSidebar.tsx
├── components/WorkspaceRightPanel.tsx
├── components/WorkspaceRightPanelRail.tsx
└── ...
```

`Workspaces.css` 只在 `Workspaces/index.tsx` 内导入：

```ts
import "./Workspaces.css";
```

这样不会破坏全局唯一 CSS 入口的设计，也不会影响其它页面。
