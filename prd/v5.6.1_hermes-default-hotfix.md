# v5.6.1 Local Hermes Hotfix

**版本**：`v5.6.1_hotfix`  
**基线**：`v5.6` Local Hermes（`local-hermes` 顶栏 Tab）  
**约束**：未修改 `HermesShell.tsx` 组件树（无 `HermesStatusCards` 恢复）。

---

## 变更摘要

### Main（P0）

- `syncGatewayModelSection`：将扁平 `config.yaml` 顶层 `default`/`provider`/`base_url` 同步为 Gateway 可读的 `model:` 段。
- `startGateway`：若 sync 改写配置且 Gateway 已在运行 → `restartGateway` 后返回。
- `restartGateway`：仅当 `!isGatewayRunning()` 时提前返回（外部启动的 Gateway 也可 reload）。
- 单测：`tests/sync-gateway-model-section.test.ts`。

### Renderer（Local Hermes）

| 模块 | 改动 |
|------|------|
| Chat | `toolProgress`、`startNewChat`、hook 内唯一 `onSessionDone`；移除页面重复 `onChatDone` |
| Models | `setActive` 后 `runtime.refresh()`；行内 Edit |
| Sessions | 行内重命名（`workspaces.sessions.rename`） |
| Skills | 左右分栏 + `skills.read()` 预览 |
| Providers | Environment + Config（`HERMES_CONFIG_KEYS`）+ Credential Pool + 只读 Connection |
| i18n | `workspaces.hermes.*`（en / zh-CN） |

## §19 手测清单

| # | 场景 | 通过标准 |
|---|------|----------|
| 1 | 顶栏 Local Hermes | 进入三栏，Network 无 `copilot-serve` / `workspaceChat` 请求 |

### P0 专项（model 400）

1. 删除 `~/.hermes/config.yaml` 内 `model:` 段，保留顶层 `default: <model>`。
2. 重启 Electron → Local Hermes → Start Gateway → 发消息。
3. 确认 yaml 自动出现 `model:\n  default: …`，Chat 无 `model is required`。
