# v5.7.4 SidePanel Hermes — WebOperator Hermes Panel 实施方案

> **版本**：v5.7.4  
> **状态**：Implemented（v5.7.4）  
> **范围**：`copilot-desktop` WebOperator 侧栏 `hermes-task` 面板；Hermes Panel 组件实现于 `src/renderer/src/components/hermes/`  
> **前置**：v5.7.3（`getFrameHtml` / PageStructure 抽取）、v5.6.4（`window.hermesDefaultChat` Local Hermes Chat IPC）

## 1. 背景与目标

### 1.1 问题

当前 `HermesTaskPanel.tsx` 仅为占位 UI：任务输入框 `handleSendTask` 空实现，与 Web Content Host 无上下文联动，也未接入本地 Hermes Agent。

Portal `frontend` 已在邮件等模块落地 **`HermesChatPanel`** 嵌入方案（`useHermesPanelChat` + workspace 注入 + 流式对话）。WebOperator 需要**独立复刻**同等交互形态，但**必须**走 Desktop 本地 IPC，不得复用 Portal 的 HTTP BFF（`/api/hermes/runtime/*`、`/api/email/ai-completion` 等）。

### 1.2 目标

| # | 目标 | 验收 |
|---|------|------|
| G1 | 在 `src/renderer/src/components/hermes/` 建立可复用 Hermes Panel 组件族 | 不 import `@portal/web` / `frontend/modules/hermes` |
| G2 | `HermesTaskPanel` 挂载新 Panel，支持 Web 页面内容与 Hermes Agent 对话 | 选页/抽 HTML 后可在 Hermes 面板提问并流式回复 |
| G3 | **唯一** Chat 传输：`window.hermesDefaultChat`（v5.6.4） | 代码审查无 `fetch('/api/...')`、无 `workspaceChat` |
| G4 | 与 v5.7.3 内容抽取链路打通 | `snapshot` / `getFrameHtml` 产物可注入首轮对话 |
| G5 | 保留现有 Pending Actions 审批 UI | 与 Chat 同面板分区展示，互不阻塞 |

### 1.3 非目标（本版不做）

- 不修改 WebOperator 外壳布局（SideRail + SectionPanel 折叠策略与 v5.7.3 一致）
- 不接入 `workspaceChat` / copilot-serve Profile Chat
- 不引入 Portal `RuntimeWorkspacePanel` 的 BFF 文件树（Desktop 暂无等价 IPC；见 §6.4 Phase 2）
- 不在 Renderer 读写 `config.yaml` / API Key（遵守 API_CONTRACTS § 强制约束）

## 4. Web 内容 → Hermes Agent 数据流

### 4.1 上下文来源（Web Content Host）

| 来源 | API | 产物 |
|------|-----|------|
| 页面快照 | `window.aiosBrowser.snapshot({ includeFrames, includeInteractiveElements })` | frames、elements、url、title |
| Frame HTML | `window.aiosBrowser.getFrameHtml({ frameId, selector?, outer?, maxLength })` | `html` / `text` / `truncated` |
| 浏览器状态 | `useBrowserState` / `useBrowserRuntimeState` | 当前 URL、loading、frameCount |
| CRM 事件（可选） | `useCrmBridgeEvents` | 最近一次 CRM context payload |

### 4.2 上下文聚合：`WebOperatorPageContext`

在 `WebOperatorScreen` 层提供 React Context（或轻量 store），供 PageStructure 与 Hermes Panel 共享：

```ts
type WebOperatorPageContext = {
  /** 稳定键：origin + pathname + selectedFrameId */
  scopeKey: string;
  summary: string; // UI 标题行，如「CRM 客户详情 · iframe#app」
  payload: {
    url: string;
    title: string;
    frameId: string | null;
    frameMeta?: { name; origin; path; url };
    htmlExcerpt?: string;   // getFrameHtml 结果，已截断
    textExcerpt?: string;
    truncated?: boolean;
    capturedAt?: string;
    interactiveElementCount?: number;
  };
};
```

**刷新策略**：

- PageStructure 执行 `getFrameHtml` 成功后，调用 `setPageContext(...)` 更新 Context。
- Toolbar「Refresh Snapshot」可选择性触发主 frame 整页 HTML 抽取（`selector` 空）并写入 Context。
- Hermes 面板标题展示 `context.summary`；无 Context 时禁用 preset 与发送（与 frontend `!context` 行为一致）。

### 4.3 注入 Hermes Agent 的策略（v1：Attachment + 首条 Prefix）

Desktop **无** Portal runtime BFF，v1 对齐 v5.6.4 已有能力：

```
用户首次发送（或无 resumeSessionId）
  │
  ├─ 1. sessionId = resumeSessionId ?? `draft_weboperator`
  │
  ├─ 2. 若 pageContext 有 htmlExcerpt 且本轮尚未注入：
  │      window.hermesDefaultChat.uploadAttachmentBuffers({
  │        session_id: sessionId,
  │        files: [{
  │          name: "web-context/body.html",
  │          mime_type: "text/html",
  │          data: utf8Bytes(htmlExcerpt)
  │        }, {
  │          name: "web-context/meta.json",
  │          mime_type: "application/json",
  │          data: utf8Bytes(JSON.stringify(payload))
  │        }]
  │      })
  │      → 得到 attachment_ids[]
  │
  ├─ 3. 构造 message（首条）：
  │      presetSystemPrompt
  │      + buildWebContextPrefix(context)   // 类似 frontend buildContextPrefix，≤12k
  │      + "[用户]\n" + userText
  │
  └─ 4. window.hermesDefaultChat.sendMessage({
         message, resumeSessionId, history, attachment_ids, model_id?
       })
       ← chat-chunk / chat-tool-progress / chat-done(sessionId)
```

**说明**：

- Hermes Gateway 会将 `attachment_ids` 对应文件纳入 Agent 可读 workspace（与全页 Chat 附件机制相同）。
- 首条 prefix 提供摘要；大 HTML 走附件，避免 message 超长。
- `chat-done` 返回真实 `sessionId` 后：`migrateSessionModelBinding(draft, realId)` 由 Main 自动处理；Renderer 更新 `resumeSessionId` 与 localStorage 绑定。

### 4.4 Session 续聊绑定

```ts
// components/hermes/lib/web-operator-hermes-session-binding.ts
const STORAGE_KEY = "weboperator-hermes-panel-session-bindings";

export function scopeKeyWebOperatorPage(scopeKey: string): string {
  return `weboperator:${scopeKey}`;
}
```

- `scopeKey` 变化（切换 URL/frame）→ 尝试 `getPanelSessionBinding` 恢复 session → `hermesAPI.getSessionMessages(sessionId)` 灌入 MessageList。
- 用户点「清空」→ 清除 binding + 清空 messages + session 回到 draft。

## 6. 核心模块设计

### 6.1 `useWebOperatorHermesPanelChat`

**职责**：Desktop 版 `useHermesPanelChat`，API 面收窄为 v5.6.4。

**Options**：

```ts
type UseWebOperatorHermesPanelChatOptions = {
  pageContext: WebOperatorPageContext | null;
  presetSystemPrompt?: string;
  persistenceKey?: string;  // scopeKeyWebOperatorPage(scopeKey)
  profile?: string;         // 固定 "default"，勿暴露 UI 改 profile
};
```

**Returns**（与 frontend 对齐，便于 Panel 复刻）：

```ts
{
  messages, toolCalls, busy, error, restoring,
  sessionId, workspacePath,  // workspacePath v1 恒 null；Phase 2 填充
  send(text: string), cancel(), clear(),
}
```

**send 伪代码**：

```ts
async function send(userText: string) {
  const isFirst = messages.length === 0;
  let attachmentIds: string[] = [];
  let sessionKey = sessionIdRef.current ?? HERMES_DRAFT_SESSION_ID;

  if (isFirst && pageContext && !injectedRef.current) {
    const upload = await injectWebContextAttachments(sessionKey, pageContext);
    if (!upload.ok) { setError(upload.error); return; }
    attachmentIds = upload.attachmentIds;
    injectedRef.current = true;
  }

  const message = isFirst
    ? `${systemPrompt}\n\n${buildWebContextPrefix(pageContext)}\n[用户]\n${userText}`
    : userText;

  subscribe hermesDefaultChat events;
  await window.hermesDefaultChat.sendMessage({
    message,
    resumeSessionId: sessionIdRef.current,
    history: priorTurns,
    attachment_ids: attachmentIds,
  });
}
```

**事件订阅**：与 `useHermesDefaultChatStream` 相同——`onChunk` 追加 assistant 流式内容；`onToolProgress` 推入 toolCalls；`onDone(sessionId)` 持久化；`onError`；`abort()` → `hermesDefaultChat.abort()`。

**历史恢复**：mount 时若 `persistenceKey` 有 binding → `getSessionMessages` → 映射为 `WebOperatorHermesMessage[]`。

## 9. 验收标准

### 9.1 功能

1. **Gateway 就绪**：default profile Gateway 运行中，Hermes 面板可流式对话。
2. **无 Portal HTTP**：DevTools Network 无 `/api/hermes`、无 `/api/email` 请求；仅 IPC。
3. **上下文注入**：PageStructure 对 CRM iframe 执行 Get HTML → Hermes 面板首条提问「总结页面」→ 回复内容与 iframe 相关。
4. **续聊**：同 URL+frame 关闭侧栏再打开，历史消息恢复（binding + getSessionMessages）。
5. **切换页面**：`scopeKey` 变化后为新会话（或命中新 binding），不串台。
6. **Pending Actions**：Agent 触发需确认操作时，面板底部仍展示 Confirm/Reject。
7. **停止生成**：Composer「停止」调用 `abort-chat`，流式中断。

### 9.2 约束合规

- [ ] 未使用 `workspaceChat`
- [ ] 未在 Renderer 写 `config.yaml` / 模型默认
- [ ] Chat 模型解析走 Main `resolveSendModelId`（不传 model_id 时用 session/default）
- [ ] 附件经 Main 落盘 `profileHome/desktop/chat-attachments/`

### 9.3 手工测试场景（CRM + iframe）

```
1. 打开 WebOperator → 加载 CRM 页
2. focusedPanel → page-structure → 选 iframe → Get HTML
3. focusedPanel → hermes-task → 点「页面摘要」
4. 观察 chat-chunk 流式输出
5. 输入追问「有哪些可点击按钮」→ 回答合理
6. BrowserActionLog / PendingActions 行为与改前一致
```

## 10. 风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| HTML 过大 | 附件上传慢 / Gateway 超限 | `maxLength` 默认 100k；inject 前再截断；meta 与 body 分离 |
| 跨域 iframe 无 HTML | 上下文为空 | UI 提示「该 frame 无法抽取 HTML」；仍可用 snapshot elements 元数据作 prefix |
| `draft_weboperator` 与全页 Chat 共用 draft 键 | 模型绑定串扰 | WebOperator 使用独立 draft 键：`draft_weboperator`（勿用 `draft_default`） |
| 无 workspace 文件 UI | 用户不可浏览注入文件 | v1 toast「已注入 web-context 附件」；Phase 2 文件树 |
| Gateway 未启动 | send 失败 | Panel 顶栏展示 gateway 状态 + 链到 Hermes 屏启动（只读 `hermesAPI.gatewayStatus`） |
