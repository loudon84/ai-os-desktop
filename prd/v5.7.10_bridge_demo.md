# copilot-desktop CRM JSSDK 双向交互验证 PRD

版本：`ver5.7.10-crm-lite-bridge-test`

## 1. 目标

在 copilot-desktop 的 WebOperator 中打开 `crm-lite-layui-electron-demo`，完成两条链路验证：

```text
链路 A：CRM 页面 → Electron

商品查看页点击「同步到 Electron」
→ CRM JSSDK 收集商品主表 + 供应商子表
→ Electron CRM Preload 接收
→ Main Process 校验并记录事件
→ Renderer WebOperator 展示接收到的 CRM 上下文
```

```text
链路 B：Electron → CRM 页面

Electron 生成商品数据
→ Main Process 向当前 CRM WebContentsView 下发 command
→ CRM Preload 转发给页面
→ 商品新增页接收 command
→ 填充表单或写入 data/products.json
```

本版本只做验证闭环，不接入真实 CRM 登录、权限、复杂 iframe、Hermes Agent 工具调用。

## 5. Electron 侧改造范围

### 5.1 新增 CRM bridge preload

新增文件：

```text
src/preload/crm-bridge-preload.ts
```

职责：

```text
1. 向 CRM 页面暴露 window.CopilotDesktopCRM.emit
2. 监听 CRM JSSDK 的 window.postMessage fallback
3. 校验事件来源、事件类型、payload 大小
4. 调用 ipcRenderer.invoke("crm-bridge:emit", event)
5. 接收 Main Process 下发的 crm-bridge:command
6. 转发 command 给 CRM 页面
```

preload 暴露 API：

```ts
contextBridge.exposeInMainWorld("CopilotDesktopCRM", {
  version: "0.1.0",
  isAvailable: () => true,
  emit: (event: unknown) => ipcRenderer.invoke("crm-bridge:emit", event)
});
```

下发 command：

```ts
ipcRenderer.on("crm-bridge:command", (_event, command) => {
  window.postMessage(
    {
      source: "copilot-desktop",
      channel: "crm.desktop.command",
      version: "0.1.0",
      command
    },
    window.location.origin
  );
});
```

注册 IPC：

```ts
ipcMain.handle("crm-bridge:emit", async (event, input) => {
  const result = validateCrmBridgeEvent({
    sender: event.sender,
    senderFrame: event.senderFrame,
    input
  });

  if (!result.ok) {
    return result;
  }

  await saveCrmBridgeEvent(result.event);

  mainWindow?.webContents.send("crm-bridge:on-event", result.event);

  return {
    ok: true,
    requestId: result.event.requestId,
    message: "CRM event received"
  };
});
```

## 7. 校验规则

### 7.1 CRM event 校验

必须校验：

```text
1. event.source === "crm-web"
2. event.type 在白名单中
3. requestId 存在
4. payload.product 存在
5. event.senderFrame.url 属于允许域名
6. payload 大小不超过 512KB
7. 当前 webContents 是 web-operator view
```

事件白名单：

```ts
const ALLOWED_CRM_EVENTS = [
  "crm.product.context.submit",
  "crm.context.submit"
];
```

允许域名：

```ts
const ALLOWED_ORIGINS = [
  "http://localhost:5178",
  "https://crm.company.com"
];
```

### 7.2 command 校验

必须校验：

```text
1. command.commandId 存在
2. command.type 在白名单中
3. payload.product 存在
4. 当前 active WebOperator 已打开 CRM 页面
5. 禁止向非白名单 URL 下发 command
```

command 白名单：

```ts
const ALLOWED_CRM_COMMANDS = [
  "desktop.crm.product.fillForm",
  "desktop.crm.product.create"
];
```

```text

4. 打开商品查看页
5. 点击「同步到 Electron」
```


## 11. Definition of Done

```text
1. WebOperator 可打开 crm-lite-layui-electron-demo。
2. 商品查看页点击「同步到 Electron」后，Electron 能收到完整商品上下文。
3. Electron 能向商品新增页下发商品数据并填充表单。
4. Electron 能向商品新增页下发商品数据并写入 data/products.json。
5. 所有新增 IPC 均通过 preload 暴露，不允许 Renderer 直接访问 Node。
6. 非白名单 origin 的事件被拒绝。
7. 非白名单 command 被拒绝。
8. 现有 browser.open / screenshot / click / type 不受影响。
9. 新增功能有最小测试用例。
```
