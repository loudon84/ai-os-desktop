# copilot-desktop v5.7.9 copilot-desktop-tray 需求方案 PRD

## 1. 版本信息

版本号：`v5.7.9 copilot-desktop-tray`

目标模块：

```text
src/main/index.ts
src/main/shell/tray-manager.ts
src/main/shell/tray-icon-path.ts
electron-builder.yml
build/icon.ico
resources/icon.png
```

目标平台：

```text
Windows 10+
Windows 11
```

本版本解决 Copilot Desktop 在 Windows 环境下关闭主窗口后，系统托盘图标无法显示、应用直接退出、托盘图标资源加载不稳定的问题。

---

## 2. 当前问题

### 2.1 现象

在 Windows 上运行 Copilot Desktop：

```text
1. 点击窗口右上角关闭按钮。
2. 主窗口关闭。
3. Windows 托盘区域没有显示 Copilot Desktop 图标。
4. 应用进程可能直接退出。
5. 后台 Hermes / Copilot Serve / Web Operator 不能保持托管运行。
```

### 2.2 影响

```text
1. 无法实现关闭窗口后后台运行。
2. 用户无法通过托盘重新打开主窗口。
3. 用户无法通过托盘菜单执行退出。
4. Windows 自启动到托盘模式不可用。
5. 桌面端作为本地 Hermes 托管入口的体验不完整。
```

---

## 3. 源码现状

### 3.1 index.ts 已有托盘初始化

当前主进程在 `app.whenReady()` 后执行：

```ts
trayManager = createTrayManager(mainWindow);
trayManager?.create();
```

说明项目已经有托盘模块，不需要新增新的托盘架构。

### 3.2 index.ts 已有 close-to-tray 逻辑

当前窗口 close 事件中已有：

```ts
if (trayManager?.isCreated() && !isQuitting) {
  event.preventDefault();
  mainWindow!.hide();
}
```

说明关闭窗口时隐藏到托盘的设计已经存在。

### 3.3 window-all-closed 仍会直接退出

当前逻辑：

```ts
app.on("window-all-closed", () => {
  if (process.platform !== "darwin") {
    stopGateway();
    stopSshTunnel();
    stopClaw3d();
    if (browserToolServer) browserToolServer.stop();
    teardownCrmBridge();
    app.quit();
  }
});
```

问题：

```text
1. 如果 Tray 创建失败，close 事件不会 preventDefault。
2. BrowserWindow 真正关闭。
3. window-all-closed 被触发。
4. 应用直接退出。
```

### 3.4 Windows 托盘图标路径错误

当前 `tray-manager.ts` 默认配置：

```ts
windowsIconPath: join(__dirname, "../../resources/icon.ico")
```

实际仓库资源：

```text
build/icon.ico      存在
resources/icon.png  存在
resources/icon.ico  不存在
```

因此 Windows 下 `nativeImage.createFromPath(resources/icon.ico)` 会得到空图标或加载失败。

### 3.5 electron-builder 未复制托盘 ico 到 resources 根目录

当前 `electron-builder.yml` 的 `extraResources` 复制了：

```text
resources/profiles/
resources/skills/
resources/crm-bridge/
resources/profile-presets/
```

未复制：

```text
build/icon.ico
```

打包后主进程不能稳定通过 `process.resourcesPath/icon.ico` 获取 Windows 托盘图标。

## 5. 总体方案

### 5.1 托盘生命周期

```text
app.whenReady
  ↓
createWindow()
  ↓
createTrayManager(mainWindow)
  ↓
trayManager.create()
  ↓
主窗口 ready-to-show
  ↓
正常显示窗口或 --hidden 模式保持隐藏

用户点击窗口关闭
  ↓
mainWindow.close
  ↓
trayManager.isCreated=true 且 isQuitting=false
  ↓
event.preventDefault()
  ↓
mainWindow.hide()
  ↓
应用继续运行，托盘图标保留

用户点击托盘 Quit
  ↓
onQuit()
  ↓
isQuitting=true
  ↓
app.quit()
  ↓
before-quit 清理资源
  ↓
退出进程
```

### 5.2 图标解析策略

新增 `resolveTrayIconPath()`，按平台和运行模式解析图标路径。

Windows 优先级：

```text
1. process.resourcesPath/icon.ico
2. process.resourcesPath/resources/icon.ico
3. app.getAppPath()/build/icon.ico
4. app.getAppPath()/resources/icon.ico
5. app.getAppPath()/resources/icon.png
```

开发环境：

```text
1. build/icon.ico
2. resources/icon.ico
3. resources/icon.png
```

打包环境：

```text
1. process.resourcesPath/icon.ico
2. process.resourcesPath/resources/icon.ico
3. fallback png
```

### 5.3 关闭行为

主窗口关闭分两类：

```text
普通关闭：
  隐藏窗口，不退出进程。

真正退出：
  由托盘 Quit、菜单 Quit、app:quit IPC、更新安装流程触发。
```

判断变量：

```ts
let isQuitting = false;
```

所有真正退出路径必须先设置：

```ts
isQuitting = true;
```

## 6. 文件级改造

## 6.1 新增 tray-icon-path.ts

新增文件：

```text
src/main/shell/tray-icon-path.ts
```

职责：

```text
1. 解析 Windows / macOS / Linux 托盘图标路径。
2. 兼容开发环境和打包环境。
3. 检查文件是否存在。
4. 找不到图标时抛出明确错误。
5. 输出调试日志。
```

参考实现：

```ts
import { app } from "electron";
import { existsSync } from "fs";
import { join } from "path";

function unique(paths: string[]): string[] {
  return Array.from(new Set(paths));
}

export function resolveTrayIconPath(): string {
  const appPath = app.getAppPath();
  const resourcesPath = process.resourcesPath;

  const windowsCandidates = unique([
    join(resourcesPath, "icon.ico"),
    join(resourcesPath, "resources", "icon.ico"),
    join(appPath, "build", "icon.ico"),
    join(appPath, "resources", "icon.ico"),
    join(appPath, "resources", "icon.png"),
  ]);

  const darwinCandidates = unique([
    join(resourcesPath, "iconTemplate.png"),
    join(resourcesPath, "resources", "iconTemplate.png"),
    join(appPath, "resources", "iconTemplate.png"),
    join(appPath, "resources", "icon.png"),
  ]);

  const linuxCandidates = unique([
    join(resourcesPath, "icon.png"),
    join(resourcesPath, "resources", "icon.png"),
    join(appPath, "resources", "icon.png"),
  ]);

  const candidates =
    process.platform === "win32"
      ? windowsCandidates
      : process.platform === "darwin"
        ? darwinCandidates
        : linuxCandidates;

  const found = candidates.find((candidate) => existsSync(candidate));

  if (!found) {
    throw new Error(`[TRAY] Tray icon not found. candidates=${candidates.join("; ")}`);
  }

  console.log(`[TRAY] Tray icon resolved: ${found}`);
  return found;
}
```

export function createTrayManager(
  mainWindow: BrowserWindow,
  iconConfig?: Partial<TrayIconConfig>,
  menuConfig?: Partial<TrayMenuConfig>
): TrayManager {
  const defaultIconConfig: TrayIconConfig = {
    iconPath: join(__dirname, "../../resources/icon.png"),
    macTemplateIconPath: join(__dirname, "../../resources/iconTemplate.png"),
    windowsIconPath: join(__dirname, "../../resources/icon.ico"),

改为：

```ts
export function createTrayManager(
  mainWindow: BrowserWindow,
  iconConfig?: Partial<TrayIconConfig>,
  menuConfig?: Partial<TrayMenuConfig>,
  onQuit: () => void = () => app.quit(),
): TrayManager {
  const trayIconPath = resolveTrayIconPath();

  const defaultIconConfig: TrayIconConfig = {
    iconPath: trayIconPath,
    macTemplateIconPath: trayIconPath,
    windowsIconPath: trayIconPath,
    ...iconConfig,
  };

  const defaultMenuConfig: TrayMenuConfig = {
    showWindowLabel: "Show Window",
    hideWindowLabel: "Hide Window",
    quitLabel: "Quit",
    gatewayStatusLabel: "Gateway",
    currentProfileLabel: "Profile",
    ...menuConfig,
  };

  trayManagerInstance = new TrayManager(
    mainWindow,
    defaultIconConfig,
    defaultMenuConfig,
    onQuit,
  );

  return trayManagerInstance;
}
```

### 6.3.1 提前初始化 Tray

当前 Tray 初始化在 ModalManager、ShellView、Copilot Serve 等之后。

调整为：

```text
createWindow() 之后，只要 mainWindow 存在，立即创建 Tray。
```

推荐位置：

```ts
setupIPC();

createWindow();

if (mainWindow) {
  try {
    trayManager = createTrayManager(
      mainWindow,
      undefined,
      undefined,
      () => {
        isQuitting = true;
        app.quit();
      },
    );
    trayManager.create();
    console.log("[TRAY] Tray initialized");
  } catch (err) {
    console.error("[TRAY] Failed to initialize tray:", err);
  }
}
```

然后删除后面重复的 Tray 初始化块，避免重复创建。

### 6.3.2 修改 window close 事件

当前：

```ts
mainWindow.on("close", (event) => {
  if (trayManager?.isCreated() && !isQuitting) {
    event.preventDefault();
    mainWindow!.hide();
    console.log("[TRAY] Window hidden to tray");
  }
});
```

改为：

```ts
mainWindow.on("close", (event) => {
  if (isQuitting) return;

  if (trayManager?.isCreated()) {
    event.preventDefault();
    mainWindow!.hide();
    trayManager.updateMenuState?.();
    console.log("[TRAY] Window hidden to tray");
    return;
  }

  console.warn("[TRAY] Tray is not available, window close will quit the app");
});
```

如果不新增 `updateMenuState()`，保留现有 `hideWindow()` 内部更新菜单即可，不强制添加。

### 6.3.3 修改 window-all-closed

当前：

```ts
app.on("window-all-closed", () => {
  if (process.platform !== "darwin") {
    stopGateway();
    stopSshTunnel();
    stopClaw3d();
    if (browserToolServer) browserToolServer.stop();
    teardownCrmBridge();
    app.quit();
  }
});
```

改为：

```ts
app.on("window-all-closed", () => {
  if (process.platform === "darwin") return;

  if (trayManager?.isCreated() && !isQuitting) {
    console.log("[TRAY] window-all-closed ignored because tray is active");
    return;
  }

  stopGateway();
  stopSshTunnel();
  stopClaw3d();
  if (browserToolServer) browserToolServer.stop();
  teardownCrmBridge();
  app.quit();
});
```

## 6.5 新增图标资源检查脚本

新增文件：

```text
scripts/check-tray-assets.cjs
```

职责：

```text
1. 检查 build/icon.ico 是否存在。
2. 检查 resources/icon.png 是否存在。
3. Windows 打包前发现缺失直接失败。
```

参考实现：

```js
const fs = require("fs");
const path = require("path");

const required = [
  "build/icon.ico",
  "resources/icon.png",
];

let failed = false;

for (const file of required) {
  const abs = path.resolve(process.cwd(), file);
  if (!fs.existsSync(abs)) {
    console.error(`[TRAY ASSET] Missing: ${file}`);
    failed = true;
  } else {
    console.log(`[TRAY ASSET] OK: ${file}`);
  }
}

if (failed) {
  process.exit(1);
}
```

修改 `package.json`：

```json
{
  "scripts": {
    "check:tray-assets": "node scripts/check-tray-assets.cjs",
    "build:win": "npm run check:tray-assets && npm run build && electron-builder --win"
  }
}
```

如果不希望修改现有 `build:win`，新增：

```json
{
  "scripts": {
    "build:win:checked": "npm run check:tray-assets && npm run build && electron-builder --win"
  }
}
```

本版本推荐修改 `build:win`，避免 Windows 包缺少托盘资源。

## 11. 风险与处理

### 11.1 风险：开发环境和打包环境路径不同

处理：

```text
使用 resolveTrayIconPath() 同时支持 app.getAppPath() 和 process.resourcesPath。
```

### 11.2 风险：nativeImage 加载空图标

处理：

```text
调用 icon.isEmpty()。
为空直接抛错，并打印路径。
```

### 11.3 风险：Tray 未创建导致窗口关闭后退出

处理：

```text
1. Tray 初始化提前到 createWindow 后立即执行。
2. close 事件中只有 trayManager.isCreated=true 才 preventDefault。
3. window-all-closed 增加 tray active 兜底判断。
```

### 11.4 风险：Quit 菜单未设置 isQuitting

处理：

```text
TrayManager 不直接 app.quit。
通过 onQuit 回调统一设置 isQuitting=true。
```

### 11.5 风险：打包遗漏 icon.ico

处理：

```text
1. electron-builder.yml 增加 extraResources。
2. package.json build:win 前执行 check:tray-assets。
```
