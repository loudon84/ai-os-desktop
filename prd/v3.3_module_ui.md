## v3.3 方案结论

采用 **组件与样式迁移 + 页面改造 + Portal WebContentsView 嵌入** 的混合方案可以落地，但 v3.3 的核心不是先改样式，而是先补齐 **Desktop Auth Endpoint Config + Main 侧 Token Vault + AIOS Home 专用 Header Injection**。

当前 `ai-os-desktop` 已经具备基础接入点：`preload/index.ts` 已暴露 `desktopAuth`、`desktopUserConfig`、`shellView` 等桥接对象；Main 侧已注册 `registerAuthIpc()`、`installTokenHeaderInjector()`，并在窗口创建后注册 `registerUserConfigIpc(mainWindow)` 与 `ShellViewManager`。这说明 v3.3 不需要推翻现有 IPC 架构，只需要补强 auth 配置、token 隔离、WebContentsView partition 策略。  

`ai-os-full/frontend` 是 Next 14 前端工程，依赖中已有 `next-auth`、`axios`、`react-hook-form`、`zod` 等可复用登录页相关基础能力；`tsconfig` 显示其使用 `app/[lang]` 多语言 App Router 结构。因此登录迁移应按“抽 UI 组件 + 适配 Desktop Auth Bridge”的方式做，不把 Web 端 token 存储逻辑原样迁入 Desktop Renderer。

```ts
export function normalizeBaseUrl(url: string): string {
  return url.replace(/\/+$/, "");
}

export function normalizePrefix(prefix: string): string {
  const value = prefix.trim();
  if (!value.startsWith("/")) return `/${value}`;
  return value.replace(/\/+$/, "");
}

export function buildAuthUrl(
  config: AuthEndpointConfig,
  path: "login" | "refresh" | "me",

  return `${normalizeBaseUrl(config.backendUrl)}${normalizePrefix(
```

## 9. Token Header Injector

### 9.1 只绑定 aios-home session

修改：

```text
src/main/auth/token-header-injector.ts
```

核心规则：

```ts
import { session } from "electron";
import { SHELL_PARTITIONS } from "../../shared/shell/browser-partitions";

export function installTokenHeaderInjector(): void {
  const aiosHomeSession = session.fromPartition(SHELL_PARTITIONS.AIOS_HOME);

  aiosHomeSession.webRequest.onBeforeSendHeaders(
    { urls: ["http://*/*", "https://*/*"] },
    (details, callback) => {
      const policy = getCurrentInjectionPolicy();

      if (!policy.enabled) {
        callback({ requestHeaders: details.requestHeaders });
        return;
      }

      if (!isAllowedUrl(details.url, policy.allowedOrigins)) {
        callback({ requestHeaders: details.requestHeaders });
        return;
      }

      const token = getCachedAccessToken();
      if (!token) {
        callback({ requestHeaders: details.requestHeaders });
        return;
      }

      callback({
        requestHeaders: {
          ...details.requestHeaders,
          Authorization: `Bearer ${token}`,
        },
      });
    },
  );
}
```

### 9.2 白名单生成

```ts
export function buildAllowedOrigins(config: AuthEndpointConfig): string[] {
  const origins = new Set<string>();

  origins.add(new URL(config.aiosHomeUrl).origin);
  origins.add(new URL(config.backendUrl).origin);

  return Array.from(origins);
}
```

### 9.3 URL 判断

```ts
export function isAllowedUrl(url: string, allowedOrigins: string[]): boolean {
  try {
    const origin = new URL(url).origin;
    return allowedOrigins.includes(origin);
  } catch {
    return false;
  }
}
```

### 9.4 external-browser / web-operator 隔离验收

必须通过以下测试：

```text

   http://127.0.0.1:3000/*
```
