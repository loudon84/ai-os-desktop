# WebOperatorHermesChatPanel 附件上传改造方案

版本建议：`v5.7.5_weboperator-hermes-panel-attachments`

## 1. 源码结论

`WebOperatorHermesChatPanel.tsx` 当前只是把用户文本透传给 `chat.send(text)`，没有附件状态、上传入口、附件列表、拖拽上传能力。面板结构为：

* `useWebOperatorHermesPanelChat(...)` 管理会话、流式输出、工具调用。
* `WebOperatorHermesPanelMessageList` 渲染消息。
* `WebOperatorHermesPanelComposer` 负责输入与发送。当前发送调用为 `onSend={(text) => void chat.send(text)}`。

`WebOperatorHermesPanelComposer.tsx` 当前只维护 textarea 文本，`submit()` 要求 `value.trim()` 非空；发送按钮也用 `!value.trim()` 禁用，因此不支持“只上传附件后发送”。 

Hermes 模块 Chat 已有完整附件模式：

* `HermesDefaultWebChatSurface` 从 `useHermesDefaultWebChat()` 取出 `attachments`。
* 支持拖拽文件 `attachments.uploadFiles(files)`。
* 发送时把 `attachments.attachments.map((a) => a.id)` 作为 `attachment_ids` 传给 `stream.send(...)`。
* `ComposerBar` 已支持附件列表、选择上传、删除附件、拖拽上传。 

底层 IPC 已经可复用，不需要新增 main/preload 通道：

* preload 已暴露 `uploadAttachments`、`uploadAttachmentBuffers`、`uploadDroppedAttachments`、`removeAttachment`、`sendMessage`。
* main 已注册 `hermes-chat:upload-attachments`、`hermes-chat:upload-attachment-buffers`、`hermes-chat:remove-attachment`、`hermes-chat:send-message`。
* main 的附件落盘逻辑已保存到 profile 的 `desktop/chat-attachments/{sessionId}`，并写入 index。 
* 发送前 `buildUserMessageContent(...)` 会按 `attachmentIds` 读取附件，文本类附件拼入 prompt，图片转成 base64 multimodal part。

因此本次改造应只改 Renderer 侧 WebOperator Hermes Panel，不改 Main IPC。

---

## 2. 改造目标

给 `src/renderer/src/components/hermes/panel/WebOperatorHermesChatPanel.tsx` 增加与 Hermes Chat 一致的附件能力：

1. 点击按钮选择本地文件上传。
2. 支持拖拽文件到输入区上传。
3. 输入区上方显示已上传附件 chip。
4. 支持删除附件。
5. 发送消息时携带 `attachment_ids` 给 hermes-agent。
6. 支持“只有附件，无文本”发送。
7. 保持现有 WebOperator 页面上下文附件注入能力不丢失。

## 4. 关键实现设计

### 4.1 hermesPanelApi 扩展

当前 `hermesPanelApi` 已有 `uploadAttachmentBuffers(...)`，用于 WebOperator 自动注入页面 HTML 上下文。

新增：

```ts
uploadAttachments(payload)
uploadDroppedAttachments(payload, files)
removeAttachment(attachmentId)
```

全部继续走 `window.hermesDefaultChat`，profile 固定使用 `HERMES_PANEL_DEFAULT_PROFILE`。该常量当前为 `"default"`，并且 panel draft session 已隔离为 `draft_weboperator`。

### 4.2 useWebOperatorHermesPanelChat 增加附件状态

新增状态：

```ts
const [attachments, setAttachments] = useState<HermesChatAttachmentMeta[]>([]);
```

新增方法：

```ts
uploadAttachments()
uploadDroppedAttachments(files: FileList)
removeAttachment(id: string)
clearAttachments()
```

session 选择规则：

```ts
sessionIdRef.current
  ?? task?.sessionId
  ?? HERMES_PANEL_DRAFT_SESSION_ID
```

原因：

* 现有 WebOperator 首次发送前没有真实 Hermes session。
* 当前已有 `draft_weboperator`，就是为了隔离 WebOperator 草稿会话。
* 发送时 main 会通过 `attachment_ids` 从 index 读取文件内容，不依赖真实 sessionId 目录名。

### 4.3 sendInternal 合并两类附件

`useWebOperatorHermesPanelChat` 当前第一次发送会调用 `injectWebContextAttachments(...)`，把页面 HTML / meta.json 作为附件传给 Hermes。

改造后需要合并：

```ts
const userAttachmentIds = attachmentsRef.current.map((a) => a.id);
const webContextAttachmentIds = inj.attachmentIds;

const attachmentIds = [
  ...webContextAttachmentIds,
  ...userAttachmentIds,
];
```

发送 payload：

```ts
await hermesPanelApi.sendMessage({
  message,
  resumeSessionId,
  history,
  attachment_ids: attachmentIds.length ? attachmentIds : undefined,
});
```

保持现有页面上下文注入逻辑不变，只是追加用户附件。

### 4.4 支持附件-only 发送

当前 `sendInternal` 有：

```ts
if (!trimmed || busy || restoring) return;
```

需要调整为：

```ts
const hasUserAttachments = attachmentsRef.current.length > 0;
if ((!trimmed && !hasUserAttachments) || busy || restoring) return;
```

展示消息：

```ts
content: displayContent || "(attachments)"
```

底层 Hermes Chat 参考实现也允许 `!trimmed && attachmentIds.length > 0`。

## 5. UI 行为

### 5.1 Composer 增加 props

`WebOperatorHermesPanelComposer` 新增：

```ts
attachments: HermesChatAttachmentMeta[];
onUploadAttachment: () => void;
onUploadDroppedAttachments: (files: FileList) => void;
onRemoveAttachment: (id: string) => void;
```

`onSend` 保持：

```ts
onSend: (text: string) => void;
```

### 5.2 发送按钮禁用规则

原规则：

```ts
disabled={busy || disabled || !value.trim()}
```

新规则：

```ts
const canSend = value.trim().length > 0 || attachments.length > 0;

disabled={busy || disabled || !canSend}
```

### 5.3 UI 放置

在 textarea 上方放附件 tray：

```text
composer
  ├─ attachment tray
  ├─ textarea
  └─ actions
      ├─ 上传附件
      ├─ 停止
      └─ 发送
```

不建议直接复用 `screens/Hermes/pages/Chat/AttachmentTray.tsx`，避免 `components/hermes` 反向依赖 `screens/Hermes` 页面模块。可以在 `WebOperatorHermesPanelComposer.tsx` 内部实现轻量 chip，或新增：

```text
src/renderer/src/components/hermes/panel/WebOperatorHermesPanelAttachmentTray.tsx
```

## 8. 注意点

当前底层附件处理对文本、Markdown、JSON、图片支持较好；PDF 目前只是按已有逻辑读取字节并尝试转文本预览，未做 OCR 或 PDF parser。这是 Hermes Chat 现有能力边界，本次 WebOperator Panel 按“与 Hermes Chat 一致”复用即可，不在本次范围内扩展 PDF 解析。
