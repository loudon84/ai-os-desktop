## 结论

现有 `ai-os-desktop` 已经具备 multi profiles 的基础骨架：SQLite 表、Profile Runtime Manager、Hermes Local Adapter、IPC、Profile Entry、Capability、Skill、Delegation、Session Share 都已存在。下一阶段不要重做模块，应在现有 `profile-runtime` 体系上补齐 **角色库导入、Profile 预设、角色文件生成、Hermes Gateway 真隔离启动、UI 管理入口**。

核心改造点：

1. 保留现有 `profiles / runtime_instances / profile_entries / profile_capabilities / profile_skills` 结构。
2. 新增 `profile_role_specs` 表，记录角色来源、源文件、角色版本、checksum。
3. 新增 `resources/profile-presets/hermes-expert-profiles.v1.yaml`，一次性创建 6 个专家 profile。
4. 新增 `role-library-sync / role-compiler`，从 `agency-agents-zh` 拉取角色源并生成每个 profile 的 `SOUL.md / MEMORY.md / profile-role.json`。
5. 修正 `hermes-local-adapter`，启动时必须让每个 Hermes Gateway 读取自己的 profile home，否则多端口不等于多角色隔离。
6. Settings Drawer 增加 Multi Profiles 管理页，支持安装预设、查看角色来源、启动/停止、同步角色、查看日志。

## 1. 当前 ai-os-desktop multi profiles 源码判断

当前项目已经有 SQLite profile runtime 体系。`profile-runtime-db.ts` 中已经定义了 `profiles`、`runtime_instances`、`profile_entries`、`profile_capabilities`、`profile_skills`、`skill_sync_events`、`shared_contexts`、`delegation_events`、`audit_events` 等表，适合作为 multi profiles 的持久层基础。

现有 Runtime Manager 已实现 profile 状态机、端口占用检查、启动超时、启动/停止/重启/批量启动能力，下一阶段只需要增强角色创建和启动隔离，不需要推翻。

现有 `config-importer.ts` 已经支持从 YAML 导入 profiles，校验 profile 名称、端口、runtimeType、capabilities，并自动创建 profile 目录、`skills / memories / desktop / desktop/shared-context` 子目录和 `SOUL.md`。

现有 IPC 已暴露 `importConfig`、`listProfiles`、`startProfile`、`stopProfile`、`restartProfile`、`startAll`、`stopAll`、`delegate`、`copySkill`、`shareSessionContext`、`getGatewayLogs` 等接口，足够支撑 Settings Drawer 的 multi profiles UI。

主要风险在 `hermes-local-adapter.ts`：当前会向每个 profile home 写 `config.yaml` 中的 `api_server` 端口，并读取 profile home 下的 `.env`，但启动命令仍是 `spawn(HERMES_SCRIPT, ["gateway"], { cwd: HERMES_REPO, env })`，没有显式设置 `HERMES_HOME=profileHome` 或 Hermes profile 参数。这样会导致 Gateway 可能仍读取默认 Hermes 配置，无法保证每个 profile 的 `SOUL.md / config.yaml / skills` 被真实隔离加载。

此外，`config-importer.ts` 的 overwrite 分支调用 `require("./profile-runtime-db").getDb()`，但 `profile-runtime-db.ts` 中 `getDb()` 是内部函数，没有导出。后续如果开启覆盖导入，会有运行时风险。

## 2. agency-agents-zh 角色库取用方式

`agency-agents-zh` 是一个中文 AI 角色库，README 中说明其覆盖工程、设计、营销、金融等部门，并且不是普通提示词模板，而是带有身份定义、工作流程和交付物的角色定义。([GitHub][1])

该仓库明确支持安装到 Hermes Agent，工具列表中 Hermes Agent 的安装位置为 `~/.hermes/skills/`，并提示部分工具需要先执行格式转换。([GitHub][1])

对本项目的正确用法不是直接把整个仓库复制进 Hermes，而是：

```text
agency-agents-zh 原始角色源
        ↓
role-library-sync 拉取 / 更新
        ↓
role-compiler 选取指定目录或文件
        ↓
生成每个 Hermes profile 的：
  - SOUL.md
  - MEMORY.md
  - profile-role.json
  - skills/role-source/*.md
        ↓
profile-runtime-db 记录 role source / checksum / version
```

## 5. 新增数据库表

新增 migration version：`CURRENT_SCHEMA_VERSION = 3`

```sql
CREATE TABLE IF NOT EXISTS profile_role_specs (
  id TEXT PRIMARY KEY,
  profile_id TEXT NOT NULL,
  role_key TEXT NOT NULL,
  role_name TEXT NOT NULL,
  role_source_repo TEXT NOT NULL,
  role_source_paths_json TEXT NOT NULL,
  role_summary TEXT,
  role_manifest_path TEXT NOT NULL,
  soul_path TEXT NOT NULL,
  memory_path TEXT,
  source_checksum TEXT NOT NULL,
  installed_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY(profile_id) REFERENCES profiles(id) ON DELETE CASCADE,
  UNIQUE(profile_id, role_key)
);

CREATE INDEX IF NOT EXISTS idx_profile_role_specs_profile
ON profile_role_specs(profile_id);

CREATE INDEX IF NOT EXISTS idx_profile_role_specs_key
ON profile_role_specs(role_key);
```

不建议把这些字段塞进 `profiles.description` 或 `config_json`，原因是角色源、checksum、同步状态属于可审计数据，应独立持久化。

## 8. Hermes Local Adapter 必须修正

文件：

```text
src/main/hermes-local-adapter.ts
```

当前启动代码需要补充 profile home 环境隔离。

目标修改：

```ts
const env = { ...process.env } as Record<string, string>;

env.HERMES_HOME = home;
env.HERMES_PROFILE = profile.name;
env.HERMES_PROFILE_HOME = home;
env.HERMES_GATEWAY_HOST = instance.host;
env.HERMES_GATEWAY_PORT = String(instance.port);
```

同时 stop 阶段必须使用当前 profile home：

```ts
const pidPath = join(profileHome(profile.name), "gateway.pid");
```

而不是：

```ts
const pidPath = join(profileHome(), "gateway.pid");
```

否则停止非 default profile 时可能误读 default pid 文件。
