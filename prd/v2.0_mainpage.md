已确认采用 **方案 B**。

执行边界定义如下：

```txt
MainPage 替代 DesktopShell 成为一级桌面容器；
DesktopSidebar 保留，但降级为 MainPage 内部的二级功能导航；
WorkspaceOutlet / StatusBar / ModalLayer / DrawerLayer 保留；
PageHeader 不再作为全局 Header，只作为页面内 Header 或逐步废弃；
多 WebView Tabs 由 MainPage 顶部统一承载。
```

## 1. 当前改造目标

当前 `Layout.tsx` 直接组合 `DesktopShell + DesktopSidebar + PageHeader + WorkspaceOutlet + StatusBar`。
当前 `DesktopShell` 是固定的 `aside + section + header + main + footer` 结构。

改造后：

```tsx
App
  └─ Layout
      └─ MainPage
          ├─ MainTopBar
          ├─ DesktopSidebar
          ├─ WorkspaceOutlet
          ├─ StatusBar
          ├─ ModalLayer
          └─ DrawerLayer
```

`Layout` 暂时保留，负责组织 navigation hook、profile、update state；`DesktopShell` 从主链路移除。

## 6. 顶部 Tabs 初版

当前 `package.json` 没有 DnD 依赖，不能直接复制 Mattermost 的 `react-beautiful-dnd` 模式。
因此第一版先做 **可点击 Tabs**，不引入新依赖；DnD 排序作为第二阶段。

`src/renderer/src/screens/MainPage/MainViewTabs.tsx`

```tsx
interface MainViewTabsProps {
  activeView: string;
  onNavigate: (view: string) => void;
}

const WORKSPACE_TABS = [
  { id: "aios-home", label: "AI-OS" },
  { id: "aios-workspace", label: "Workspace" },
  { id: "web-operator", label: "Web Operator" },
];

export function MainViewTabs({
  activeView,
  onNavigate,
}: MainViewTabsProps): React.JSX.Element {
  return (
    <nav className="MainViewTabs no-drag" aria-label="Workspace tabs">
      {WORKSPACE_TABS.map((tab) => (
        <button
          key={tab.id}
          type="button"
          className={`MainViewTabs__item ${activeView === tab.id ? "active" : ""}`}
          onClick={() => onNavigate(tab.id)}
        >
          <span>{tab.label}</span>
        </button>
      ))}
    </nav>
  );
}
```

后续 profile workspace tabs 再从 `profileEntries` 动态生成。

## 9. Sidebar 降级规则

`DesktopSidebar` 不删除，只调整职责：

```txt
原职责：
  主窗口一级导航

新职责：
  MainPage 内部二级功能导航
```

第一阶段保留 232px 展开态。第二阶段再做：

```txt
expanded: 232px
rail: 56px
hidden: 0px
```

不要首版直接压缩成 56px，否则当前 nav label、profileEntries、update button 都要同步重构，改动过大。当前 `DesktopSidebar` 内部还有 profile entries、update 状态等逻辑。

## 10. 本轮不动的部分

```txt
ShellViewManager
WebContentsHost
WorkspaceOutlet
Browser IPC
Profile Runtime IPC
ModalManager
DropdownManager
TrayManager
```

原因：

当前 `WebContentsHost` 已经通过 `ResizeObserver` 同步 DOM bounds 到 `window.shellView.setBounds`，这部分是多 WebView 的正确基础。

本轮只做主窗口壳层替换，避免同时改 IPC、native view 生命周期和样式，降低风险。
