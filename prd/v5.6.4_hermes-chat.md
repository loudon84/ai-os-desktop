# PRD：ver5.6.4 Hermes Default 多模型与 Session 级模型切换修复

## 1. 版本目标

版本号：`ver5.6.4`

目标：

```txt
1. 修复 Hermes Models 页面写入 ~/.hermes/config.yaml 的模型配置结构。
2. Models 页面负责模型注册、custom_providers 同步、默认模型设置。
3. Chat 页面只负责当前 session 的模型选择。
4. Chat 页面移除 Save as Default 能力。
5. 普通聊天切换模型不修改 ~/.hermes/config.yaml。
6. 普通聊天切换模型不重启 Hermes Gateway。
7. Chat 模型运行方式参考 hermes-webui：session.model → AIAgent(model=...) → run_conversation()。
```

## 2. 当前问题

### 2.1 Models 页面配置写入问题

当前 `src/main/config.ts` 的 `setModelConfig()` 只稳定维护根级：

```yaml
provider:
default:
base_url:
```

但 Hermes 可运行多模型时，需要同时维护：

```yaml
provider: "custom"
default: "gemma4:26b"

model:
  provider: "custom"
  default: "gemma4:26b"
  base_url: "http://192.168.70.249:11434/v1"

custom_providers:
- name: gemma4:26b
  base_url: http://192.168.70.249:11434/v1
  model: gemma4:26b
  api_key: ollama
```

问题结果：

```txt
1. root.default 与 model.default 可能不一致。
2. Gateway / AIAgent 仍读取旧的 model.default。
3. models.json 中的多模型没有同步到 config.yaml.custom_providers。
4. Chat 下拉选择模型后，实际仍可能使用默认模型。
```

### 2.2 Chat 页面模型切换问题

当前 Chat 发送链路已经传递 `model_id`，但 Main 侧会执行：

```txt
model_id
  → resolve saved model
  → syncGatewayToSavedModel()
  → setModelConfig()
  → restartGateway()
  → 再调用 Gateway chat completions
```

这不是 session 级模型切换，而是全局默认模型切换。

ver5.6.4 必须改为：

```txt
model_id
  → resolve saved model
  → bind session model
  → AIAgent(model=selected_model)
  → run_conversation()
```

## 3. 关键产品规则

### 3.1 Models 页面规则

```txt
1. Models 页面是模型注册中心。
2. Models 页面维护 models.json。
3. Models 页面同步 ~/.hermes/config.yaml.custom_providers。
4. Models 页面提供 Set Default 操作。
5. Set Default 才允许修改 config.yaml 的 root.default 与 model.default。
6. Add / Edit / Delete 模型后必须同步 custom_providers。
```

### 3.2 Chat 页面规则

```txt
1. Chat 页面模型下拉只影响当前 session。
2. Chat 页面不显示 Save as Default。
3. Chat 页面不调用 setModelConfig。
4. Chat 页面不修改 ~/.hermes/config.yaml。
5. Chat 页面不重启 Gateway。
6. 新 session 第一条消息使用当前下拉选择模型。
7. 已存在 session 继续使用该 session 已绑定模型。
8. 用户在已存在 session 中切换模型后，更新该 session 的模型绑定。
```

### 3.3 默认模型规则

```txt
默认模型只能在 Models 页面设置。
Chat 页面不提供默认模型设置入口。
```

### 7.3 custom_providers 生成规则

```ts
export function buildCustomProviderEntry(model: SavedModel): HermesCustomProvider {
  const entry: HermesCustomProvider = {
    name: model.model,
    base_url: model.baseUrl.replace(/\/+$/, ""),
    model: model.model,
  };

  if (model.apiKeyEnv) {
    entry.key_env = `\${${model.apiKeyEnv}}`;
  }

  if (model.apiKeyLiteral) {
    entry.api_key = model.apiKeyLiteral;
  }

  return entry;
}
```

### 7.4 setDefaultHermesModel 规则

```ts
export function setDefaultHermesModel(profile: string | undefined, modelId: string): void {
  const saved = resolveSavedModelById(modelId);
  const doc = readHermesConfig(profile);

  doc.provider = "custom";
  doc.default = saved.model;

  doc.model = {
    provider: "custom",
    default: saved.model,
    base_url: saved.baseUrl.replace(/\/+$/, ""),
  };

  doc.platforms = {
    ...(doc.platforms ?? {}),
    api_server: {
      host: "127.0.0.1",
      port: 8642,
    },
  };

  doc.smart_model_routing = {
    enabled: false,
  };

  doc.streaming = true;

  doc.custom_providers = buildCustomProvidersFromModels();

  writeHermesConfig(profile, doc);
}
```

## 11. desktop_chat_runner.py 规范

### 11.1 文件职责

```txt
1. 读取 stdin JSON。
2. 设置当前 profile 上下文。
3. 设置模型相关环境变量。
4. 实例化 hermes-agent AIAgent。
5. 调用 run_conversation()。
6. 将 token/tool/usage/done/error 输出为 JSONL。
```

### 11.2 核心伪代码

```python
import json
import os
import sys

def emit(event):
    print(json.dumps(event, ensure_ascii=False), flush=True)

def main():
    payload = json.loads(sys.stdin.read())

    model = payload["model"]
    session_id = payload.get("session_id")
    message = payload["message"]
    history = payload.get("history") or []

    if payload.get("api_key_env"):
        key = payload["api_key_env"]
        value = os.environ.get(key)
        if not value:
            emit({"type": "error", "message": f"Missing API key env: {key}"})
            return

    def on_token(text):
        if text:
            emit({"type": "token", "text": text})

    def on_tool(tool):
        emit({
            "type": "tool",
            "name": str(tool),
            "preview": str(tool)[:120],
        })

    from agent.run_agent import AIAgent

    agent = AIAgent(
        model=model,
        platform="cli",
        quiet_mode=True,
        session_id=session_id,
        stream_delta_callback=on_token,
        tool_progress_callback=on_tool,
    )

    result = agent.run_conversation(
        user_message=message,
        conversation_history=history,
        task_id=session_id,
    )

    emit({
        "type": "done",
        "session_id": session_id or result.get("session_id"),
    })

if __name__ == "__main__":
    try:
        main()
    except Exception as exc:
        emit({"type": "error", "message": str(exc)})
```

Cursor 实施时需要根据当前 hermes-agent 实际 import path 校正：

```txt
agent.run_agent import AIAgent
hermes_agent.agent.run_agent import AIAgent
```

以当前项目已安装 hermes-agent 的真实路径为准。
