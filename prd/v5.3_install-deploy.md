# 1. 命名规则

## 1.1 Runtime 目录命名

| 旧目录                     | 新目录              | 说明                      |
| ----------------------- | ---------------- | ----------------------- |
| `runtime/hermes-agent`  | `runtime/hermes` | Hermes Agent Runtime    |
| `runtime/copilot-serve` | `runtime/serve`  | Copilot Serve Runtime   |
| `runtime/ai-os-full`    | `runtime/portal` | Portal Frontend Runtime |

## 1.2 源码目录命名

源码统一放到各 Runtime 的 `src` 子目录：

```txt
runtime/hermes/src
runtime/serve/src
runtime/portal/src
```

不要直接把 git clone 内容平铺到 `runtime/hermes` 或 `runtime/serve` 根目录。这样可以保持：

```txt
runtime/hermes/venv
runtime/hermes/src
runtime/hermes/logs
```

结构清晰，不和源码文件混在一起。

# 2. venv 创建规则

采用你的规则：

```powershell
uv venv --python=3.12 runtime/hermes/venv
uv venv --python=3.12 runtime/serve/venv
```

落到 Windows 脚本中应写成：

```powershell
uv venv --python=3.12 "$InstallRoot\runtime\hermes\venv"
uv venv --python=3.12 "$InstallRoot\runtime\serve\venv"
```

对应 Python 路径：

```txt
runtime/hermes/venv/Scripts/python.exe
runtime/serve/venv/Scripts/python.exe
```

## 8.3 `copilot-serve-paths.ts`

旧逻辑要支持：

```txt
runtime/copilot-serve
```

新逻辑改为：

```txt
runtime/serve/src
```

核心判断：

```ts
function isValidServeRoot(dir: string): boolean {
  return existsSync(join(dir, "pyproject.toml"));
}
```

新解析顺序：

```ts
export function resolveCopilotServeRoot(): string | null {
  const fromEnv = process.env.COPILOT_SERVE_ROOT?.trim();
  if (fromEnv && isValidServeRoot(fromEnv)) return fromEnv;

  const loc = resolveInstallLocation();

  const sourceRoot = join(loc.runtimeRoot, "serve", "src");
  if (isValidServeRoot(sourceRoot)) return sourceRoot;

  return null;
}
```

venv 路径：

```ts
const serveRuntimeRoot = join(loc.runtimeRoot, "serve");
const serveVenvPython = join(serveRuntimeRoot, "venv", "Scripts", "python.exe");
```

不要再查：

```txt
serveRoot/.venv/Scripts/python.exe
```

而是查：

```txt
runtime/serve/venv/Scripts/python.exe
```
