## ai-os-desktop 安装包统一实施方案

### 1. 当前问题定位

当前仓库的安装包命名存在 3 个不一致点：

| 位置                             |                                     当前值 | 问题                     |
| ------------------------------ | --------------------------------------: | ---------------------- |
| `electron-builder.yml`         |              `productName: SMC Copilot` | 默认产品目录会带空格             |
| `electron-builder.yml`         |    `win.executableName: smc-ai-copilot` | 打包后主程序不是 `desktop.exe` |
| `build/installer.nsh`          |    `$LOCALAPPDATA\Programs\SMC Copilot` | 自定义 NSIS 默认安装目录仍带空格    |
| `install-location-resolver.ts` | `DEFAULT_PROGRAM_FOLDER = SmartCopilot` | 应用运行时兜底目录与安装器目录不一致     |

仓库当前使用 `electron-builder + NSIS` 打包，`electron-builder.yml` 已启用 `nsis.include: build/installer.nsh`，说明默认安装目录实际由 `installer.nsh` 的 `preInit` 接管。 electron-builder 的 NSIS 文档也明确说明，`include` 可用于注入 `preInit/customInstall/customUnInstall` 等宏来自定义安装行为。([electron.build][1])

---

# 2. 统一目标

本次统一后的安装包标准：

```text
默认安装目录:
%LOCALAPPDATA%\Programs\SMC-Copilot

主程序:
desktop.exe

业务注册表:
HKCU\Software\SMC\copilot

保留兼容读取:
HKCU\Software\SMC\Copilot
HKLM\Software\SMC\Copilot
HKCU\Software\SMC\CopilotSMC
HKCU\Software\SMC\HermesDesktop
HKCU\Software\Microsoft\Windows\CurrentVersion\Uninstall\com.nousresearch.hermes
```

注意：`electron-builder` 自己的卸载注册表仍会根据 `appId/guid` 生成。这里的 `HKCU\Software\SMC\copilot` 是业务安装信息注册表，用于 Desktop / Runtime / copilot-serve / HermesLocalService 查找安装目录。

# 3. 文件级实施方案

## 3.1 修改 `electron-builder.yml`

当前配置中 `productName` 为 `SMC Copilot`，`win.executableName` 为 `smc-ai-copilot`，NSIS 快捷方式与卸载名也使用 `SMC Copilot`。

### 修改目标

```yaml
appId: com.smc.smc-ai-copilot
productName: SMC-Copilot

win:
  executableName: desktop
  target:
    - target: nsis
      arch:
        - x64

nsis:
  guid: "7D7C5222-4F0C-4BD0-B877-6D62ED5B941A"
  artifactName: SMC-Copilot-${version}-setup.${ext}
  shortcutName: SMC-Copilot
  uninstallDisplayName: SMC-Copilot
  createDesktopShortcut: always
  createStartMenuShortcut: true
  oneClick: false
  allowToChangeInstallationDirectory: true
  perMachine: false
  selectPerMachineByDefault: false
  deleteAppDataOnUninstall: false
  include: build/installer.nsh
  installerIcon: build/icon.ico
  uninstallerIcon: build/icon.ico
```

### 处理原则

`appId` 和 `nsis.guid` 不建议改。electron-builder 文档说明，Windows 安装升级依赖 `appId/guid` 识别应用，随意修改会破坏升级路径。([electron.build][1])

---

## 3.2 修改 `build/installer.nsh`

当前 `installer.nsh` 已经写入 `Software\SMC\Copilot`，并在首次安装时默认到 `$LOCALAPPDATA\Programs\SMC Copilot`。

### 目标策略

旧目录只作为兼容读取，不再作为默认安装目录。

即：

```text
如果已有安装目录是 SMC-Copilot：
  继续使用

如果已有安装目录是 SMC Copilot / HermesDesktop / CopilotSMC：
  不直接复用为默认目录
  只记录为 LegacyInstallDir
  默认安装到 SMC-Copilot

如果是全新安装：
  默认安装到 SMC-Copilot
```

### 关键替换

#### 1）增加标准注册表常量

在变量区后增加：

```nsh
!define SMC_COPILOT_REG_KEY "Software\SMC\copilot"
!define SMC_COPILOT_LEGACY_REG_KEY "Software\SMC\Copilot"
!define SMC_COPILOT_DEFAULT_DIR "$LOCALAPPDATA\Programs\SMC-Copilot"
```

#### 2）修改 `preInit`

将当前逻辑改成：

```nsh
!macro preInit
  SetRegView 64

  ; 1. Primary normalized registry
  ReadRegStr $ExistingInstallDir HKCU "${SMC_COPILOT_REG_KEY}" "InstallLocation"

  ; 2. HKLM normalized registry
  ${If} $ExistingInstallDir == ""
    ReadRegStr $ExistingInstallDir HKLM "${SMC_COPILOT_REG_KEY}" "InstallLocation"
  ${EndIf}

  ; 3. Legacy SMC Copilot registry
  ${If} $ExistingInstallDir == ""
    ReadRegStr $LegacyInstallDir HKCU "${SMC_COPILOT_LEGACY_REG_KEY}" "InstallLocation"
  ${EndIf}

  ; 4. Legacy CopilotSMC
  ${If} $ExistingInstallDir == ""
    ReadRegStr $LegacyInstallDir HKCU "Software\SMC\CopilotSMC" "InstallLocation"
  ${EndIf}

  ; 5. Legacy HermesDesktop
  ${If} $ExistingInstallDir == ""
    ReadRegStr $LegacyInstallDir HKCU "Software\SMC\HermesDesktop" "InstallLocation"
  ${EndIf}

  ; 6. Legacy com.nousresearch.hermes uninstall entry
  ${If} $ExistingInstallDir == ""
    ReadRegStr $LegacyInstallDir HKCU "Software\Microsoft\Windows\CurrentVersion\Uninstall\com.nousresearch.hermes" "InstallLocation"
  ${EndIf}

  ; 7. Only reuse normalized non-space install dir
  ${If} $ExistingInstallDir == ""
    StrCpy $ExistingInstallDir "${SMC_COPILOT_DEFAULT_DIR}"
  ${EndIf}

  ; Write electron-builder internal install location for NSIS directory page
  WriteRegExpandStr HKCU "${INSTALL_REGISTRY_KEY}" InstallLocation "$ExistingInstallDir"
  SetRegView 32
  WriteRegExpandStr HKCU "${INSTALL_REGISTRY_KEY}" InstallLocation "$ExistingInstallDir"

  StrCpy $INSTDIR "$ExistingInstallDir"
!macroend
```

#### 3）修改 `customInstall` 的注册表写入

替换：

```nsh
WriteRegExpandStr HKCU "Software\SMC\Copilot" "InstallLocation" "$INSTDIR"
WriteRegExpandStr HKCU "Software\SMC\Copilot" "RuntimeRoot" "$INSTDIR\runtime"
WriteRegExpandStr HKCU "Software\SMC\Copilot" "BinDir" "$INSTDIR\bin"
WriteRegStr HKCU "Software\SMC\Copilot" "AppVersion" "${VERSION}"
WriteRegStr HKCU "Software\SMC\Copilot" "InstallMode" "per-user"
```

为：

```nsh
WriteRegExpandStr HKCU "${SMC_COPILOT_REG_KEY}" "InstallLocation" "$INSTDIR"
WriteRegExpandStr HKCU "${SMC_COPILOT_REG_KEY}" "RuntimeRoot" "$INSTDIR\runtime"
WriteRegExpandStr HKCU "${SMC_COPILOT_REG_KEY}" "BinDir" "$INSTDIR\bin"
WriteRegStr HKCU "${SMC_COPILOT_REG_KEY}" "AppVersion" "${VERSION}"
WriteRegStr HKCU "${SMC_COPILOT_REG_KEY}" "InstallMode" "per-user"
```

保留旧注册表读取，但卸载时可以清理旧 key：

```nsh
DeleteRegKey HKCU "${SMC_COPILOT_REG_KEY}"
DeleteRegKey HKCU "${SMC_COPILOT_LEGACY_REG_KEY}"
DeleteRegKey HKCU "Software\SMC\CopilotSMC"
```

## 3.3 修改 `desktop-runtime.json` 生成内容

当前 `installer.nsh` 写入：

```json
{
  "productName": "SMC Copilot",
  "appId": "com.smc.smc-ai-copilot",
  "executableName": "smc-ai-copilot"
}
```

需要改成：

```nsh
FileWrite $2 '  "productName": "SMC-Copilot",$\r$\n'
FileWrite $2 '  "appId": "com.smc.smc-ai-copilot",$\r$\n'
FileWrite $2 '  "executableName": "desktop",$\r$\n'
```

建议同时新增：

```nsh
FileWrite $2 '  "registryKey": "HKCU\\\\Software\\\\SMC\\\\copilot",$\r$\n'
FileWrite $2 '  "legacyProductNames": ["SMC Copilot", "CopilotSMC", "HermesDesktop"],$\r$\n'
```

原因：`desktop-runtime-config.ts` 会读取 `desktop-runtime.json`，并将安装目录、runtimeRoot、binDir、agentDir 等运行时路径作为 Desktop 后续部署 Hermes / serve / portal 的基础。

## 3.4 修改 `src/main/enterprise/windows/install-location-resolver.ts`

当前运行时解析器已经读取：

```ts
HKCU\Software\SMC\Copilot
HKLM\Software\SMC\Copilot
HKCU\Software\SMC\CopilotSMC
HKCU\Software\SMC\HermesDesktop
```

并且 `DEFAULT_PROGRAM_FOLDER` 当前是 `SmartCopilot`。

### 修改目标

```ts
const REGISTRY_KEY_PRIMARY = "HKCU\\Software\\SMC\\copilot";
const REGISTRY_KEY_PRIMARY_HKLM = "HKLM\\Software\\SMC\\copilot";

const REGISTRY_KEY_LEGACY_SMC_COPILOT = "HKCU\\Software\\SMC\\Copilot";
const REGISTRY_KEY_LEGACY_SMC = "HKCU\\Software\\SMC\\CopilotSMC";
const REGISTRY_KEY_LEGACY_HERMES = "HKCU\\Software\\SMC\\HermesDesktop";
const REGISTRY_KEY_LEGACY_UNINSTALL =
  "HKCU\\Software\\Microsoft\\Windows\\CurrentVersion\\Uninstall\\com.nousresearch.hermes";

const DEFAULT_PROGRAM_FOLDER = "SMC-Copilot";
```

### 修改 `readRegistryInstallInfo()`

注册表读取顺序调整为：

```ts
const keys = [
  REGISTRY_KEY_PRIMARY,
  REGISTRY_KEY_PRIMARY_HKLM,
  REGISTRY_KEY_LEGACY_SMC_COPILOT,
  REGISTRY_KEY_LEGACY_SMC,
  REGISTRY_KEY_LEGACY_HERMES,
  REGISTRY_KEY_LEGACY_UNINSTALL,
];
```

这样可以保证：

```text
新安装：
  HKCU\Software\SMC\copilot

旧安装兼容：
  HKCU\Software\SMC\Copilot
  HKCU\Software\SMC\CopilotSMC
  HKCU\Software\SMC\HermesDesktop
```

---

## 3.5 修改 runtime 目录解析

`runtime-paths.ts` 当前通过 `resolveInstallLocation()` 得到 `installRoot/runtimeRoot/binDir`，因此只要 `install-location-resolver.ts` 改成 `SMC-Copilot`，Hermes / serve / portal 的标准 runtime 路径会自动切到：

```text
%LOCALAPPDATA%\Programs\SMC-Copilot\runtime
%LOCALAPPDATA%\Programs\SMC-Copilot\bin
%LOCALAPPDATA%\Programs\SMC-Copilot\runtime\hermes
%LOCALAPPDATA%\Programs\SMC-Copilot\runtime\serve
%LOCALAPPDATA%\Programs\SMC-Copilot\runtime\portal
```

`runtime-paths.ts` 已经统一从 `resolveInstallLocation()` 推导这些目录，不需要重复硬编码。

# 4. 兼容迁移策略

## 4.1 新安装

```text
安装包默认目录:
%LOCALAPPDATA%\Programs\SMC-Copilot

安装完成:
%LOCALAPPDATA%\Programs\SMC-Copilot\desktop.exe
%LOCALAPPDATA%\Programs\SMC-Copilot\bin
%LOCALAPPDATA%\Programs\SMC-Copilot\runtime

注册表:
HKCU\Software\SMC\copilot
```

## 4.2 已安装到 `SMC Copilot` 的旧用户

实施口径：

```text
不继续把新版本默认安装到 SMC Copilot
新版本默认安装到 SMC-Copilot
旧目录作为 legacy 目录识别
用户数据不删除
Hermes Home 仍使用 %USERPROFILE%\.hermes
runtime 迁移由后续安装/修复流程完成
```

原因：如果继续读取旧 `Software\SMC\Copilot` 并把 `$INSTDIR` 设为旧目录，仍会复现目录空格问题。

## 4.3 是否删除旧目录

不在本次安装器里自动删除旧目录。

处理方式：

```text
安装完成后提示或后续提供清理功能：
- 检测 %LOCALAPPDATA%\Programs\SMC Copilot
- 检测旧桌面快捷方式
- 检测旧注册表 key
- 用户确认后删除
```

安装器不直接删除旧目录，避免误删旧 runtime、venv、logs、downloads。

# 7. 风险点

| 风险                                        | 处理方式                                             |
| ----------------------------------------- | ------------------------------------------------ |
| 旧版本已安装到 `SMC Copilot`                     | 不复用旧目录，作为 legacy 目录记录                            |
| 修改 `appId` 导致升级断裂                         | 不修改 `appId`                                      |
| 修改 `guid` 导致卸载/升级断裂                       | 不修改 `nsis.guid`                                  |
| `desktop-runtime.json` 保留旧 executableName | 安装器生成时同步改为 `desktop`                             |
| PATH 仍指向旧 bin                             | `AddToPathSafe` 写入新 `$INSTDIR\bin`，卸载旧版时清理旧 PATH |
| Electron 自动更新仍使用旧目录                       | 后续需要单独处理 autoUpdater 的迁移策略，本次先保证安装包新装与手动升级路径     |

# 8. 最小代码变更范围

```text
electron-builder.yml
build/installer.nsh
src/main/enterprise/windows/install-location-resolver.ts
```

不需要改：

```text
src/main/runtime/runtime-paths.ts
src/main/enterprise/desktop-runtime-config.ts
src/main/index.ts
renderer/*
```

核心原因：运行时路径已经通过 `resolveInstallLocation()` 集中推导，安装目录统一后，下游 Hermes / serve / portal runtime 路径会自动收敛到 `SMC-Copilot`。
