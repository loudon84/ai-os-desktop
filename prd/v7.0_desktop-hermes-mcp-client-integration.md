## 1. 版本定位

本版本只负责 copilot-desktop 客户端改造。

目标是让 copilot-desktop 能完整消费 nodeskclaw v4.3 提供的 Hermes MCP Gateway Client Contract。

Desktop 侧不重做 MCP Gateway，不重做 GeneHub 安装器，不做本地审批，只补充：

```text
1. 读取 nodeskclaw /system/info 的 mcp descriptor。
2. 调用 Hermes Client Bootstrap。
3. 展示 common-writer 等 Agent Alias。
4. 支持按 Agent/Profile 查看 remote tools。
5. 支持 readiness-check。
6. 支持从 MCP tools/call structuredContent 中识别 task_id / event_token_url / result_url。
7. 支持通过 events-token 订阅 task events。
8. 支持展示 task result / primary artifact。
9. 提供 Postman 与 Desktop 联调诊断入口。
```

---

## 2. Desktop 职责

copilot-desktop 负责：

```text
1. 用户登录 nodeskclaw。
2. Main Process 保存和注入 access token。
3. 启动 Local MCP Proxy。
4. 注册 Hermes profile MCP server URL。
5. 注入 Desktop device/profile headers。
6. 拉取 nodeskclaw MCP descriptor。
7. 拉取 client bootstrap。
8. 拉取 client agents。
9. 拉取 client tools。
10. 展示 readiness-check。
11. 展示 tools preview。
12. 解析 tools/call 返回的 structuredContent。
13. 订阅 task events。
14. 展示 task result。
15. 展示 artifact preview/download。
```

copilot-desktop 不负责：

```text
1. 不裁决最终授权。
2. 不在 Renderer 保存 token。
3. 不把 token 写入 Hermes config.yaml。
4. 不让 Renderer 直接请求 nodeskclaw MCP。
5. 不让 Renderer 直接写本地 Hermes 文件。
6. 不让用户输入任意远端 MCP URL 覆盖企业配置。
7. 不直接访问 common-writer Hermes Agent 内部端口。
8. 不绕过 nodeskclaw 调用 Skill。
```

## 3. 目标链路

```text
Desktop App 启动
  ↓

  ↓

  ↓
GET /api/v1/system/info
  ↓

  ↓

  ↓

  ↓
GET /api/v1/hermes/client/bootstrap
  ↓
GET /api/v1/hermes/client/agents
  ↓
GET /api/v1/hermes/client/tools?agent_alias=common-writer
  ↓
Hermes Agent 通过 Local Proxy 调用 MCP tools/list
  ↓
Hermes Agent 通过 Local Proxy 调用 MCP tools/call
  ↓
Desktop 读取 task_id / result_url / event_token_url
  ↓
订阅 task events
  ↓
展示 task result 和 primary artifact
```

## 11. tools/call structuredContent 处理

Local Proxy structured logs 需要识别：

```json
{
  "structuredContent": {
    "task_id": "task_uuid",
    "event_url": "...",
    "event_token_url": "...",
    "artifact_url": "...",
    "result_url": "..."
  }
}
```

处理规则：

```text
1. 如果出现 task_id，写入 recent task cache。
2. 如果出现 result_url，Task Result Panel 可直接打开。
3. 如果出现 event_token_url，优先使用 events-token。
4. 如果只有 event_url，保留旧 Bearer 模式。
```

本地缓存：

```text
appData/hermes-mcp/recent-tasks.json
```

字段：

```ts
interface RecentHermesTask {
  taskId: string;
  toolName?: string;
  agentAlias?: string;
  profileName?: string;
  status?: string;
  eventUrl?: string;
  eventTokenUrl?: string;
  resultUrl?: string;
  createdAt: string;
}
```

由 Main Process 或 Renderer fetch 带 token 方式二选一。

P0 建议：

```text

```

```ts
window.hermesMcpClient.previewArtifact(artifactId)
window.hermesMcpClient.downloadArtifact(artifactId)
```

如果沿用现有浏览器下载，需要由 Main Process 生成受控下载请求。

### 13.2 Download

要求：

```text
1. 下载 URL 来自 nodeskclaw。
2. Authorization 由 Main Process 注入。
3. Renderer 不接触 Bearer Token。
4. 下载路径由用户选择或系统下载目录，不允许服务端控制本地路径。
```
