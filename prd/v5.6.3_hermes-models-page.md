# v5.6.3 — Local Hermes Models 页

目标：在 **Local Hermes（固定 default profile）** 中复刻 Workspaces `Models.tsx` 页面（卡片网格、搜索、增删改弹窗、模型发现），数据经 `hermesDefaultApi.models` / `hermesDefaultApi.providers` → 既有 `window.hermesAPI` IPC，不新增 channel，不修改 `HermesShell.tsx`。

## 变更摘要

- 新增 `HermesDefaultModelsSurface.tsx`、`detect-provider.ts`（自 Workspaces 复制）。
- `HermesModelsPage.tsx` 改为薄壳：`visible={activeNavItem === "models"}` 时刷新列表。
- 移除旧版内联列表与 **Set active**（活跃模型由 Chat `ModelSelector` + `setModelConfig` 管理，与 Workspaces 一致）。
- UI 复用全局 `main.css` 的 `.models-*` / `.settings-container`；i18n 使用顶层 `models.*`。

## 数据层

| UI 操作 | API |
|---------|-----|
| 列表 | `hermesDefaultApi.models.list()` → `listModels` |
| 新增 | `hermesDefaultApi.models.add(...)` |
| 更新 | `hermesDefaultApi.models.update(id, fields)` |
| 删除 | `hermesDefaultApi.models.remove(id)` |
| Custom API Key | `hermesDefaultApi.providers.setEnv(key, value)`（profile `default`） |
| 模型发现 | `useDiscoveredModels({ profile: "default", ... })` |

## 手测清单

1. 打开 Local Hermes → Models：加载卡片网格，与 Workspaces Models 布局一致。
2. 搜索：按名称 / provider / model id 过滤。
3. 新增：弹窗填写 provider、model id、可选 base URL；custom provider 可填 API key。
4. 编辑：点击卡片打开弹窗，保存后列表更新。
5. 删除：卡片内确认删除。
6. 导航：切到其他侧栏再回 Models，列表刷新（含在其他页新增的模型）。
7. 隔离：无 copilot-serve / `workspaceChat` 请求。

## 与 Workspaces 差异

- 外层为 `hermes-page` + `settings-container`（无 `WorkspacesPageShell`）。
- 无列表级「Set active」按钮。
