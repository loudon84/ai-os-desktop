# PRD：ver5.7_webcontentsview 浏览器核心能力增强

## 1. 版本定义

```text
版本号：ver5.7_webcontentsview
项目：ai-os-desktop
范围：Electron WebContentsView 浏览器核心能力
目标：补齐 iframe、DOM Snapshot、元素定位、截图、动作执行、浏览器状态、审计日志能力
排除：Hermes Agent 工具调用、Profile Runtime、Skill、Gateway、Agent Tool Server
```

现有桌面架构应继续保持 Renderer / Preload / Main Process 分层：Renderer 无 Node.js 访问能力，Preload 是 Renderer 与 Main Process 的唯一安全桥接，Main Process 统一注册 IPC 并管理特权能力。
Web Operator 继续基于 Electron WebContentsView 实现，不替换为 iframe、webview 或 Tauri/Wails。Electron 对 WebContentsView、webContents、DevTools、session、partition 的控制能力适合当前浏览器自动化核心能力建设。

---

## 2. 背景

当前 `WebOperatorScreen.tsx` 已经形成浏览器操作台页面结构：

```text
src/renderer/src/screens/WebOperator/WebOperatorScreen.tsx
├─ BrowserToolbar
├─ WebContentsHost
├─ BrowserStatePanel
├─ ScreenshotPanel
└─ BrowserActionLog
```

当前 `WebContentsHost.tsx` 的核心职责是把 React DOM 占位区域与 Main Process 管理的 `WebContentsView` 对齐：

```text
src/renderer/src/components/shell/WebContentsHost.tsx
├─ 计算 host DOM bounds
├─ 调用 shellView.setBounds(layerId, bounds)
├─ 页面隐藏时调用 shellView.hide(layerId)
└─ 通过 ResizeObserver / IntersectionObserver 同步位置
```

当前问题：

```text
1. WebContentsHost 只负责显示区域定位，不具备页面结构解析能力。
2. 当前浏览器能力主要面向 main frame，遇到 iframe 页面时元素定位不稳定。
3. 缺少统一 DOM Snapshot / Frame Tree / Interactive Elements 数据结构。
4. 缺少 iframe 内元素坐标换算能力。
5. 缺少浏览器动作的结构化执行结果。
6. 缺少浏览器核心操作日志与回放数据。
7. Renderer 面板只能展示基础状态，不能展示 Frame / DOM / Element 结构。
```

## 3. 目标

### 3.1 产品目标

```text
G-001：Web Operator 能稳定承载外部网页。
G-002：能识别网页 main frame 与 iframe frame tree。
G-003：能采集页面 DOM Snapshot 与可交互元素列表。
G-004：能定位 iframe 内按钮、输入框、链接、表单元素。
G-005：能执行浏览器核心动作：open、reload、back、forward、click、type、select、scroll、screenshot。
G-006：能输出动作前后状态、截图、错误、耗时、frameId、selector。
G-007：Renderer 页面能展示浏览器状态、Frame Tree、元素列表、截图、动作日志。
G-008：不引入 Hermes Agent 工具调用逻辑。
```

### 3.2 工程目标

```text
E-001：WebContentsHost 继续只负责 WebContentsView 的显示定位。
E-002：iframe / DOM / 元素定位逻辑全部放在 Main Process。
E-003：Renderer 只能通过 Preload IPC 调用浏览器能力。
E-004：所有 IPC 参数和返回值必须定义 TypeScript DTO。
E-005：禁止 Renderer 直接访问 Node.js、Electron webContents、SQLite、本地文件。
E-006：TypeScript strict 模式下无 any 扩散。
E-007：保留现有页面结构，不大改 Workspaces / Layout / MainPage。
```

## 6. 模块边界

### 6.1 WebContentsHost.tsx

职责保留：

```text
1. 提供 React DOM host。
2. 计算 DOM bounds。
3. 调用 shellView.setBounds。
4. 调用 shellView.hide。
5. 监听 resize、visibility、intersection。
```

禁止增加：

```text
1. 禁止在 WebContentsHost 内解析 DOM。
2. 禁止在 WebContentsHost 内处理 iframe。
3. 禁止在 WebContentsHost 内执行 click/type。
4. 禁止在 WebContentsHost 内接入业务工具调用。
5. 禁止在 WebContentsHost 内保存浏览器动作日志。
```

职责：

```text
1. 页面组合。
2. 管理右侧面板布局。
3. 调用 hooks 读取浏览器状态。
4. 触发人工浏览器动作。
```

禁止增加：

```text
1. 禁止直接访问 Electron webContents。
2. 禁止直接执行 DOM 脚本。
3. 禁止绕过 preload IPC。
```

职责：

```text
1. 获取 active WebContentsView。
2. 执行导航与页面动作。
3. 获取 frame tree。
4. 在指定 frame 内执行安全脚本。
5. 采集 DOM Snapshot。
6. 计算 iframe 元素绝对坐标。
7. 执行点击、输入、滚动、选择。
8. 记录动作日志。
```

## 9. 功能需求

## FR-001：WebContentsView 生命周期保持稳定

### 需求

```text
1. WebContentsHost 继续按 layerId 管理 WebContentsView 显示区域。
2. bounds 变化时同步 Main Process。
3. host 不可见时隐藏 WebContentsView。
4. WebOperator 页面切换回来时恢复 WebContentsView。
5. 不因右侧 panel 展开、窗口 resize、页面滚动导致 WebContentsView 错位。
```

### 验收

```text
1. 页面 resize 后 WebContentsView 与 host 完全重合。
2. 切换到其它页面后 WebContentsView 不残留在界面上。
3. 切回 WebOperator 后当前页面不丢失。
4. 右侧 panel 滚动不影响 WebContentsView 坐标。
```

---

## FR-002：浏览器基础导航

### 需求

```text
1. 支持 open(url)。
2. 支持 reload()。
3. 支持 back()。
4. 支持 forward()。
5. 支持 loading 状态。
6. 支持 title、url、canGoBack、canGoForward。
```

```ts
export interface BrowserState {
  url: string;
  title: string;
  loading: boolean;
  canGoBack: boolean;

```

### 验收

```text
1. 输入 URL 后页面正常加载。
2. loading 状态正确变化。
3. 页面 title 与 URL 正确显示。
4. back / forward 按钮状态正确。
```

## FR-003：Frame Tree 采集

### 需求

```text
1. 支持 listFrames()。
2. 返回 main frame 与所有 child frames。
3. 每个 frame 返回 frameId、url、origin、name、depth、path。
4. 支持多层 iframe。
5. 支持跨域 iframe 的元信息采集。
```

### 验收

```text
1. 无 iframe 页面返回 1 个 main frame。
2. 单 iframe 页面返回 main frame + child frame。
3. 多层 iframe 页面返回完整 depth / path。
4. 跨域 iframe 至少返回 url / origin / depth / path。
```

---

## FR-004：DOM Snapshot 采集

### 需求

```text
1. 支持 snapshot({ includeFrames, includeInteractiveElements })。
2. 默认采集 main frame 与所有可执行 frame。
3. 每个 frame 内采集可交互元素。
4. 可交互元素包括：
   - a
   - button
   - input
   - textarea
   - select
   - [role]
   - [aria-label]
   - [contenteditable="true"]
5. 每个元素输出 selector、role、text、ariaLabel、placeholder、rect、visible、enabled。
6. 跨域 iframe 通过 frame 级执行采集，不从父页面直接访问 iframe DOM。
7. 不可执行 frame 返回 snapshot error，不中断整个页面 snapshot。
```

### 验收

```text
1. 普通页面能返回按钮、链接、输入框。
2. iframe 内按钮能出现在 elements 中。
3. iframe 内输入框能出现在 elements 中。
4. 不可访问 frame 记录 errors。
5. snapshot 不因单个 frame 错误整体失败。
```

---

## FR-005：元素定位

### 需求

```text
1. 支持按 selector 定位。
2. 支持按 text 定位。
3. 支持按 role 定位。
4. 支持按 ariaLabel 定位。
5. 支持按 placeholder 定位。
6. 支持按 hrefIncludes 定位。
7. 支持限定 frame。
8. 未限定 frame 时，从 main frame 到 child frames 顺序查找。
9. 多个候选元素时返回优先级最高的可见可用元素。
```

### 优先级

```text
1. elementId
2. selector + frame
3. selector
4. role + text
5. ariaLabel
6. placeholder
7. text
8. hrefIncludes
```

### 验收

```text
1. 可通过 selector 找到 main frame 元素。
2. 可通过 text 找到 iframe 内按钮。
3. 可通过 placeholder 找到 iframe 内输入框。
4. 不存在元素时返回 null，不抛未处理异常。
```

---

## FR-006：iframe 坐标换算

### 需求

```text
1. 元素 rect 必须包含 frame 内坐标 rect。
2. 元素 rect 必须包含主页面坐标 rectInMainFrame。
3. 多层 iframe 时递归累加 iframe 偏移。
4. 点击动作必须使用 rectInMainFrame 的中心点。
5. 滚动后重新计算坐标，不复用旧坐标。
```

### 验收

```text
1. 单层 iframe 内按钮点击准确。
2. 多层 iframe 内按钮点击准确。
3. 页面滚动后 iframe 内元素仍可点击。
4. 右侧 panel 布局变化后点击位置不偏移。
```

---

## FR-007：点击动作

### 需求

```text
1. 支持 clickElement(target)。
2. 优先执行 frame 内 DOM click。
3. DOM click 失败时使用 WebContents 坐标点击。
4. 点击前校验元素 visible / enabled。
5. 点击后返回 BrowserActionResult。
6. 点击后刷新 BrowserState。
7. 点击后可选生成 screenshot。
```

### 执行策略

```text
1. resolve target
2. resolve frame
3. collect fresh rect
4. attempt DOM click
5. fallback coordinate click
6. wait short idle
7. update state
8. write action log
```

### 验收

```text
1. main frame button 可点击。
2. iframe button 可点击。
3. disabled button 不执行点击并返回 ELEMENT_DISABLED。
4. 被遮挡元素返回 ELEMENT_NOT_CLICKABLE 或 fallback 失败原因。
```
