# PRD：v6.3.1 WebOperator 任务 Chat 持久化 hotfix

## 版本

- **版本号**：`v6.3.1_hotfix`
- **依赖**：v6.3（`HermesTaskStartDialog` / `HermesPanelSkill` / `HermesPanelSession`）

## 问题

1. 切换 WebOperator 侧栏（尤其 Host Context）时，弹 Dialog 路径曾 `setCurrentTask(null)`，导致 Chat 像新开一轮。
2. `currentTask` 在 `HermesTaskPanel` 本地 state，侧栏折叠卸载 `WebOperatorPanels` 后任务丢失。
3. 无 `analysisRequest` 时不会从 SQLite 恢复最后一次任务会话。

## 方案

| 项 | 实现 |
|----|------|
| A | 打开 `HermesTaskStartDialog` 时**不**清空 `currentTask`；仅 Dialog 确认后覆盖 |
| B | `web-operator-task-session:get-last-active` + Provider mount hydrate |
| Context | `currentTask` / `setCurrentTask` / `taskSessionUpsertedIdRef` 在 `WebOperatorPageContext` |
| 轻量缓存 | `sessionStorage` 键 `weboperator-current-task-v1` |
| 恢复优先级 | **SQLite `getLastActive` 优先**；无 record 再读 sessionStorage |
| 守卫 | `requestHermesAnalysis` 与 `HostBridgePanel` 对同 `taskId` 跳过重复分析（非 `force`） |

## 验收

1. 有任务 Chat 时切 `host-context` 再回 `hermes-task`：消息仍在。
2. 折叠侧栏再展开：`currentTask` 与历史消息恢复。
3. 退出 WebOperator 再进入：加载 SQLite 最后一次 active 任务（`getLastActive`）。
4. HostBridge 自动事件：同 `pageUrl` 活跃任务时不弹 Dialog、不清 Chat。
5. Dialog 确认「新建会话」：覆盖任务并 autoRun。
6. `pnpm run typecheck` 通过。
