## 2. 背景

v6.7 已实现 GeneHub MCP Registration 主链路：

```text
Hermes Agent
  -> nodeskclaw MCP Gateway
  -> genehub.skill.register_to_hermes
  -> nodeskclaw 创建 install job
  -> copilot-desktop 拉取 pending jobs
  -> 用户确认
  -> 下载 bundle
  -> 校验
  -> 写入 Hermes profile skills
  -> restart / reload Hermes
  -> sync installed skills
  -> 回写 job status
  -> Agent 查询 registration.status
```

当前需要在 v6.7.1 修复以下问题：

```text
1. Desktop 对 MCP job 的识别依赖 source=mcp_agent_request，但 pending jobs 返回字段不完整时无法稳定显示。
2. bundle preview 当前可能依赖 bundle download，pending 状态下不应 claim job。
3. syncInstalledSkills 使用本地 profileId，可能与 nodeskclaw server profile id 不一致。
4. ignoreInstallJob 不能只写本地状态，应同步服务端 cancelled / ignored。
5. 安装完成后 pending cache 与 UI 状态刷新不完整。
6. verifySignature 当前只做到阻断占位，需要补基础可验收校验路径。
7. scripts 写入需要增加 provenance 保护，避免覆盖非 GeneHub 管理脚本。
```

## 3. 产品目标

### 3.1 核心目标

```text
1. Desktop 能稳定识别 source=mcp_agent_request 的 install job。
2. MCP 发起的 install job 默认进入“待确认”，不自动安装。
3. 用户确认前可查看 bundle preview，但 preview 不 claim job、不改变 job 状态。
4. 用户确认后完整执行 claim、download、validate、write、restart、sync、status callback。
5. installed skills sync 使用 nodeskclaw server profile id。
6. ignore / cancel 状态同步 nodeskclaw。
7. 安装成功、失败、忽略后的 UI 状态与本地 cache 同步。
8. verifySignature=true 时能对 signature 缺失或非法状态做阻断。
9. scripts 写入必须检查 GeneHub provenance。
```

### 3.2 用户价值

```text
1. IT 管理人员可在 Desktop 侧明确处理 Agent 发起的安装请求。
2. Agent 不获得本地写入权限。
3. Desktop 不因字段缺失漏显示待处理任务。
4. 安装前可预览影响范围。
5. 安装后状态能被 Agent 正确查询。
6. 安装失败能定位具体原因。
```

## 6. 功能需求

## 6.1 Pending Job 完整字段接入

Desktop 需要兼容 nodeskclaw v6.7.1 的 pending job schema。

字段要求：

```ts
interface GeneHubInstallJob {
  jobId: string;
  source: "mcp_agent_request" | "desktop_manual" | "server_assigned";
  profileId: string;        // server profile id
  profileName: string;      // local profile name
  geneSlug: string;
  geneVersion: string;
  skillName: string;
  action: "install" | "update";
  status:
    | "pending"
    | "claimed"
    | "downloading"
    | "validating"
    | "installing"
    | "installed"
    | "failed"
    | "cancelled";
  createdAt?: string;
  assignedAt?: string;
  claimedAt?: string;
  lastUpdatedAt?: string;
  errorCode?: string | null;
  errorMessage?: string | null;
}
```

处理规则：

```text
1. source 缺失时不得默认识别为 mcp_agent_request。
2. source=mcp_agent_request 且 status=pending 时进入待确认列表。
3. source=server_assigned 且 autoInstallAssignedJobs=true 时才允许自动安装。
4. source=desktop_manual 继续按用户手动安装链路处理。
5. profileName 为空时，尝试从本地 profile mapping 解析；解析失败显示 unknown profile 并禁止安装。
```

## 6.7 基础签名校验增强

v6.7.1 做基础可验收签名校验，不做完整 CA 体系。

配置项：

```ts
verifySignature: boolean; // default true
trustedPublicKeys: string[];
```

规则：

```text
1. verifySignature=true 且 manifest.signature 为空，阻断安装。
2. verifySignature=true 且 trustedPublicKeys 为空，阻断安装。
3. signature 覆盖 manifestHash + bundleHash。
4. signature 校验失败，阻断安装并回传 GENEHUB_SIGNATURE_INVALID。
5. verifySignature=false 时允许跳过，但必须写 warning log。
```

验收标准：

```text
1. signature 缺失时安装失败。
2. signature 无效时安装失败。
3. signature 有效时可继续安装。
4. install log 不记录私钥或 token。
```

## 6.8 Scripts Provenance 保护

写入 scripts 前检查：

```text
1. scripts relativePath 必须在受控 scripts 目录内。
2. 不允许绝对路径。
3. 不允许 ../ 路径穿越。
4. 如果目标脚本已存在：
   - 存在 genehub provenance 且 geneSlug/jobId 匹配，可覆盖。
   - 存在 genehub provenance 但 geneSlug 不匹配，阻断。
   - 无 provenance，阻断。
5. 覆盖前备份旧文件。
```

新增错误码：

```text
GENEHUB_SCRIPT_PROVENANCE_MISMATCH
GENEHUB_SCRIPT_OVERWRITE_BLOCKED
```
