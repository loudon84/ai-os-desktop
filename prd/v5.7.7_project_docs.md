# v5.7.6_renderer_docs_split 文档拆解实施方案

## 1. 结论

可以拆，而且应该拆。

当前 `docs/INDEX.md` 已经承担项目总入口职责，包含项目定位、版本演进、核心目录、开发前必须阅读、技术栈、禁止行为等内容。Renderer 目前只在核心目录中以一行概括为“主要开发区”，粒度不足，无法支撑 Cursor 按 `src/renderer/src` 功能模块快速建立上下文。 

当前 `docs/READING_GUIDE.md` 已经有 UI 层阅读顺序和 Web Operator 阅读顺序，但仍是阅读路径，不是模块文档。 

当前 Renderer 实际入口链路是：

```text
App.tsx
  → useStartupGate
  → Layout
  → MainPage
  → WorkspaceOutlet
  → WorkspaceRenderer
  → screens/*
```

`App.tsx` 负责 `splash/login/welcome/installing/setup/main` 启动路由。
`Layout.tsx` 负责主界面状态、水合、workspace 二级面板、WebOperator layout 状态和 `WorkspaceOutlet` 注入。 
`WorkspaceRenderer.tsx` 负责把 workspace id 映射到 `Portal / WebOperator / Workspaces / TaskWorkbench / Hermes / CRM / Office` 等 Screen。 

因此文档应新增 `docs/renderer/` 目录，并按 `src/renderer/src` 的功能边界拆解；`docs/INDEX.md` 只保留总入口，不承载 Renderer 细节。

## 5. `docs/renderer/INDEX.md` 内容骨架

# Renderer Process 文档索引

## 1. 定位

`src/renderer/src` 是 SMC Copilot 的 React UI 层，负责：

- 启动路由
- 主界面布局
- Workspace 渲染
- Screen 页面
- UI 状态

| `constants.ts` | Renderer 常量 |

  → MainPage
  → WorkspaceOutlet

## 8. WebOperator 文档必须单独加强

当前 WebOperator 已经是高复杂度模块：`WebContentsHost`、`WebOperatorPanels`、`WebOperatorSideRail`、`PageStructurePanel`、CRM Bridge、Hermes Task Panel、task session DB 都在该链路内。`docs/ARCHITECTURE.md` 已经记录 V5.7.5 的 Page Structure `[分析内容]` 到 Hermes Task 的任务流。:contentReference[oaicite:12]{index=12}

新增：

```text
docs/renderer/screens/WebOperator.md
docs/renderer/screens/web-operator/
  PANELS.md
  PAGE_CONTEXT.md
  HERMES_TASK_FLOW.md
  CRM_BRIDGE_UI.md
  FRAME_HTML_INSPECTOR.md
````

`WebOperator.md` 应覆盖：

```text
1. WebOperatorScreen 作为 composite workspace
2. WebContentsHost 与 WEB_OPERATOR_LAYER_ID
3. WebOperatorPanels 面板体系
4. SideRail 面板切换
5. PageStructurePanel / Get HTML / 分析内容
6. HermesTaskPanel 与 components/hermes
7. WebOperatorTaskStartDialogHost 遮罩策略
8. WebOperatorPageContext
9. 与 Main browser IPC 的边界
```

当前 `WebOperatorScreen.tsx` 已经接入 `WebContentsHost`、`WebOperatorSideRail`、`WebOperatorPanels`、`WebOperatorTaskStartDialogHost`，并在任务对话框打开时关闭 WebContentsHost 输入层，文档需要明确这个遮挡处理规则。

当前 `WebOperatorPanels.tsx` 已经把 5 个面板组件集中到 `panels` 目录内统一渲染，包括 `BrowserStatePanel / CrmEventPanel / HermesTaskPanel / PageStructurePanel / BrowserActionLog`。
