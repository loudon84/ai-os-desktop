# PRD：copilot-desktop GeneHub Hermes Skill Sync

版本号：v6.5_genehub-hermes-skill-sync
适用仓库：copilot-desktop
实施对象：Cursor / Coding Agent
依赖服务端版本：nodeskclaw team_v3.4_genehub-hermes-skill-registry
优先级：P0
核心结论：Desktop 是本地 Hermes Skill 安装执行器，只安装、更新、卸载、同步状态，不允许上传、发布、审核。

---

## 1. 背景

企业 GeneHub Registry 由 nodeskclaw 服务端提供。管理员在 nodeskclaw 后台上传、审核、发布和分配 Skill。Desktop 端只负责根据当前登录用户权限，拉取可安装 Skill 或后台分配的安装任务，并将 Skill 安装到本机 Hermes-agent Profile。

Desktop 不拥有企业技能库写权限，不允许提供上传、发布、审核、编辑企业 Registry 的入口。

---

## 2. 产品定位

### 2.1 Desktop 角色

Desktop 是本地安装执行器：

```text
GeneHub Registry → nodeskclaw 授权 / 分配 → Desktop 拉取 → 本机 Hermes-agent 安装 → 状态回传
```

Desktop 负责：

1. 用户登录。
2. 注册本机设备。
3. 识别 Hermes Profile。
4. 注册 Hermes Profile。
5. 查询有权限 Skill。
6. 拉取安装任务。
7. 下载 Bundle。
8. 校验 Bundle。
9. 写入本机 Hermes Skill 目录。
10. 写入本机 Hermes Scripts 目录。
11. 触发 Hermes-agent reload / restart。
12. 回传安装、更新、卸载、失败状态。
13. 展示安装日志。

Desktop 禁止：

1. 上传 Skill。
2. 发布 Skill。
3. 审核 Skill。
4. 编辑企业 GeneHub Registry。
5. 绕过 nodeskclaw 安装未授权 Bundle。
6. 从任意外部 URL 安装 Skill。
7. Renderer 进程直接写本地 Hermes 文件。

---

## 3. 目标

### 3.1 用户目标

1. 员工打开 Desktop 后，可以看到自己有权限使用的 Hermes Skill。
2. 员工可以点击安装 Skill。
3. 员工可以看到管理员分配给自己的待安装 Skill。
4. 员工可以更新或卸载有权限的 Skill。
5. 安装失败时可以看到明确错误原因。
6. 安装成功后 Hermes-agent 能识别新 Skill。

### 3.2 技术目标

1. Main Process 负责所有本地文件写入。
2. Renderer 只负责展示和触发 IPC。
3. 所有 Bundle 从 nodeskclaw 拉取。
4. 所有 Bundle 安装前必须校验。
5. Skill 写入必须防路径穿越。
6. 安装过程支持回滚。
7. 安装状态必须回传 nodeskclaw。
8. 支持 Windows / macOS / Linux 路径。
9. 支持多个 Hermes Profile。
10. 不破坏现有 Hermes 模块。

## 5. 用户流程

### 5.1 Desktop 启动流程

```text
Desktop 启动
  ↓
读取当前登录用户 token
  ↓
生成 / 读取 device_fingerprint
  ↓
POST /desktop/devices/register
  ↓
扫描本机 Hermes profiles
  ↓
POST /desktop/hermes/profiles/register
  ↓
POST /desktop/heartbeat
  ↓
GET /desktop/genehub/skills
  ↓
GET /desktop/hermes/install-jobs/pending
  ↓
展示 Skill Center
```

---

### 5.2 用户自助安装流程

```text
用户打开 Hermes Skill Center
  ↓
选择“可安装”
  ↓
点击安装
  ↓
POST /desktop/hermes/install-jobs
  ↓
POST /install-jobs/{job_id}/claim
  ↓
GET /install-jobs/{job_id}/bundle
  ↓
校验 Bundle
  ↓
写入本地 Hermes skill
  ↓
清理 snapshot
  ↓
restart / reload Hermes-agent
  ↓
POST /install-jobs/{job_id}/status installed
  ↓
刷新已安装列表
```

---

### 5.3 后台分配安装流程

```text
管理员在 nodeskclaw 分配 Skill 给用户
  ↓
Desktop 定时拉取 pending jobs
  ↓
发现 pending install job
  ↓
展示“待安装任务”
  ↓
用户点击安装 / 或按配置自动安装
  ↓
claim job
  ↓
下载 bundle
  ↓
本地安装
  ↓
回传状态
```

本版本默认策略：后台分配任务需要用户点击安装，不自动静默安装。

配置项允许后续扩展：

```text
genehub.autoInstallAssignedJobs = false
```

---

### 5.4 更新流程

```text
Desktop 查询可安装 Skill
  ↓
发现服务端 version > 本地 version
  ↓
展示 update_available
  ↓
用户点击更新
  ↓
创建 update job
  ↓
下载新 bundle
  ↓
备份旧 skill
  ↓
安装新 skill
  ↓
restart / reload Hermes-agent
  ↓
回传 installed
```

---

### 5.5 卸载流程

```text
用户点击卸载
  ↓
校验 uninstall 权限
  ↓
创建 uninstall job
  ↓
备份当前 skill
  ↓
删除 ~/.hermes/skills/{skill_name}
  ↓
可选删除 scripts 中由该 skill 管理的文件
  ↓
清理 snapshot
  ↓
restart / reload Hermes-agent
  ↓
回传 uninstalled
```

## 6. 本地目录规则

### 6.1 Hermes Home

默认：

```text
Windows: %USERPROFILE%\.hermes
macOS:   ~/.hermes
Linux:   ~/.hermes
```

但应优先读取当前 Desktop Hermes 模块已有 profile 配置。

Profile 示例：

```json
{
  "profileName": "default",
  "hermesHome": "C:\\Users\\loudon\\.hermes",
  "gatewayUrl": "http://127.0.0.1:8642"
}
```

---

### 6.2 Skill 安装目录

```text
{hermesHome}/skills/{skill_name}/SKILL.md
```

例如：

```text
C:\Users\loudon\.hermes\skills\contact-to-order\SKILL.md
```

---

### 6.3 Scripts 安装目录

```text
{hermesHome}/scripts/*
```

例如：

```text
C:\Users\loudon\.hermes\scripts\parse_order.py
```

---

### 6.4 Snapshot 清理

安装、更新、卸载后必须删除：

```text
{hermesHome}/.skills_prompt_snapshot.json
```

如果文件不存在，不报错。

---

### 6.5 安装记录

Desktop 本地保存安装记录：

```text
{hermesHome}/genehub/installed/{gene_slug}.json
```

示例：

```json
{
  "gene_slug": "contact-to-order",
  "gene_version": "1.0.0",
  "skill_name": "contact-to-order",
  "installed_at": "2026-06-08T00:00:00Z",
  "source": "nodeskclaw-genehub",
  "job_id": "job_xxx"
}
```

## 7. Main Process 模块设计

新增目录：

```text
src/main/genehub/
  genehubClient.ts
  deviceIdentity.ts
  hermesProfileResolver.ts
  skillInstallWorker.ts
  skillPackageValidator.ts
  hermesSkillWriter.ts
  hermesRestartService.ts
  installedSkillStore.ts
  genehubScheduler.ts
  genehubTypes.ts
```

如果当前项目已有 Hermes 模块，应放到现有 Hermes main process 目录下，并保持模块边界一致。

---

### 7.1 genehubClient.ts

职责：

1. 调用 nodeskclaw Desktop API。
2. 附带用户 token。
3. 处理错误码。
4. 不持久化明文敏感 token。
5. 提供 typed client 方法。

方法：

```ts
registerDevice(payload): Promise<RegisterDeviceResult>
registerHermesProfile(payload): Promise<RegisterProfileResult>
heartbeat(payload): Promise<HeartbeatResult>
listAuthorizedSkills(profileId): Promise<GeneHubSkill[]>
createInstallJob(payload): Promise<InstallJob>
listPendingJobs(profileId): Promise<InstallJob[]>
claimJob(jobId): Promise<InstallJob>
downloadBundle(jobId): Promise<GeneHubBundle>
updateJobStatus(jobId, payload): Promise<void>
syncInstalledSkills(payload): Promise<void>
```

---

### 7.2 deviceIdentity.ts

职责：

1. 生成 device fingerprint。
2. 跨重启保持稳定。
3. 不上传敏感硬件明文。
4. 使用 OS user + app id + machine id 生成 hash。
5. 如 machine id 不可用，则生成 UUID 并保存到 app config。

返回：

```ts
{
  deviceName: string
  deviceFingerprint: string
  osType: 'windows' | 'macos' | 'linux'
  osVersion: string
  appVersion: string
}
```

---

### 7.3 hermesProfileResolver.ts

职责：

1. 读取当前 Desktop 已配置 Hermes Profile。
2. 推断 hermesHome。
3. 推断 gatewayUrl。
4. 检测 Hermes-agent 版本。
5. 返回 profiles。

返回：

```ts
type HermesProfile = {
  profileName: string
  hermesHome: string
  gatewayUrl?: string
  gatewayPort?: number
  runtimeVersion?: string
  capabilities: {
    skills: boolean
    scripts: boolean
    reload: boolean
  }
}
```

---

### 7.4 skillPackageValidator.ts

职责：

1. 校验 bundle schema。
2. 校验 manifest schema。
3. 校验 signature。
4. 校验 manifest hash。
5. 校验 bundle hash。
6. 校验 compatibility。
7. 校验 file path。
8. 校验 skill_name。
9. 禁止绝对路径。
10. 禁止路径穿越。

必须校验：

```regex
^[A-Za-z0-9][A-Za-z0-9._-]{0,127}$
```

非法路径示例：

```text
../x
skills/../../x
C:\Windows\system32\x
/root/.ssh/id_rsa
\\server\share\x
```

---

### 7.5 hermesSkillWriter.ts

职责：

1. 写入 Skill。
2. 写入 scripts。
3. 创建临时目录。
4. 备份旧版本。
5. 原子替换。
6. 失败回滚。
7. 清理 snapshot。
8. 写本地 installed metadata。

安装步骤：

```text
1. resolve hermesHome
2. validate skillName
3. create temp dir: {hermesHome}/genehub/tmp/{job_id}
4. write temp skill files
5. backup existing skill: {hermesHome}/genehub/backup/{skill_name}/{timestamp}
6. rename temp skill to {hermesHome}/skills/{skill_name}
7. write scripts
8. remove .skills_prompt_snapshot.json
9. write installed metadata
```

必须保证：

1. Renderer 不能直接调用 fs。
2. 所有 path.resolve 后必须在 hermesHome 内。
3. Windows 路径兼容。
4. 文件写入失败要回滚。

---

### 7.6 hermesRestartService.ts

职责：

1. 调用现有 Desktop Hermes 模块 restart / reload。
2. 如 Hermes-agent 支持 reload skill，则优先 reload。
3. 不支持 reload 时 restart。
4. restart 失败时返回错误。
5. restart 后检测 gateway health。

方法：

```ts
reloadOrRestart(profile): Promise<{
  ok: boolean
  mode: 'reload' | 'restart'
  error?: string
}>
```

优先级：

```text
1. gateway reload skills API
2. Desktop Hermes manager restart profile
3. 提示用户手动重启
```

如果项目当前没有 reload API，则实现 restart。

---

### 7.7 skillInstallWorker.ts

职责：

1. 接收 install / update / uninstall / rollback job。
2. 调用 genehubClient 下载 Bundle。
3. 调用 validator 校验。
4. 调用 writer 写入。
5. 调用 restartService 重启。
6. 回传状态。
7. 写安装日志。

状态流转：

```text
claimed
  → downloading
  → validating
  → installing
  → installed

任意阶段失败 → failed
```

每个阶段都调用：

```ts
updateJobStatus(jobId, { status, client_report })
```

---

### 7.8 genehubScheduler.ts

职责：

1. Desktop 启动后注册 device。
2. 注册 profiles。
3. 周期 heartbeat。
4. 周期拉取 pending jobs。
5. 自动刷新 Skill Center 数据。
6. 不自动静默安装，除非配置开启。

默认配置：

```ts
const DEFAULT_GENEHUB_SYNC = {
  heartbeatIntervalMs: 60_000,
  pendingJobsIntervalMs: 60_000,
  autoInstallAssignedJobs: false
}
```

## 12. 安全要求

1. Renderer 不直接写文件。
2. Renderer 不接收任意 Bundle URL。
3. Bundle 只能来自 nodeskclaw API。
4. Bundle 安装前必须校验。
5. 路径必须限制在 hermesHome 内。
6. 禁止路径穿越。
7. 禁止绝对路径。
8. 禁止隐藏 skill name。
9. token 不写入普通日志。
10. 安装日志不记录完整敏感 token。
11. Desktop 不提供上传按钮。
12. Desktop 不提供发布按钮。
13. Desktop 不提供审核按钮。
14. Desktop 不提供编辑企业 Skill Manifest 按钮。
15. 本地写入失败必须回滚。

## 16. 验收标准

### 16.1 功能验收

1. Desktop 启动后成功注册 device。
2. Desktop 成功注册 Hermes Profile。
3. Desktop 能显示用户有权限的 Skill。
4. Desktop 能显示后台分配的 pending jobs。
5. 用户可以点击安装 Skill。
6. 安装后本地出现 `SKILL.md`。
7. 安装后本地 scripts 写入成功。
8. 安装后 snapshot 被清理。
9. 安装后 Hermes-agent 被 reload 或 restart。
10. 安装后状态回传服务端。
11. 更新可以覆盖旧版本并保留备份。
12. 卸载可以删除 Skill 并回传状态。
13. 安装失败时显示错误。
14. 安装失败时服务端能看到 failed。
15. 安装日志可查看。

1. Desktop 无上传按钮。
2. Desktop 无发布按钮。
3. Desktop 无审核按钮。
4. Renderer 不能直接写 Skill 文件。
5. Bundle 只能从 nodeskclaw 下载。
6. 非法路径被拒绝。
7. 非法 skill name 被拒绝。
8. hash mismatch 被拒绝。
9. signature invalid 被拒绝。
10. token 不进入安装日志。

### 16.3 兼容验收

1. Windows 正常安装。
2. macOS 路径逻辑正常。
3. Linux 路径逻辑正常。
4. default Hermes Profile 正常。
5. 多 Profile 可区分。
6. 现有 Hermes Chat / WebUI 功能不受影响。
7. 现有本地 Hermes 启停功能不受影响。
