# v5.6.2 — Local Hermes WebChat Surface

目标：在 **Local Hermes（固定 default profile）** 中复刻 Workspaces 的 `HermesWebChatSurface` 体验（模型选择器 + 附件上传/拖拽 + 富文本滚动区 + ComposerBar），数据源全部走 `hermesDefaultApi` / `window.hermesDefaultChat`（Main 新 IPC），不引入 `workspaceChat` / copilot-serve，也不修改 `HermesShell.tsx`。

## 变更摘要

- 新增 `hermes-chat:*` IPC：模型列表/配置、附件上传与删除、带附件与 model override 的 sendMessage。
- Hermes Chat 页面替换为 WebChat Surface：支持模型选择与保存默认、附件托盘、拖拽上传、Stop/Send、More actions（新对话/清空/跳转 Sessions）、滚动区显示 streaming/tool/usage。

## 手测清单

1. **模型选择器**
   - 打开模型列表，能看到按 provider 分组的 models.json 条目。
   - 选择模型后点击「保存为默认」，右侧 Runtime 的 model config 与 `config.yaml` 生效（下次发消息使用新模型）。
2. **附件**
   - 点击附件菜单选择文件上传；托盘展示附件名与大小；可删除。
   - 拖拽文件到输入区可上传；托盘同步更新。
3. **发送**
   - 纯文本 / 仅附件 / 文本+附件均可发送。
   - 流式输出可见；工具进度提示可见；usage（tokens）可见。
4. **新对话**
   - 「新对话」：清空消息列表与附件；回到 draft session（activeSessionId 置空）。
   - 「清空」：只清空当前消息与附件，不修改 activeSessionId。
5. **隔离**
   - Hermes 目录无 `workspaceChat` 引用；DevTools Network 中无 copilot-serve 请求。

## 已知限制

- 附件的 Gateway 多模态：图片会以 `image_url`（base64 data URL）注入；非图片以文本 preview（截断）注入，属于降级路径。
