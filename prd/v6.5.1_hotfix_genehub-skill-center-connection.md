# PRD 1：copilot-desktop GeneHub Skill Center 连接与协议修复

版本号：v6.5.1_genehub-skill-center-connection-fix
适用仓库：loudon84/ai-os-desktop
实施对象：Cursor
关联后端版本：nodeskclaw team_v3.4.1_genehub-descriptor-api-alignment
优先级：P0
目标：修复 Hermes Skill Center 当前连接报错，并完成 Desktop 与 nodeskclaw GeneHub API 协议对齐。

---

## 1. 问题背景

当前 Desktop 的 Hermes Skill Center 页面打开后出现：

```text
Cannot read properties of undefined (reading 'probeConnection')
```

终端同时出现：

```text
[GENEHUB] auto-init skipped: system/info did not return genehub block
```

这说明当前存在两个独立问题：

1. Renderer 调用 `window.genehubRuntime.probeConnection()`，但 preload 在 contextIsolation 模式下没有暴露 `genehubRuntime`。
2. Desktop 自动初始化时访问 nodeskclaw `/api/v1/system/info`，但后端没有返回 `genehub` descriptor。

即使这两个问题修复，后续仍会遇到 profile 注册、heartbeat、install job、skill list、bundle 解析等协议不一致问题。因此本 PRD 要一次性完成 Desktop 端协议修正。

4. 点击 `Initialize` 能完成 device/profile 注册。
5. Desktop 能拉取授权 Skill 列表。
6. Desktop 能拉取 pending jobs。
7. Desktop 能创建 install/update/uninstall job。
8. Desktop 能下载 bundle 并执行安装。
9. Desktop 不增加上传、发布、审核入口。

### 2.2 技术目标

1. 修复 preload API 暴露。
2. Desktop 与 nodeskclaw `system/info.genehub` descriptor 对齐。
3. Desktop 与 nodeskclaw Desktop API schema 对齐。
4. 修复 profile 注册字段。
5. 修复 heartbeat 字段。
6. 修复 install job 创建字段。
7. 修复 skill list 字段映射。
8. 修复 bundle 格式解析。
9. 修复非法 `claimed` 状态回传。
10. 增加必要的兜底错误提示，避免页面直接报 JS runtime error。

### 问题

`src/preload/genehub-runtime-api.ts` 已经定义 `genehubRuntimeApi`，但 `src/preload/index.ts` 的 contextIsolation 分支没有：

```ts
contextBridge.exposeInMainWorld("genehubRuntime", genehubRuntimeApi)
```

### 修改要求

文件：

```text
src/preload/index.ts
```

```ts
contextBridge.exposeInMainWorld("genehubRuntime", genehubRuntimeApi);
```

建议放在：

```ts
contextBridge.exposeInMainWorld("mcpSkillGatewayRuntime", mcpSkillGatewayRuntimeApi);
```

之后。

### 验收

1. 打开 Hermes Skill Center 不再报 `probeConnection` undefined。
2. DevTools 执行 `window.genehubRuntime` 能看到 API 对象。
3. `window.genehubRuntime.probeConnection` 是 function。

更新 `mapSkill()`：

```ts
function mapSkill(raw: Record<string, unknown>): GeneHubSkill {
  const permissionsList = Array.isArray(raw.permissions)
    ? raw.permissions.map(String)
    : [];

  const permissionsObj =
    raw.permissions && !Array.isArray(raw.permissions)
      ? (raw.permissions as Record<string, unknown>)
      : {};

  const installedStatus = String(raw.installed_status ?? raw.installedStatus ?? "");

  return {
    geneSlug: String(raw.gene_slug ?? raw.geneSlug ?? raw.slug ?? ""),
    geneVersion: String(raw.gene_version ?? raw.geneVersion ?? raw.version ?? ""),
    skillName: String(raw.skill_name ?? raw.skillName ?? raw.slug ?? ""),
    displayName: String(raw.display_name ?? raw.displayName ?? raw.name ?? raw.slug ?? ""),
    description: String(raw.description ?? raw.short_description ?? ""),
    category: raw.category ? String(raw.category) : undefined,
    tags: Array.isArray(raw.tags) ? raw.tags.map(String) : [],
    installed: Boolean(raw.installed ?? installedStatus === "installed"),
    installedVersion: raw.installed_version
      ? String(raw.installed_version)
      : raw.installedVersion
        ? String(raw.installedVersion)
        : undefined,
    updateAvailable: Boolean(raw.update_available ?? raw.updateAvailable),
    permissions: {
      canInstall:
        permissionsList.includes("install") ||
        Boolean(permissionsObj.can_install ?? permissionsObj.canInstall),
      canUpdate:
        permissionsList.includes("update") ||
        Boolean(permissionsObj.can_update ?? permissionsObj.canUpdate),
      canUninstall:
        permissionsList.includes("uninstall") ||
        Boolean(permissionsObj.can_uninstall ?? permissionsObj.canUninstall),
    },
  };
}
```

### 验收

3. 有 install 权限时按钮可用。

## 5.11 P2：mapBundle 兼容后端 bundle 格式

### 问题

Desktop 当前认为：

```ts
raw.files 是数组
raw.scripts 是数组
```

但后端可能返回：

```json
{
  "files": {
    "skills/contact-to-order/SKILL.md": "..."
  }
}
```

### 修改要求

文件：

```text
src/main/genehub/genehub-client.ts
```

`mapBundle()` 需要同时支持：

格式 A：数组格式，推荐：

```json
{
  "files": [
    {
      "relative_path": "skills/contact-to-order/SKILL.md",
      "content": "...",
      "encoding": "utf-8"
    }
  ],
  "scripts": [
    {
      "relative_path": "scripts/parse_order.py",
      "content": "...",
      "encoding": "utf-8"
    }
  ]
}
```

格式 B：dict 格式，兼容：

```json
{
  "files": {
    "skills/contact-to-order/SKILL.md": "..."
  }
}
```

实现 helper：

```ts
function normalizeFiles(input: unknown): Array<{
  relativePath: string;
  content: string;
  encoding: "utf-8" | "base64";
}> {
  if (Array.isArray(input)) {
    return input.map((f) => ({
      relativePath: String((f as any).relative_path ?? (f as any).relativePath ?? ""),
      content: String((f as any).content ?? ""),
      encoding: ((f as any).encoding as "utf-8" | "base64" | undefined) ?? "utf-8",
    }));
  }

  if (input && typeof input === "object") {
    return Object.entries(input as Record<string, unknown>).map(([relativePath, content]) => ({
      relativePath,
      content: String(content ?? ""),
      encoding: "utf-8",
    }));
  }

  return [];
}
```

### 验收

1. 后端返回数组格式时可安装。
2. 后端返回 dict 格式时可安装。
3. `bundle.files.length` 正确。
4. `SKILL.md` 能被正确写入。
