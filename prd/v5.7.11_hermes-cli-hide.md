# v5.7.11_hermes_cli.md

## 1. 需求目标

解决 Windows 下通过 Desktop `src/renderer/src/modules/hermes-runtime` 启动 default Hermes Gateway 时出现 CMD 黑窗口的问题。

目标效果：

```text
SettingsDrawer → Hermes Runtime → Gateway → Start
```

启动 default profile 后：

```text
不显示 d:\SMC-Copilot\desktop\runtime\hermes\venv\Scripts\hermes.exe CMD 窗口
Gateway 仍正常监听 127.0.0.1:8642
Hermes chat / WebOperator / HermesTaskPanel 可正常调用
```

---

## 2. 当前源码结论

`HermesRuntimePanel` 实际只挂载 `HermesRuntimeSettings`。

`HermesRuntimeSettings` 中 Gateway 页签渲染 `HermesGatewaySection`。

`HermesGatewaySection` 点击 Start 后调用：

```ts
window.hermesAPI.startGateway()
```

该调用链来自 renderer 的 hermes-runtime 模块。

`preload/index.ts` 将 `startGateway` 映射到 IPC：

```ts
startGateway: () => ipcRenderer.invoke("start-gateway")
```



`main/index.ts` 中 `start-gateway` 最终调用 `startGateway()`。当前没有 profile 参数，默认启动 default profile。

问题点在 `src/main/hermes.ts`：

```ts
gatewayProcess = spawn(getHermesScript(), ["gateway"], {
  cwd: getHermesRepo(),
  env: gatewayEnv,
  stdio: "ignore",
  detached: true,
});
```

这里在 Windows 下使用 `getHermesScript()`，而 `getHermesScript()` 在 Windows 解析为 `runtime/hermes/venv/Scripts/hermes.exe`。当前 spawn 配置缺少 `windowsHide: true`，并且直接启动 `hermes.exe` console shim，所以会出现 CMD 窗口。 

同一项目中已经有可参考实现：`hermes-local-adapter.ts` 的 `buildHermesGatewaySpawnOptions` 已经设置了：

```ts
windowsHide: process.platform === "win32"
```



另外，`src/main/hermes.ts` 的 CLI fallback 已经说明 Windows 下不应直接依赖 `hermes.exe` shim，而是使用：

```ts
python -m hermes_cli.main
```

并设置：

```ts
windowsHide: true
```



---

## 3. 必须增加的参数

Windows 隐藏子进程控制台窗口的关键参数是：

```ts
windowsHide: true
```

该参数是 Node.js `child_process.spawn / execFile` 的进程创建参数，不是 Hermes CLI 参数。

错误方向：

```text
不要给 hermes gateway 增加 --hidden
不要使用 cmd.exe /c start /min
不要通过 powershell Start-Process -WindowStyle Hidden 包一层
```

正确方向：

```ts
spawn(command, args, {
  cwd,
  env,
  stdio: "ignore",
  detached: true,
  windowsHide: process.platform === "win32",
  shell: false,
});
```

## 5. 实施方案

## 5.1 在 `src/main/hermes.ts` 增加 Gateway spawn helper

新增函数位置：

```text
src/main/hermes.ts
放在 spawnHermesCli 前后均可，建议放在 CLI fallback 区域之前。
```

新增代码：

```ts
function spawnHermesGatewayProcess(
  cliArgs: string[],
  env: Record<string, string>,
): ChildProcess {
  const commonOptions = {
    cwd: getHermesRepo(),
    env,
    stdio: "ignore" as const,
    detached: true,
    windowsHide: process.platform === "win32",
    shell: false,
  };

  if (process.platform === "win32") {
    return spawn(
      getHermesPython(),
      ["-m", "hermes_cli.main", ...cliArgs],
      commonOptions,
    );
  }

  return spawn(getHermesScript(), cliArgs, commonOptions);
}
```

核心规则：

```text
Windows:
  command = getHermesPython()
  args    = ["-m", "hermes_cli.main", "gateway"]

非 Windows:
  command = getHermesScript()
  args    = ["gateway"]
```

原因：

```text
1. 避免直接启动 venv/Scripts/hermes.exe console shim
2. 使用 python -m hermes_cli.main 保持 CLI 行为一致
3. windowsHide 隐藏 Windows 子进程控制台窗口
4. stdio: "ignore" + detached + unref 保持后台运行
```

---

## 5.2 替换 `startGateway()` 中的 spawn

当前代码：

```ts
gatewayProcess = spawn(getHermesScript(), ["gateway"], {
  cwd: getHermesRepo(),
  env: gatewayEnv,
  stdio: "ignore",
  detached: true,
});
```

替换为：

```ts
gatewayProcess = spawnHermesGatewayProcess(["gateway"], gatewayEnv);
```

保留：

```ts
gatewayProcess.unref();
```

完整目标结构：

```ts
gatewayProcess = spawnHermesGatewayProcess(["gateway"], gatewayEnv);

gatewayProcess.unref();

gatewayProcess.on("close", () => {
  gatewayProcess = null;
  gatewayStartedByApp = false;
  apiServerAvailable = false;
  startHealthPolling();
});
```

## 6. 可选增强：统一 `hermes-local-adapter.ts`

当前 `hermes-local-adapter.ts` 已设置 `windowsHide`，但仍直接 spawn `hermesScript`。

为避免后续 profile runtime 也出现同类问题，可新增共享 helper。

### 6.1 新增文件

```text
src/main/hermes-process-spawn.ts
```

内容：

```ts
import { spawn, type ChildProcess, type SpawnOptions } from "child_process";
import { getHermesPython, getHermesRepo, getHermesScript } from "./installer";

export function buildHermesDesktopSpawnOptions(
  env: Record<string, string>,
  options?: Partial<SpawnOptions>,
): SpawnOptions {
  return {
    cwd: getHermesRepo(),
    env: {
      ...env,
      PYTHONUNBUFFERED: "1",
      PYTHONIOENCODING: "utf-8",
    },
    stdio: "ignore",
    detached: true,
    windowsHide: process.platform === "win32",
    shell: false,
    ...options,
  };
}

export function spawnHermesDesktopCli(
  cliArgs: string[],
  env: Record<string, string>,
  options?: Partial<SpawnOptions>,
): ChildProcess {
  const spawnOptions = buildHermesDesktopSpawnOptions(env, options);

  if (process.platform === "win32") {
    return spawn(
      getHermesPython(),
      ["-m", "hermes_cli.main", ...cliArgs],
      spawnOptions,
    );
  }

  return spawn(getHermesScript(), cliArgs, spawnOptions);
}
```

### 6.2 `src/main/hermes.ts` 使用

```ts
import { spawnHermesDesktopCli } from "./hermes-process-spawn";
```

替换：

```ts
gatewayProcess = spawnHermesDesktopCli(["gateway"], gatewayEnv);
```

### 6.3 `src/main/hermes-local-adapter.ts` 后续可统一

保持日志采集需要：

```ts
spawnHermesDesktopCli(
  ["gateway", "run", "--replace"],
  env,
  {
    stdio: ["ignore", "pipe", "pipe"],
  },
);
```

本次 v5.7.11 可只改 `src/main/hermes.ts`，后续再做统一抽取。

## 7. 验收标准

### 7.1 Windows 手工验收

环境：

```text
Windows 10+
SMC Copilot Desktop
本地 Hermes runtime 已安装
default profile 使用 8642
```

步骤：

```text
1. 关闭当前 CMD 窗口中的 hermes.exe
2. 打开 Desktop
3. 进入 SettingsDrawer → Hermes Runtime → Gateway
4. 点击 Stop
5. 点击 Start
6. 观察桌面是否出现 CMD 黑窗口
7. 点击 Status 刷新
8. 打开 WebOperator / Hermes 面板发送一条消息
```

通过条件：

```text
1. 不出现 d:\SMC-Copilot\desktop\runtime\hermes\venv\Scripts\hermes.exe CMD 窗口
2. gatewayStatus 返回 running
3. http://127.0.0.1:8642/health 可访问
4. Hermes chat 可正常响应
5. Desktop 退出后，stopGateway(true) 可停止 default gateway
```

---

### 7.2 进程检查

PowerShell：

```powershell
Get-Process python, hermes -ErrorAction SilentlyContinue | Select-Object Id,ProcessName,Path
```

预期：

```text
允许看到 python.exe 后台进程
不应弹出可见 CMD 窗口
```

端口检查：

```powershell
netstat -ano | findstr :8642
```

预期：

```text
127.0.0.1:8642 LISTENING
```

---

### 7.3 回归检查

```text
1. startGateway()
2. stopGateway(true)
3. restartGateway()
4. 首次 send-message 时 lazy-start gateway
5. gateway.pid 读取与清理
6. API_SERVER_KEY 自动生成
7. API_SERVER_ENABLED=true
```

当前 `send-message` 会在 gateway 未运行时 lazy-start `startGateway(profile)`，所以该路径也会复用本次隐藏窗口修复。
