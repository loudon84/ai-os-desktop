# PRD：v5.7.5_hermes_integration

## 1. 当前源码结论

`WebOperatorScreen.tsx` 当前已经用 `WebOperatorPageContextProvider` 包裹 WebOperator 页面，并通过 `WebOperatorPanels` 渲染右侧面板；`onFocusedPanelChange` 已经在 Screen 层具备切换侧面板的入口。 

`PageStructurePanel.tsx` 内部实际渲染 `[Get HTML]` 功能的位置不是主文件本身，而是 `PageFrameHtmlInspector`。`PageStructurePanel` 负责把 `selectedFrameId` 和 `frames` 传入该组件。

`PageFrameHtmlInspector.tsx` 当前在点击 `Get HTML` 后调用 `window.aiosBrowser.getFrameHtml`，成功后通过 `buildPageContextFromFrameHtml` 构造 `pageContext` 并写入 WebOperator 页面上下文。

`HermesTaskPanel.tsx` 当前只读取 `pageContext`，然后把它直接传给 `WebOperatorHermesChatPanel`，没有 `taskId / sessionId / pageUrl / action` 的任务会话控制。

`WebOperatorHermesChatPanel` 当前只接收 `pageContext`，内部通过 `useWebOperatorHermesPanelChat` 处理会话。

当前 Hermes 面板的会话绑定使用 `localStorage`，不是本地 SQLite 表。

Hermes 会话历史读取已经通过 `window.hermesAPI.getSessionMessages(sessionId)` 实现；会话 ID 当前是在 `chat-done` 后回写到前端状态。 

当前 `hermesDefaultChat` IPC 合约没有 `create empty conversation` 接口，也没有 `skills` 字段；`sendMessage` 是当前创建或续接会话的入口。 

## 2. 实施约束

1. `task_session` 必须走 Electron Main Process + IPC + Preload API，不允许 Renderer 直接访问 SQLite。
2. `task_session` 不写入 Hermes Agent 自有 `state.db`，新增 Desktop 自有 SQLite 文件，避免破坏 Hermes Agent 会话库结构。
3. `sessionId` 只能在 Hermes 首次实际发送消息后取得；`action='pending'` 不创建 Hermes 会话、不写入 `task_session`。
4. `skills` 字段在 v5.7.5 作为任务提示词元信息注入，不扩展 Hermes Gateway 的运行时 skill 强制调用协议。
5. `[Get HTML]` 只负责获取内容；新增 `[分析内容]` 才触发 Hermes 任务流程和侧面板切换。

# 3. 目标

在 WebOperator 的 Page Structure 面板中，把当前 CRM iframe / 页面 HTML 内容转为 Hermes 分析任务。

实现链路：

```text
Page Structure
  └─ Get HTML
       └─ Analyze Content
            ├─ 构造 pageContext
            ├─ 计算 pageUrl
            ├─ 激活右侧 Hermes Section
            └─ HermesTaskPanel 查询 task_session
                    ├─ 已存在 sessionId：加载历史会话
                    └─ 不存在 sessionId：显示任务对话框
                            ├─ 确认：提交 pageContext + userPrompt 给 Hermes
                            └─ 取消：只显示 Hermes 面板，不发送
```

# 5. 数据模型

## 5.1 SQLite 表

文件位置：

```text
<HERMES_HOME>/desktop/web-operator-task-session.db
```

表名：

```sql
CREATE TABLE IF NOT EXISTS task_session (
  task_id TEXT PRIMARY KEY,
  page_url TEXT NOT NULL UNIQUE,
  session_id TEXT NOT NULL UNIQUE,
  page_context_json TEXT NOT NULL,
  skill TEXT NOT NULL DEFAULT '',
  status TEXT NOT NULL DEFAULT 'active',
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_task_session_page_url
  ON task_session(page_url);

CREATE INDEX IF NOT EXISTS idx_task_session_session_id
  ON task_session(session_id);
```

## 5.2 taskId 生成规则

```ts
taskId = "wot_" + sha256(normalizePageUrl(pageUrl)).slice(0, 32)
```

`normalizePageUrl` 规则：

```text
1. trim
2. 保留 origin + pathname + search
3. iframe 为 about:srcdoc / about:blank 时：
   使用父页面 URL + "#framePath=<path>&frameTitle=<title>"
4. 不使用随机 ID
```

# 9. pageUrl 计算规则

新增文件：

```text
src/renderer/src/screens/WebOperator/utils/derive-page-url.ts
```

输入：

```ts
{
  frame: BrowserFrameSnapshot | null;
  frames: BrowserFrameSnapshot[];
  result: BrowserFrameHtmlResult;
}
```

输出：

```ts
string
```

规则：

```text
1. frame.url 是正常 http/https URL：
   pageUrl = frame.url

2. frame.url 是 about:srcdoc / about:blank：
   查找父 frame：
     - parent.path.length = frame.path.length - 1
     - parent.path 是 frame.path 的前缀
   pageUrl = parent.url + "#framePath=<frame.path>&frameTitle=<frame.title || frame.name>"

3. 无父 frame：
   pageUrl = result.url || frame.url || "unknown://web-operator-frame/<frameId>"
```

`buildPageContextFromFrameHtml` 增加 `pageUrl` 入参，避免 iframe `about:srcdoc` 被误当成任务唯一 URL。

# 12. Main Process 实现

## 12.1 `web-operator-task-session-store.ts`

职责：

```text
1. 初始化 Desktop WebOperator SQLite DB
2. 创建 task_session 表
3. resolve(pageUrl)
4. upsert(taskId, pageUrl, sessionId, pageContext, skill)
5. remove(taskId)
```

关键实现：

```ts
import Database from "better-sqlite3";
import { createHash } from "crypto";
import { existsSync, mkdirSync } from "fs";
import { join } from "path";
import { HERMES_HOME } from "./installer";

const DB_DIR = join(HERMES_HOME, "desktop");
const DB_PATH = join(DB_DIR, "web-operator-task-session.db");

function getDb(): Database.Database {
  if (!existsSync(DB_DIR)) mkdirSync(DB_DIR, { recursive: true });
  const db = new Database(DB_PATH);
  db.pragma("journal_mode = WAL");
  db.pragma("foreign_keys = ON");
  migrate(db);
  return db;
}

export function buildTaskId(pageUrl: string): string {
  return `wot_${createHash("sha256").update(pageUrl.trim()).digest("hex").slice(0, 32)}`;
}
```

# 14. 验收标准

## 14.1 Page Structure

```text
1. 点击 Get HTML：
   - 正常获取 iframe HTML
   - 不切换到 Hermes 面板

2. 点击 分析内容：
   - 成功获取当前 HTML
   - WebOperator 右侧面板切换到 Hermes Section
   - HermesTaskPanel 收到 pageContext + pageUrl
```

## 14.2 task_session

```text
1. 首次分析某 pageUrl：
   - task_session 查询为空
   - 显示任务对话框

2. 确认任务：
   - Hermes 自动发送 pageContent
   - chat-done 后写入 task_session
   - task_id 唯一
   - page_url 唯一
   - session_id 唯一

3. 再次分析同一 pageUrl：
   - 不显示任务对话框
   - 自动加载历史 Hermes 会话
```

## 14.3 pending

```text
1. 首次分析某 pageUrl
2. 任务对话框点击取消
3. Hermes 面板显示上下文
4. 不发送消息
5. 不写入 task_session
```

## 14.4 iframe srcdoc

```text
1. iframe.url = about:srcdoc
2. pageUrl 不允许直接使用 about:srcdoc
3. pageUrl 必须使用父页面 URL + framePath
4. 同一个父页面下不同 iframe 生成不同 taskId
```

## 14.5 skills

```text
1. skills 下拉默认显示 default
2. 安装了 skills 时显示已安装 skill
3. 选择 skill 后进入 pageContent 的 [技能] 字段
4. 不修改 Hermes Gateway skill 执行协议
```
