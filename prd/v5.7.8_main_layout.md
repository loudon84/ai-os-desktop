# copilot-desktop v5.7.8 Main Layout 全局 Overlay 与 Native Layer Gate PRD

## 1. 版本信息

版本号：`v5.7.8 MainLayout`

模块范围：

```text
src/renderer/src/screens/Layout/**
src/renderer/src/screens/MainPage/**
src/renderer/src/components/layout/**
src/renderer/src/components/shell/**
src/renderer/src/screens/WebOperator/**
src/shared/**
```

核心目标：

```text
解决 Electron WebContentsView 原生层遮挡 React Drawer/Dialog 的问题。
建立全局 DialogLayer / DrawerLayer / OverlayProvider。
所有阻塞型 Dialog / Drawer 打开时，统一隐藏 active WebContentsView。
关闭 Dialog / Drawer 后，WebContentsView 原页面无刷新恢复。
```

---

## 2. 背景问题

当前 WebOperator 使用 WebContentsView 承载外部 Web 页面，例如 CRM、ERP、Web 系统页面。

WebContentsView 属于 Electron 原生 View，不是 React DOM 节点。React 层的 Drawer/Dialog 即使 z-index 更高，也可能被 WebContentsView 遮挡。

当前已出现问题：

```text
点击 MainPage.tsx 中 setting / user profile / runtime setting 后，
SettingsDrawer 正常 open，
但 WebContentsView 仍处于 active 状态，
导致 Drawer/Dialog 被白色 WebContentsView 覆盖，无法正常交互。
```

局部解决方式已经存在：

```tsx
<WebContentsHost
  layerId={WEB_OPERATOR_LAYER_ID}
  className="web-operator-layout__viewport"
  enabled={enabled && !isTaskStartDialogOpen}
/>
```

该方式只解决 WebOperator 内部任务启动弹窗，不适用于全局 SettingsDrawer、ConfigDiffConfirmDrawer、后续 CRM ConfirmDialog、Browser Sensitive Action Dialog 等场景。

---

## 3. 设计结论

新增全局 Overlay 系统：

```text
OverlayProvider
  ├─ DialogLayer
  ├─ DrawerLayer
  ├─ NativeShellLayerGate
  └─ useDialog / useDrawer / useOverlayState
```

核心原则：

```text
Dialog/Drawer 是 React UI 层。
WebContentsView 是 Electron native 层。
不能只依赖 z-index。
必须通过 NativeShellLayerGate 统一隐藏 / 恢复 native WebContentsView。
```

最终效果：

```text
任何阻塞型 Dialog 或 Drawer 打开
  ↓
OverlayProvider 标记 nativeBlocked=true
  ↓
WebContentsHost effectiveEnabled=false
  ↓
window.shellView.hide(activeLayerId)
  ↓
React Dialog / Drawer 正常显示

Dialog / Drawer 关闭
  ↓
nativeBlocked=false
  ↓
WebContentsHost 重新 setBounds
  ↓
WebContentsView 恢复显示，不 reload，不 destroy
```

## 5. 总体架构

```text
Layout.tsx
  └─ OverlayProvider
       ├─ NativeShellLayerGateProvider
       │    └─ MainPage
       │         ├─ WorkspaceOutlet
       │         │    └─ WebContentsHost
       │         ├─ modalLayer
       │         │    └─ DialogLayer
       │         └─ drawerLayer
       │              └─ DrawerLayer
       │                   ├─ SettingsDrawer
       │                   └─ ConfigDiffConfirmDrawer
       │
       └─ overlay state
            ├─ dialogStack
            ├─ drawerStack
            ├─ hasBlockingDialog
            ├─ hasBlockingDrawer
            └─ nativeBlocked
```

运行链路：

```text
业务组件调用 useDialog().open()
  ↓
OverlayProvider 添加 DialogDescriptor
  ↓
DialogLayer 渲染 Dialog
  ↓
nativeBlocked=true
  ↓
WebContentsHost effectiveEnabled=false
  ↓
window.shellView.hide(layerId)

Dialog close
  ↓
DialogDescriptor 移除
  ↓
nativeBlocked=false
  ↓
WebContentsHost syncBoundsWithRetry()
  ↓
window.shellView.setBounds(layerId, bounds)
```

### 6.2 修改 Layout.tsx

文件：

```text
src/renderer/src/screens/Layout/Layout.tsx
```

目标：

```text
1. 引入 OverlayProvider。
2. 引入 DialogLayer。
3. 引入 DrawerLayer。
4. 引入 NativeShellLayerGateProvider。
5. 将 SettingsDrawer / ConfigDiffConfirmDrawer 接入 drawer blocking 状态。
6. 在 blocking 状态变化时，主动 hide 当前 active shell layer。
```

伪代码：

```tsx
const hasLegacyBlockingDrawer =
  settingsDrawerOpen || Boolean(pendingBootstrapDiff?.diff?.length);

useLayoutEffect(() => {
  if (!hasLegacyBlockingDrawer) return;

  const layerId = resolveActiveShellLayerId(navigation.view);
  if (!layerId) return;

  void window.shellView.hide(layerId).catch(() => {});
}, [hasLegacyBlockingDrawer, navigation.view]);

return (
  <OverlayProvider legacyDrawerBlocking={hasLegacyBlockingDrawer}>
    <NativeShellLayerGateProvider>
      <MainPage
        ...
        modalLayer={<DialogLayer />}
        drawerLayer={
          <>
            <DrawerLayer />
            <SettingsDrawer ... />
            <ConfigDiffConfirmDrawer ... />
          </>
        }
      />
    </NativeShellLayerGateProvider>
  </OverlayProvider>
);
```

短期保留 `settingsDrawerOpen` 在 Layout 中，后续再迁移 SettingsDrawer 到 `useDrawer()`。

### 6.3 修改 WebContentsHost.tsx

文件：

```text
src/renderer/src/components/shell/WebContentsHost.tsx
```

目标：

```text
1. 读取 nativeBlocked。
2. 生成 effectiveEnabled。
3. 所有 enabled 判断改为 effectiveEnabled。
4. nativeBlocked=true 时立即 hideLayer。
5. nativeBlocked=false 时重新 syncBoundsWithRetry。
```

关键逻辑：

```tsx
const { nativeBlocked } = useNativeShellLayerGate();

const effectiveEnabled = enabled && !nativeBlocked;
```

替换：

```tsx
if (!enabled) {
  await hideLayer();
  return false;
}
```

为：

```tsx
if (!effectiveEnabled) {
  await hideLayer();
  return false;
}
```

替换：

```tsx
useLayoutEffect(() => {
  if (!enabled) {
    void hideLayer();
    return;
  }
  ...
}, [enabled, ...]);
```

为：

```tsx
useLayoutEffect(() => {
  if (!effectiveEnabled) {
    void hideLayer();
    return;
  }
  ...
}, [effectiveEnabled, ...]);
```

保留：

```text
ResizeObserver
IntersectionObserver
syncBoundsWithRetry
hideLayer
error retry
```

## 7. API 设计

### 7.1 DialogDescriptor

```ts
export type DialogType =
  | "confirm"
  | "danger-confirm"
  | "web-operator-task-start"
  | "browser-sensitive-action"
  | "crm-command-confirm"
  | "custom";

export interface DialogDescriptor<TPayload = unknown, TResult = unknown> {
  id: string;
  type: DialogType;
  title?: string;
  payload?: TPayload;
  nativeBlocking?: boolean;
  closeOnEsc?: boolean;
  closeOnBackdrop?: boolean;
  createdAt: number;
  resolve?: (result: TResult) => void;
  reject?: (reason?: unknown) => void;
}
```

默认值：

```text
nativeBlocking=true
closeOnEsc=true
closeOnBackdrop=false
```

---

### 7.2 DrawerDescriptor

```ts
export type DrawerType =
  | "settings"
  | "config-diff-confirm"
  | "runtime-setting"
  | "profile-detail"
  | "custom";

export interface DrawerDescriptor<TPayload = unknown> {
  id: string;
  type: DrawerType;
  title?: string;
  payload?: TPayload;
  nativeBlocking?: boolean;
  createdAt: number;
}
```

默认值：

```text
nativeBlocking=true
```

---

### 7.3 OverlayState

```ts
export interface OverlayState {
  dialogs: DialogDescriptor[];
  drawers: DrawerDescriptor[];
  legacyDrawerBlocking: boolean;
  hasBlockingDialog: boolean;
  hasBlockingDrawer: boolean;
  nativeBlocked: boolean;
}
```

计算规则：

```ts
nativeBlocked =
  legacyDrawerBlocking ||
  dialogs.some((dialog) => dialog.nativeBlocking !== false) ||
  drawers.some((drawer) => drawer.nativeBlocking !== false);
```

---

### 7.4 useDialog

```ts
export interface DialogApi {
  open<TPayload = unknown, TResult = unknown>(
    input: Omit<DialogDescriptor<TPayload, TResult>, "id" | "createdAt">,
  ): Promise<TResult>;

  close(id: string, result?: unknown): void;

  dismiss(id: string, reason?: unknown): void;

  closeAll(): void;
}
```

示例：

```tsx
const dialog = useDialog();

const confirmed = await dialog.open({
  type: "confirm",
  title: "确认操作",
  payload: {
    message: "该操作会修改 CRM 页面内容，是否继续？",
  },
});
```

---

### 7.5 useDrawer

```ts
export interface DrawerApi {
  open<TPayload = unknown>(
    input: Omit<DrawerDescriptor<TPayload>, "id" | "createdAt">,
  ): string;

  close(id: string): void;

  closeAll(): void;
}
```

## 8. UI 分层规则

```text
MainPage__content
  z-index: normal

DrawerLayer
  z-index: 50

DialogLayer
  z-index: 60

ToastLayer
  z-index: 70
```

规则：

```text
1. Dialog 高于 Drawer。
2. Toast 高于 Dialog，但 Toast 不阻塞 native layer。
3. Tooltip / Popover 不进入全局 OverlayProvider。
4. 所有阻塞型 Dialog / Drawer 默认 nativeBlocking=true。
5. 非阻塞提示必须 nativeBlocking=false。
```

---

## 9. NativeShellLayerGate 规则

### 9.1 阻塞条件

以下任意条件成立时，`nativeBlocked=true`：

```text
1. SettingsDrawer open。
2. ConfigDiffConfirmDrawer open。
3. DialogLayer 存在 nativeBlocking dialog。
4. DrawerLayer 存在 nativeBlocking drawer。
5. 后续 Browser Sensitive Action Confirm open。
6. 后续 CRM Command Confirm open。
```

### 9.2 隐藏对象

隐藏当前 active shell layer：

```text
1. web-operator
2. portal
3. external-browser:<id>
```

不处理：

```text
1. 普通 React 页面
2. Local Hermes 页面
3. Workspaces React 页面
4. 非 shellView 页面
```

### 9.3 恢复方式

恢复只允许：

```text
window.shellView.setBounds(layerId, bounds)
```

禁止：

```text
destroy WebContentsView
reload WebContentsView
recreate WebContentsView
reload CRM page
clear session
```

## 10. 与 useShellLayerVisibility 的关系

当前 `useShellLayerVisibility` 负责隐藏非活跃 shell layer。

本版本新增职责：

```text
1. useShellLayerVisibility 继续处理 tab/view 切换。
2. NativeShellLayerGate 处理 overlay blocking。
3. 两者不互相替代。
```

Layout 层增加首帧兜底：

```tsx
useLayoutEffect(() => {
  if (!nativeBlocked) return;

  const layerId = resolveActiveShellLayerId(navigation.view);
  if (!layerId) return;

  void window.shellView.hide(layerId).catch(() => {});
}, [nativeBlocked, navigation.view]);
```

WebContentsHost 层负责长期状态：

```text
effectiveEnabled=false -> hideLayer
effectiveEnabled=true -> syncBoundsWithRetry
```

## 14. 风险与处理

### 风险 1：Drawer 打开瞬间仍有一帧遮挡

处理：

```text
Layout useLayoutEffect 中先主动 hide active shell layer。
WebContentsHost 后续通过 effectiveEnabled 保持隐藏状态。
```

---

### 风险 2：WebContentsView 恢复时 bounds 错误

处理：

```text
保留 WebContentsHost 现有 syncBoundsWithRetry。
恢复后继续使用 ResizeObserver / IntersectionObserver。
```

---

### 风险 3：部分 Dialog 不需要隐藏 WebContentsView

处理：

```text
DialogDescriptor.nativeBlocking=false。
例如 Toast、轻提示、非阻塞 popover 不进入 DialogLayer。
```

---

### 风险 4：多个 Dialog / Drawer 同时打开

处理：

```text
nativeBlocked 只要有一个 blocking overlay 即为 true。
关闭最后一个 blocking overlay 后恢复。
DialogLayer 高于 DrawerLayer。
```

---

### 风险 5：SettingsDrawer 仍是 legacy 状态

处理：

```text
本版本通过 legacyDrawerBlocking 接入。
后续版本再迁移 SettingsDrawer 到 useDrawer。
```
